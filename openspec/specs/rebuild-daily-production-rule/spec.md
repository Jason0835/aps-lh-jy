# 任务：重建硫化排程日账本语义：dayN 节奏账本 + 硫化余量消费账本

## Purpose

现有硫化排程项目中，很多逻辑是基于月计划 `dayN` 建“日计划账本”，并按天消费 `dayN` 进行排产。现在需要重建日账本语义：

> `dayN` 不再作为实际排产量账本，也不再直接限制当天实际排产量。
> 实际排产应以 SKU 的硫化余量为主消费账本，按机台日/班次有效产能逐步抵扣。

本次调整目标：

```text
dayN = 节奏账本
硫化余量 = 实际消费账本
机台 = 按天/班次有效产能排产
机台数量 = 由加机台规则或续作降模减机台规则决定
```

适用场景：

```text
续作排产
换活字块排产
新增排产
```

## Requirements

### Requirement: dayN 不得截断非收尾实际排产量

系统必须（MUST）仅使用 dayN 判断排产资格和机台节奏；非收尾 SKU 已被判定为生产保留机台后，实际排产量必须由硫化余量、日标准量和机台真实有效产能共同决定。

#### Scenario: 中间零 dayN 不得清空生产保留机台班次量

- **GIVEN** 非收尾续作 SKU 的窗口日计划为 `230,0,230`
- **AND** SKU 硫化日标准产量为 46、班产为 16、硫化余量充足
- **AND** 后看日计划仍需要保留 5 台续作生产机台
- **AND** 这些机台没有停机、清洗、保养、班次管控或停产保机约束
- **WHEN** 系统分配中间零 dayN 业务日的续作班次计划量
- **THEN** 每台生产保留机台的晚班、早班、中班计划量必须为 `16、16、14`
- **AND** 5 台机台当日合计计划量必须为 230
- **AND** 系统不得仅因当日 dayN 为 0 将生产保留机台的晚班或早班裁成 0
- **AND** dayN 仍只参与续作增机台、降模和停产保机的机台节奏判断

---

## 一、总原则

### 1. dayN 的新定位

月计划 `dayN` 只作为“节奏账本”，用于以下判断：

* SKU 是否具备当前窗口排产资格；
* SKU 最早可从哪一天开始排产；
* 是否允许提前生产；
* 是否需要新增机台；
* 续作是否需要加机台；
* 续作是否需要降模减机台；
* 欠产未超过阈值时的逐日后看判断；
* 欠产超过阈值时的强制增机台判断；
* 结构机台数节奏判断。

`dayN` 不再用于：

* 截断当天实际排产量；
* 作为当天必须排多少的实际目标；
* 作为真实排产消费账本；
* 限制非收尾 SKU 排满机台有效产能；
* 按 `dayN - 已排量` 计算最终可排量。

禁止继续使用类似逻辑作为最终排产截断：

```java
当天剩余可排量 = dayN - 当天已排量;
```

如果项目中仍有类似逻辑，只能保留在“节奏判断 / 是否加机台 / 是否降模”的判断链路中，不能作为最终排产量上限。

---

### 2. 硫化余量的新定位

SKU 实际排产量以“硫化余量消费账本”为准。

硫化余量公式：

```text
硫化余量 = MAX(月计划总量 - 已完成量 + 上月超欠产量, 0)
```

其中：

```text
已完成量 = 月累计完成量（截至 T-1） + T 日排程晚班完成量
```

`上月超欠产量` 根据月计划 `lastMonthValidFlag` 判断：

```text
lastMonthValidFlag = 1 时，取 lastMonthOverdueQty
否则取 0
```

排程过程中，每次排产后按 SKU 维度扣减：

```text
SKU 剩余硫化余量 = SKU 剩余硫化余量 - 本次排产量
```

同一 SKU 多机台排产时，必须共享同一个 SKU 硫化余量消费账本，不能每台机台各自维护独立 dayN 账本。

---

## 二、核心语义

必须实现以下语义：

```text
能不能排：看 dayN / 提前生产 / 后续计划 / 欠产 / 收尾 / 规则资格
排多少：看硫化余量、收尾目标量、胎胚库存、班次计划量、机台有效产能
要几台：看加机台规则或续作降模减机台规则
```

不能完全忽略 dayN。

例如：

```text
某 SKU 在 T～T+27 的 dayN 都为 0，
T+28 才开始有月计划量，
即使当前存在硫化余量，也不能提前很多天上机排产。
```

这类 SKU 应列入未排或等待后续窗口，除非明确满足 SKU 提前生产规则。

---

## 三、窗口和班次

硫化排程窗口仍为 T～T+2 三天，共 8 个班次：

```text
class1：T 日早班
class2：T 日中班
class3：T+1 晚班
class4：T+1 早班
class5：T+1 中班
class6：T+2 晚班
class7：T+2 早班
class8：T+2 中班
```

排程结果仍按班次落表。
业务汇总可按天统计。
实际消费账本按 SKU 硫化余量扣减。

## 五、非收尾 SKU 规则

### 1. 非收尾 SKU 不受 dayN 截断

非收尾 SKU 在具备排产资格后：

```text
不再严格按 dayN 当天日计划量排产。
```

排产量应按以下逻辑计算：

```text
本次可排量 = MIN(SKU 剩余硫化余量, 机台有效产能, 修正后的班次计划量/场景允许产能)
```

注意：

```text
dayN 只判断是否需要加机台、是否具备排产资格；
不能限制非收尾 SKU 实际排产量。
```

---

### 2. 单机台非收尾

新增、续作、换活字块中的单机台非收尾 SKU：

* 只要 SKU 有硫化余量；
* 机台窗口内有有效产能；
* 不违反换模、模具、胎胚、单控、停机等约束；

应尽量排满窗口可用班次产能。

不能因为当天 `dayN` 较小就少排。

---

### 3. 多机台非收尾

多机台数量由现有加机台规则决定。

确定机台数后：

* 每台机台按班次计划量和有效产能排；
* 总排产量从 SKU 硫化余量中扣减；
* 不再用 dayN 强行截断；
* 若 SKU 余量不足以支撑所有机台排满，则进入收尾判断或释放机台。

---

## 六、加机台失败或受限时的补充规则

适用场景：

```text
续作排产
换活字块排产
新增排产
```

### 1. 触发条件

如果某 SKU 根据加机台规则判断需要继续增加机台，但实际由于以下约束导致无法再增加机台：

* 无可用产能机台；
* 候选机台不可用；
* 模具数量不足；
* 模具被占用；
* 单控机台约束不满足；
* 停机或禁排；
* 换模/换活字块次数限制；
* 胎胚、规格、结构等其他规则约束；

则不能因为“理论上需要加机台但实际加不上”而导致当前已排机台少排。

### 2. 非收尾处理

如果 SKU 非收尾，且已经有主机台或已经成功增加的机台：

```text
当前已占用/已成功增加的机台，在 SKU 仍有硫化余量的情况下，
需要继续排满剩余可排班次计划量/机台有效产能。
```

也就是：

```text
加不上更多机台时，现有机台要尽量吃满可用产能；
不能因为缺少新增机台，就让主机台或已加机台提前停止。
```

### 3. 收尾处理

如果 SKU 为收尾，则必须严格按收尾目标量排产：

```text
本次可排量 = MIN(收尾目标剩余量, 机台有效产能)
```

普通收尾不允许超排。

但以下特殊场景按项目已有规则允许例外：

* 硫化余量为奇数，需要向上调整；
* 主销产品 SKU 收尾补满；
* 共用胎胚 SKU 收尾错峰后延；
* 晚班不可换模导致的补满；
* 其他 OpenSpec 中已定义的特殊收尾例外。

相关规则以以下 spec 为准：

```text
openspec/specs/main-sale-ending-fill/spec.md
openspec/specs/shared-embryo-ending-stagger/spec.md
openspec/specs/sku-end-judgment-rule/spec.md
openspec/specs/daily-standard-shift-plan/spec.md
```

## 七、收尾 SKU 规则

SKU 收尾判断规则以以下 OpenSpec 为准：

```text
openspec/specs/sku-end-judgment-rule/spec.md
```

本次调整需要保证：

* 收尾判断不能只在排产前一次性打死；
* 每排完一个 SKU / 机台 / 业务日后，都需要动态判断 SKU 是否收尾；
* 同一 SKU 多机台排产时，需要共享余量账本并动态判断整体 SKU 是否收尾；
* 预计收尾标识和实际收尾标识需要区分；
* 实际排产过程中，应以动态判断结果为准。

---

### 1. 单胎胚 / 非共用胎胚收尾

收尾目标量：

```text
目标量 = MAX(硫化余量, 胎胚库存)
```

排产时：

```text
本次可排量 = MIN(目标量剩余, 机台有效产能)
```

---

### 2. 共用胎胚收尾

共用胎胚收尾时：

```text
目标量 = 硫化余量
```

不再用胎胚库存抬高目标量。

如果：

```text
硫化余量 = 0
```

则：

```text
不排产，列入未排结果。
```

---

### 3. 共用胎胚动态剔除

共用胎胚组内，如果某个 SKU：

* 已收尾完成；
* 余量为 0 不排产；
* 本窗口不再参与排产；

则后续判断时必须从共用组中剔除。

如果剔除后只剩一个 SKU 使用该胎胚，则剩余 SKU 应重新识别为单胎胚，按：

```text
目标量 = MAX(硫化余量, 胎胚库存)
```

排产。

## 十、续作排产规则

### 1. 续作产量按硫化余量扣减

续作 SKU 不再按 dayN 日账本消费。

每台续作机台排产量：

```text
排产量 = MIN(SKU 剩余硫化余量, 机台有效产能, 场景目标剩余量)
```

非收尾续作：

```text
尽量排满窗口有效产能。
```

收尾续作：

```text
严格按收尾目标量，不超排。
```

特殊例外按对应 OpenSpec 执行。

---

### 2. 续作加机台

续作加机台继续复用现有加机台规则。

判断顺序：

1. 判断当前续作机台数是否满足当前日月计划量；
2. 若满足，则当前日不加机台，也不继续后看；
3. 若不满足，再进入逐日后看；
4. T 看 T+1；
5. T+1 看 T+2；
6. T+2 看 T+3；
7. 若欠产超过阈值，进入强制增机台逻辑；
8. 加机台前必须校验候选机台、模具、换模、胎胚、单控、停机等约束；
9. 如果理论需要加机台，但实际加不上，则按“加机台失败或受限时的补充规则”处理。

注意：

```text
加机台只决定要不要多给机台，不决定实际只能排 dayN 数量。
```

SKU 加机台规则（适用新增/续作排产加机台）详见：openspec/specs/add-machine-rule/spec.md

---

### 3. 续作降模减机台

续作降模减机台仍按 dayN 节奏判断。

规则：

1. 同一 SKU 存在多台续作机台；
2. 后续日计划量下降，当前机台数明显过多；
3. 尝试减少机台；
4. 减少后仍能满足当前日计划节奏，则允许减少；
5. 如果减少后的机台刚好可以满足 T 日当前日计划量，则停止继续减少；
6. 被释放机台优先参与换活字块，再参与新增排产；
7. 减机台优先选择胶囊使用次数少的机台；
8. 胶囊次数相同，选择机台编码大的机台。

补充保护：

```text
降模减机台不能导致 SKU 在窗口内明显无法消化合理余量或造成明显收尾风险。
```

SKU 降模减机台规则详见：openspec/specs/continue-reduce-machine-rule/spec.md

## 十一、换活字块排产规则

### 1. 换活字块只处理窗口内真实收尾且仍有生产时间的机台

换活字块场景中：

```text
处理最新真实收尾或释放时间早于排程窗口最后班次结束时间的机台。
T、T+1、T+2 收尾机台均可参与，但换活字块后必须仍能生成正产能结果。
```

窗口边界保护：

```text
最新完工时间等于或晚于窗口最后班次结束时间时，不执行换活字块；
同一机台连续换活字块时，每轮按更新后的最新完工时间重新校验窗口上界；
实际切换后无法生成正产能结果时，沿用现有失败回滚，不生成越窗结果。
```

候选物料还必须满足“候选物料与机台当前物料不同”的硬约束。当前物料与候选物料相同时，MUST NOT 生成换活字块排程结果或模具交替计划。

S4.6 根据本次排程结果新生成模具交替计划时，正规换模与换活字块均 MUST 仅比较前后物料编码：两者均非空且相同则不生成计划，产品状态不参与比较。跳过计划后仍须更新机台滚动状态，保证后续真实换模计划的前物料和变更时间正确；滚动排程继承的既有计划不受本规则过滤。

---

### 2. 换活字块不释放模具

换活字块不重新分配模具：

```text
沿用当前机台在机实际模具号。
```

排程结果中的模具号继续写当前在机模具号。

---

### 3. 换活字块后的排产量

换活字块后的 SKU 同样按硫化余量账本扣减。

非收尾：

```text
尽量排满窗口有效产能。
```

收尾：

```text
严格按收尾目标量。
```

如果理论需要继续加机台，但实际由于机台、模具等约束加不上，则按“加机台失败或受限时的补充规则”处理。

---

## 十二、新增排产规则

### 1. 新增 SKU 排序不变

保留项目现有 SKU 排序规则，不因本次账本调整改变排序。

排序、优先级、锁交期、延误上机、结构五天内收尾、供应链优先级等规则，以项目现有逻辑和 `openspec/specs/` 下对应 spec 为准。

### 2. 候选机台优先空闲机台

如果 SKU 当天需要排产，且候选机台中存在当天可用空闲机台：

```text
必须优先选择空闲机台；
不应等待其他 SKU 收尾完成后再使用收尾释放机台。
```

只有空闲机台不可用、模具不足、规格不匹配、单控约束不满足等情况下，才考虑后续释放机台。

---

### 3. 新增加机台

新增排产加机台继续复用现有规则。

判断时 dayN 仅用于：

* 当前日机台数是否满足当前日月计划量；
* 是否需要逐日后看；
* 是否需要强制增机台；
* 提前生产日计划量前移后的节奏判断。

实际排产量不能被 dayN 截断。

如果理论需要加机台，但实际由于机台、模具等约束加不上，则按“加机台失败或受限时的补充规则”处理。

SKU 加机台规则（适用新增/续作排产加机台）详见：openspec/specs/add-machine-rule/spec.md


---

### 4. 新增非收尾 SKU 不按 dayN 截断

例如：

```text
day1 = 16
SKU 硫化余量 = 300
新增后只有 1 台机台
该机台窗口有效产能 = 96
```

如果该 SKU 为非收尾，并且具备排产资格，则应允许排到 96，而不是只排 16。

## 十三、特殊场景

### 1. T～T+2 全为 0，但 T+3～月底有计划

不能简单因为有硫化余量就上机。

必须判断是否满足 SKU 提前生产规则：

```text

```

如果不满足：

```text
不排产，列入未排或等待后续窗口。
```

---

### 2. T～月底都无计划，仅因欠产进入

如果 SKU 只是因为本月历史欠产进入新增排产，并且 T～月底后续月计划量均为 0：

* 最近一次已排过判断以“前日日完成量 > 0”为准；
* 若前日日完成量 > 0，本次不再排产；
* 列入未排结果；
* 避免连续滚动排程中重复二次上机。

---

### 3. 收尾小余量不排

如果 SKU 收尾，且：

```text
硫化余量 <= 收尾允许欠产偏差值
```

默认偏差值为 2。

并且满足前置条件，例如前日夜班未排满或无数据，则：

```text
本次不排产；
列入未排；
释放原机台；
释放机台优先参与换活字块，再参与新增排产。
```

具体以 SKU 收尾判断相关 spec 为准。

---

### 4. 晚班不可换模补满

如果机台收尾点刚好落在中班结束，下一个班是晚班，并且后续 SKU 因晚班不能换模无法上机：

```text
当前 SKU 需要继续把下一个晚班排满。
```

该规则用于避免晚班产能浪费，不能被 dayN 截断。

## 十八、验收标准

### 1. dayN 不再限制非收尾实际排产量

非收尾 SKU 具备排产资格后，实际排产量应按硫化余量、修正后的班次计划量和机台有效产能计算，不能被当天 dayN 截断。

### 2. dayN 仍然参与资格和节奏判断

不能出现“只要有硫化余量就提前任意上机”的问题。
月末才有计划的 SKU，不能在当前窗口违规提前上机。

### 3. 加机台与实际排产量解耦

加机台规则只决定需要多少机台。
实际排产量由硫化余量、班次计划量、收尾目标量和机台有效产能决定。

### 4. 加机台失败时现有机台继续排满

如果理论需要加机台，但因为机台不可用、模具不足等约束导致加不上：

* 非收尾 SKU：当前主机台/已加成功机台必须继续排满剩余可排班次；
* 收尾 SKU：严格排满收尾目标剩余量；
* 特殊超排例外按 OpenSpec 执行。

### 5. 收尾 SKU 严格目标量

普通收尾 SKU 不允许超目标量。
主销产品收尾补满、晚班不可换模补满、奇数余量修正等特殊例外继续保留。

### 6. 同 SKU 多机台正确扣减余量

同一个 SKU 多台机台排产时，必须共享同一个 SKU 硫化余量账本。
不能每台机台按 dayN 独立消费。

### 7. 共用胎胚动态刷新

共用胎胚组内 SKU 收尾、余量为 0、不排产后，必须动态剔除，并重新判断剩余 SKU 是否仍为共用胎胚。

### 8. 收尾状态动态判断

不能只在排产前打固定收尾标识。
每次排完 SKU / 机台 / 业务日后，都需要重新判断 SKU 是否收尾。

### 9. OpenSpec 兼容

以下规则必须保持兼容：

```text
openspec/specs/daily-standard-shift-plan/spec.md
openspec/specs/sku-early-production/spec.md
openspec/specs/main-sale-ending-fill/spec.md
openspec/specs/shared-embryo-ending-stagger/spec.md
openspec/specs/sku-end-judgment-rule/spec.md
openspec/specs/ 下其他相关规则
```

### 10. 性能可接受

修改后不得明显恶化排程性能。
不得出现 OOM。
需保留关键耗时日志，方便定位性能瓶颈。

## 二十、同物料多产品状态账本与落库语义

### 1. SKU 运行态唯一身份

月计划量、日计划节奏、历史欠产、硫化余量、目标量、已排量、已排机台数、收尾标记和胎胚 SKU 内部分配额度必须（MUST）以“物料编码+产品状态”作为 SKU 唯一身份。

月累计完成量、逐日完成量以及 `T_LH_SCHE_FINISH_QTY.CLASS1_FINISH_QTY` 对应的 T 日晚班完成量也必须（MUST）按“物料编码+产品状态”分别汇总和扣减；T 日晚班完成量的产品状态取 `CLASS1_LH_TYPE`，不得用一个状态的完成量抵扣另一状态的月计划或日计划额度。

同状态多机台必须共享同一 SKU 账本；不同状态必须使用独立账本，一个状态的扣减、恢复、回滚、收尾或未排不得改变另一状态。

### 2. 胎胚库存仍为共享物理资源

同物料 `S`、`T`、`X` 可独立参与排程与内部额度分配，但必须共同消费胎胚编码对应的组级库存账本。不得按产品状态复制或拆分多份物理库存，所有状态的累计消费不得突破实际胎胚库存。

### 3. 结果、未排和物理换产边界

- 同一批次允许同一物料的多个产品状态同时落库硫化排程结果。
- 未排结果必须写入产品状态，且只有“物料编码+产品状态”一致的未排记录才可查找、去重、归并或零计划清理。
- 同物料不同状态仍是同一物理物料；换模、换活字块、绿色胎换产冲突和机台当前物料判断必须继续只比较物料编码，不得因产品状态不同生成虚假换模或换活字块计划。

#### Scenario: 同物料多状态同批落库

- **WHEN** 同物料 `S`、`T`、`X` 均有正目标量并满足合法机台约束
- **THEN** 同一排程批次必须允许三个状态分别生成并落库结果
- **AND** 三个状态的目标量、已排量与未排量互不覆盖
- **AND** 三个状态共同消费的胎胚库存总量不得超库存上限

#### Scenario: 同物料不同状态未排分开记录

- **WHEN** 同物料两个产品状态均未能排产
- **THEN** 未排表必须生成产品状态明确的独立记录
- **AND** 一个状态的未排量和原因不得合并到另一状态
