# 单控机台动态单模/双模排产规则

## Purpose

规范硫化排程中单控物理机台 `K1501`、`K1502` 及其运行态侧别
`K1501L/R`、`K1502L/R` 的动态使用模式，确保同一个 SKU 在一次排程中始终使用冻结的单模或双模模式，
并保证候选、排产、扣账、降模、释放和结果校验使用同一口径。

## Requirements

### Requirement: 单控模式必须在本次排程开始时冻结

系统 MUST 在 S4.3 完成 SKU 归集、收尾标注、零余量剔除以及续作/新增分类后，且在 S4.4 续作开始前，
一次性冻结本次排程所有 SKU 的单控使用模式：

1. SKU 唯一统计键 MUST 包含物料编码和产品状态；
2. 初始待排量 MUST 优先使用进入排产链路时的实际硫化余量（`surplusQty`）；`surplusQty` 为 0 的特殊场景（停产收尾、胎胚库存收尾等）使用 `targetScheduleQty` 作为严格目标量；
3. 满排模式下仅代表窗口理论产能的 `targetScheduleQty` MUST NOT 覆盖 `surplusQty` 决定单模/双模；试制 SKU 的 `strictTargetQty=true` 不得使模式冻结跳过 `surplusQty`；
4. 模式冻结后，目标量临时改写、剩余量扣减、SKU 出队、补偿对象复制和换活字块回流 MUST NOT 触发重新计算；
5. 缺少冻结模式时 MUST 阻断对应单控排产，不得按当前剩余量或旧 SKU 类型规则兜底。

#### Scenario: 初始10条不得在剩余4条时切换单模

- **WHEN** SKU 进入本次排程时的初始目标量为10条
- **AND** SKU 已生产6条后剩余4条
- **THEN** 系统 MUST 继续按冻结的双模模式处理该 SKU

#### Scenario: 试制SKU小余量不得被理论产能误判为双模

- **WHEN** 试制 SKU 的 `strictTargetQty=true`
- **AND** `surplusQty=3`，`targetScheduleQty=160`（满排理论窗口产能）
- **THEN** 系统 MUST 使用 `surplusQty=3` 冻结模式
- **AND** 系统 MUST 冻结为单模（3 <= 4）
- **AND** 系统 MUST NOT 使用 `targetScheduleQty=160` 冻结为双模

### Requirement: 多个试制SKU必须优先强制单模

系统 MUST 使用施工阶段 `01`（`ConstructionStageEnum.TRIAL`）作为“试制 SKU”的唯一识别口径。
产品状态 `X` 继续用于示方匹配和结果落库，但 MUST NOT 作为单控规则中的第二套试制判断：

1. 冻结时只统计初始待排量大于0的不同试制 SKU；
2. SKU 必须至少存在一个通过现有机台、模具、胶囊、特殊物料和窗口静态准入的单控侧，才计入统计；
3. 统计不受本轮后续 SKU 动态占用和排产顺序影响；
4. 不同试制 SKU 数量大于等于2时，本次排程全部试制 SKU MUST 冻结为单模；
5. 多试制 SKU 规则 MUST 优先于初始待排量4条边界；
6. 试制 SKU 被排完或移出待排队列后，其他试制 SKU MUST NOT 切换为双模。

#### Scenario: 两个大余量试制SKU全部单模

- **WHEN** 存在试制 SKU A、B，初始待排量分别为20、10条
- **AND** 两个 SKU 均满足单控静态准入
- **THEN** A、B MUST 全部冻结为单模
- **AND** 系统 MUST NOT 因两者初始待排量均大于4条而改用双模

#### Scenario: 零待排量试制SKU不参与计数

- **WHEN** 存在三个试制 SKU
- **AND** 其中一个 SKU 初始待排量为0，另外两个 SKU 初始待排量大于0
- **THEN** 系统 MUST 只统计另外两个 SKU
- **AND** 本次有效试制 SKU MUST 命中强制单模规则

#### Scenario: 产品状态X不得独立定义试制类型

- **WHEN** SKU 产品状态为 `X`，但施工阶段不是 `01`
- **THEN** 该 SKU MUST NOT 计入不同试制 SKU 数量
- **AND** 当 SKU 施工阶段为 `01` 时，无论产品状态为何值，均按试制 SKU 参与单控规则

### Requirement: 其他SKU必须按4条边界判断模式

未命中多个试制 SKU 强制单模规则时，所有 SKU MUST 按冻结的初始待排量判断单控模式，不区分试制、量试、正规或小批量：

1. 初始目标量大于4条时使用双模；
2. 初始目标量小于等于4条时使用单模；
3. 小批量 `<=100条` 规则 MAY 继续用于 SKU 分类、排序和普通/单控机台偏好，但 MUST NOT 决定单模或双模。

#### Scenario: 4条和5条使用不同模式

- **WHEN** 试制 SKU A 初始目标量为4条
- **AND** 试制 SKU B 初始目标量为5条
- **AND** A、B 分别处于只有一个不同试制 SKU 的排程轮次
- **THEN** A MUST 使用单模
- **AND** B MUST 使用双模

#### Scenario: 正规小余量使用单模

- **WHEN** 正规 SKU 初始目标量为3条
- **THEN** SKU 使用单控机台时 MUST 只占用其中一侧

### Requirement: 双模候选必须按物理机台成组生成

冻结为双模的 SKU 使用单控机台时，系统 MUST 将同一物理机台的 L/R 两侧作为一个不可拆分排产单元：

1. L/R 两侧都必须通过既有机台、模具、胶囊、特殊物料和窗口约束；
2. 任意一侧被其他 SKU **仍在生产中**（未设置 `specEndTime`）的结果占用时，整台物理机台不得成为双模候选；已设置 `specEndTime`（机台已释放）的其它 SKU 结果不阻止候选，下游 `resolveMachineOccupationEndTime` 会取 L/R 两侧较晚的 `specEndTime` 作为新 SKU 的最早开工时间；
3. 最早上机时间 MUST 取两侧释放、维修、保养和清洗结束时间的较晚值；
4. L/R 两侧必须存在共同可生产时间段；
5. 候选集合中同一物理机台只保留一个左侧代表候选，实际落地必须同步生成两侧结果。

#### Scenario: 配对侧被未结束SKU占用时拒绝双模候选

- **WHEN** 双模 SKU A 匹配 `K1501`
- **AND** `K1501R` 已被 SKU B 占用且未设置 `specEndTime`（仍在生产中）
- **THEN** 系统 MUST NOT 生成 `K1501L` 或 `K1501R` 的单边候选

#### Scenario: 配对侧已释放时保留双模候选

- **WHEN** 双模 SKU A 匹配 `K1501`
- **AND** `K1501R` 曾被 SKU B 占用但已设置 `specEndTime`（机台已释放）
- **THEN** 系统 MUST 保留 `K1501` 作为双模候选
- **AND** 新 SKU 的最早开工时间 MUST 取 L/R 两侧较晚的释放时间

#### Scenario: 两侧释放时间不同

- **WHEN** `K1501L` 早于 `K1501R` 释放
- **THEN** 双模 SKU 的最早接续时间 MUST 取 `K1501R` 释放后的时间

### Requirement: 试制双模必须先尝试单控整组再尝试普通机台

本轮只有一个不同试制 SKU 且冻结初始待排量大于4条时，系统 MUST 按以下两个候选阶段选机：

1. 第一阶段只包含满足既有硬约束的 `K1501L/R`、`K1502L/R` 整组候选；
2. 系统 MUST 先尝试完全部单控整组，任一整组可承接时 MUST NOT 提前选择普通机台；
3. 全部单控整组不可用、无法选中或排产试算失败后，第二阶段 MAY 选择满足既有约束的普通机台；
4. 单控整组和普通机台 MUST NOT 混合后交给通用排序直接选择；
5. 试制 SKU 使用普通机台时仍 MUST 遵守早班完成换模或换活字块、中班开始生产、固定占用中班2小时首检产能和晚班禁止换模规则。

#### Scenario: 单控整组可用时不得提前选择普通机台

- **WHEN** 单个试制 SKU 初始待排量为10条并冻结为双模
- **AND** `K1501L/R` 整组和普通机台均满足候选约束
- **THEN** 系统 MUST 先选择 `K1501L/R` 整组
- **AND** 系统 MUST NOT 因普通机台在通用排序中更优而提前选择普通机台

#### Scenario: 全部单控整组失败后允许普通机台

- **WHEN** 单个试制 SKU 初始待排量为10条并冻结为双模
- **AND** `K1501L/R`、`K1502L/R` 整组均不可用或排产试算失败
- **AND** 存在满足既有约束的普通机台
- **THEN** 系统 MAY 使用该普通机台排产
- **AND** 试制生产时间和首检规则 MUST 保持不变

### Requirement: 双模全链路必须保持L/R同步

冻结为双模的 SKU 在新增、续作、换模、换活字块、加机台、降模、收尾和释放链路中 MUST 保持 L/R 同步：

1. 两侧必须安排相同物料、相同生产起止时间和相同班次计划量；
2. 任意一侧资源预占或结果构建失败时，另一侧 MUST 同步回滚；
3. 双模续作缺少同物料配对侧或配对侧物料不同时 MUST 阻断单边续作并记录未排原因；
4. 双模降模只能整组保留或整组释放，不得只保留其中一侧；
5. 双模换活字块 MUST 同步生成 L/R 结果，不得继续使用“整机直接阻断单边换活字块”的旧行为；
6. 双模加机台时 L/R 算一个物理排产单元，产能按两侧合计；
7. 同 SKU 多机台收尾归集、尾量错峰、辅助机台释放和失败回填 MUST 将双模 L/R 视为同一物理组，禁止把两侧数量归集成“单侧有量、配对侧为0”。

#### Scenario: 双模换活字块同步落地

- **WHEN** 双模 SKU 使用 `K1502` 执行换活字块
- **AND** `K1502L/R` 均满足约束
- **THEN** 系统 MUST 同时生成 `K1502L` 和 `K1502R` 两条相同物料、时间和班次量的结果

#### Scenario: 双模收尾尾量不得归集到单侧

- **WHEN** 双模 SKU 在同一收尾班次中 `K1501L/R` 各排5条
- **AND** 同 SKU 多机台收尾归集开始执行
- **THEN** 系统 MUST 保持 `K1501L/R` 各5条
- **AND** 系统 MUST NOT 将结果改写为 `K1501L=10、K1501R=0`

### Requirement: 普通"同SKU多机台收口"不得拆散双模物理组

同 SKU 多机台收尾归集、尾量错峰、辅助机台释放和失败回填 MUST 将双模 L/R 视为同一物理组：

1. 分配数量时整组参与，写回时均分到 L/R；
2. 任何一侧不得被归零或单独增量；
3. 辅助机台释放时 L/R 整组同步释放；
4. 失败回填时 L/R 整组同步回滚。

#### Scenario: 双模收尾后机台可被后续SKU复用

- **WHEN** 双模 SKU A 在 `K1501L/R` 排产至 17:35 收尾释放
- **AND** 双模 SKU B 随后排产
- **THEN** 系统 MUST 保留 `K1501` 作为 SKU B 的双模候选
- **AND** SKU B 的最早开工时间 MUST 取 17:35 后加换模时间

### Requirement: 双模计划量必须按整组口径消费

系统 MUST 复用现有双模合计量口径消费日计划和实际待排账本：

1. 排产结果仍分别写入 L/R，每侧保存单侧计划量；
2. 扣账前 MUST 先将两侧合并为整组总量，只消费一次日计划和 SKU 剩余账本；
3. 回裁后 MUST 将整组数量均分回 L/R；
4. 只允许落地可均分到两侧的偶数总量，奇数尾量留给未排或后续滚动排程。

#### Scenario: 双模结果不得重复扣减

- **WHEN** `K1501L/R` 每侧排产5条相同 SKU
- **THEN** SKU 待排量 MUST 合计扣减10条
- **AND** 系统 MUST NOT 按两条结果分别扣减10条

### Requirement: 单模必须允许两侧独立使用

冻结为单模的 SKU 使用单控机台时，系统 MUST 允许 L/R 两侧独立排产：

1. `K1501L`、`K1501R`、`K1502L`、`K1502R` MUST 作为独立排产单元；
2. 一侧被占用 MUST NOT 阻止另一侧成为单模候选；
3. 左右两侧 MAY 安排不同 SKU，也 MAY 安排相同 SKU；
4. 续作、换模、换活字块、加机台、降模、收尾和释放 MAY 单边独立执行。

#### Scenario: 单模一侧占用不影响另一侧

- **WHEN** 单模 SKU A 已占用 `K1501L`
- **AND** 单模 SKU B 满足 `K1501R` 的全部业务约束
- **THEN** 系统 MUST 允许 SKU B 独立使用 `K1501R`
- **AND** 系统 MUST NOT 要求 A、B 为相同 SKU

### Requirement: 反向匹配候选必须使用冻结单模范围

单模 SKU 成功排入一侧后，系统 SHALL 为配对侧反向查找本次已冻结为单模的 SKU：

1. 候选范围 MUST NOT 再固定为试制、量试或小批量 `<=100条`；
2. 规格匹配层级保持同规格 > 同断面宽 > 同英寸；
3. 断面宽 MUST 复用 `ProductSpecificationsUtils`；
4. 同一规格层级内 MAY 继续使用现有 SKU 排序规则；
5. 候选仍需满足余量、日计划、模具、胶囊、首检和班次等既有约束；
6. 无合规候选时 MAY 保持配对侧空闲；
7. 推荐配对侧后 MUST 保留现有机台预留和非目标 SKU 排除机制，目标 SKU 选中后释放预留。

#### Scenario: 正规小余量SKU参与反向匹配

- **WHEN** 正规 SKU B 初始目标量为3条并冻结为单模
- **AND** B 与当前单模 SKU A 同规格且满足配对侧全部约束
- **THEN** B MUST 允许成为配对侧反向匹配候选

#### Scenario: 无规格匹配时允许空闲

- **WHEN** 待排单模 SKU 均与当前 SKU 不同规格、不同断面宽且不同英寸
- **THEN** 系统 MUST NOT 强制补位

### Requirement: 保存前必须校验冻结模式一致性

系统 MUST 在结果保存前按冻结模式校验配置生效的单控机台结果：

1. 双模结果必须存在同物料配对侧；
2. L/R 两侧生产开始时间、结束时间、各班次计划量、换模/换活字块标识和收尾状态必须一致；
3. 双模配对侧不得存在其他 SKU 的重叠结果；
4. 校验失败 MUST 抛出排程结果校验异常并禁止落库；
5. 单模结果不要求存在配对侧，但仍必须通过现有单侧时间和资源冲突校验。

#### Scenario: 双模单边结果保存前被拦截

- **WHEN** 冻结为双模的 SKU 仅在 `K1501L` 存在正排产量
- **THEN** 系统 MUST 在保存前抛出结果校验异常
