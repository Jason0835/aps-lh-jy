## Purpose

规范硫化排程继承前一排程日班次4、5模具交替计划中的“机台+后物料SKU”关系，确保反选在普通新增排产选机前执行，同时继续使用本批次真实机台状态重新校验资源、计算时间并更新排程账本。

## Requirements

### Requirement: 反选必须位于换活字块之后和普通新增选机之前

系统 MUST 按以下顺序执行排产：

1. 续作排产；
2. 换活字块排产；
3. 前日交替计划机台反选SKU；
4. 普通新增排产。

反选阶段 MUST 读取续作和换活字块完成后的真实机台状态，MUST NOT 覆盖前序已经生成的结果。S4.5 MUST 先完成普通新增SKU业务优先级排序，再执行历史反选编排和反选SKU前置；历史反选完成后、普通新增选机前 MUST NOT 再次执行会覆盖反选顺序的SKU排序。

#### Scenario: 前序排产后执行反选

- **GIVEN** 当前批次同时存在续作、换活字块、历史反选和普通新增SKU
- **WHEN** 系统执行硫化排程
- **THEN** 系统 MUST 先完成续作和换活字块
- **AND** 系统 MUST 在普通新增选机前处理历史反选指令

#### Scenario: 普通新增业务排序不得覆盖反选SKU顺序

- **GIVEN** 普通新增SKU `B` 的业务排序优先级高于历史反选SKU `A`
- **AND** 历史指令要求SKU `A` 优先尝试指定机台 `K1001`
- **WHEN** 系统进入S4.5新增排产
- **THEN** 系统 MUST 先完成普通新增SKU业务排序
- **AND** 系统 MUST 在排序完成后把历史反选SKU `A` 前置并登记指定机台
- **AND** 普通新增选机前 MUST NOT 再次排序而让SKU `B` 提前占用 `K1001`

### Requirement: 历史范围必须固定为目标日前一日的班次4和5

系统 MUST 按当前 `scheduleTargetDate - 1` 查询历史模具交替计划，并只保留换模或换活字块类型。系统 MUST 使用 `PLAN_DATE` 相对历史排程窗口起点还原班次，只处理班次4和5：

1. 历史班次4映射当前班次1；
2. 历史班次5映射当前班次2；
3. `PLAN_DATE` 为空时 MUST 跳过并记录原因；
4. 反选历史列表 MUST 与滚动排程的前日交替列表隔离，强制重排不得改变本规则的目标日前一日口径。

#### Scenario: 只加载目标日前一日班次4和5

- **GIVEN** 当前排程目标日为 `2026-07-14`
- **AND** `2026-07-13` 的交替计划包含班次3、4、5、6
- **WHEN** 系统构建反选指令
- **THEN** 系统 MUST 只保留班次4和5
- **AND** 班次4、5 MUST 分别映射为当前班次1、2

### Requirement: 反选必须固定机台和后物料但重新计算全部时间

系统 MUST 从历史计划读取机台编码和后物料编码，MUST NOT 使用前物料作为目标SKU。历史计划中的具体交替时间只用于排序和班次还原，MUST NOT 直接复制到当前排程结果。

当前批次 MUST 根据指定机台实际占用、停机、保养、换模资源、首检资源和班次产能重新计算：

1. 换模或换活字块开始时间；
2. 首检归属和首检时间；
3. 开产时间；
4. 各班次计划量；
5. 规格收尾时间。

映射班次是指定机台首次交替尝试班次，不是正规新增换模的成功硬约束。历史反选继承的核心关系是“机台+后物料”，因此停机、保养、晚班禁换模、换模配额、首检资源或 `SYS0303004` 导致实际换模班次后移时，系统 MUST 保留该关系，并在同一指定机台继续寻找下一合法换模、首检和开产班次。只有指定机台在本批排程窗口内最终无法通过真实硬约束时，正规新增反选才允许失败并回落其他机台。换活字块继续保持映射班次硬约束。

#### Scenario: 不复制历史具体时间

- **GIVEN** 历史班次4计划在 `2026-07-14 07:00` 切换
- **AND** 当前批次指定机台在 `2026-07-14 09:00` 才释放
- **WHEN** 系统执行反选
- **THEN** 系统 MUST 按当前机台释放时间重新分配切换和开产时间
- **AND** 系统 MUST NOT 直接复制 `07:00`
- **AND** 实际切换开始仍 MUST 位于当前班次1

#### Scenario: 本批资源约束导致班次后移时保留历史指定机台

- **GIVEN** 历史班次4指定SKU `A` 使用机台 `K1001`，当前映射班次为班次1
- **AND** 班次1因停机、换模配额、首检资源或 `SYS0303004` 无法合法完成本次上机
- **WHEN** 系统按本批真实资源重新计算换模和首检时间
- **THEN** 系统 MUST 回滚班次1本次失败预占
- **AND** 系统 MUST 保留 `K1001 + A` 历史关系，从班次2重新尝试同一机台
- **AND** 班次2满足全部硬约束时 MUST 在 `K1001` 生成SKU `A` 的换模和排程结果

### Requirement: 指定机台模式必须复用现有硬约束和排产主链

系统 MUST 为反选提供指定机台匹配模式。指定机台模式 MUST 跳过普通候选机台的最早收尾窗口筛选和排序，但 MUST 继续校验：

1. 定点不可作业、机台状态和可用窗口；
2. 寸口、模套、特殊材料和SKU基础适配；
3. 模具、模壳、胶囊、活字块及模具状态；
4. 单控整机或单边粒度；
5. 停机、保养、清洗和机台占用；
6. 晚班禁换模、换模或换活字块资源；
7. 首检、班次总量、日计划账本和机台产能；
8. SKU实际剩余量、收尾目标量和胎胚库存；
9. 其他现有新增或换活字块硬约束。

系统 MUST 复用现有换活字块和新增换模结果落地主链，MUST NOT 复制一套独立排产实现。

#### Scenario: 指定机台绕过普通排序但不绕过硬约束

- **GIVEN** 历史指定机台不在普通最早收尾候选窗口
- **AND** 该机台通过全部硬约束，并能从映射班次起点开始找到有效切换时间
- **WHEN** 系统执行反选
- **THEN** 系统 MUST 优先尝试该指定机台
- **AND** 系统 MUST NOT 重新参加普通候选排序

#### Scenario: 指定机台资源不足

- **GIVEN** 历史指定机台的目标SKU模具或胶囊资源不足
- **WHEN** 系统执行反选
- **THEN** 本条反选 MUST 失败
- **AND** 系统 MUST NOT 强制突破资源约束生成结果

### Requirement: 实际交替类型必须按当前机台状态重新判断

系统 MUST 按当前机台物料、胎胚、模具和活字块关系重新判断实际交替类型：

1. 满足现有换活字块条件时，委托现有换活字块指定机台入口；
2. 不满足换活字块条件时，登记为新增换模指定机台指令；
3. 换活字块已经进入实际资源分配但失败时，本次反选 MUST 直接失败，MUST NOT 在反选阶段静默降级为普通换模；
4. 反选失败后，SKU MAY 在普通新增阶段重新竞争其他机台。

#### Scenario: 历史换模在当前状态变为换活字块

- **GIVEN** 历史计划类型为换模
- **AND** 当前机台与后物料满足现有同胎胚、同模具和活字块切换条件
- **WHEN** 系统执行反选
- **THEN** 系统 MUST 按换活字块主链重新排产
- **AND** 当前模具交替计划 MUST 记录实际换活字块类型

### Requirement: 历史指令必须稳定排序并幂等处理

系统 MUST 按历史班次4、班次5、交替开始时间、机台编码、计划顺位和主键稳定处理：

1. 相同历史班次、机台和后物料只处理一次；
2. 同机台同一交替时刻出现不同后物料时，按计划顺位和主键保留第一条；
3. 其余冲突记录 MUST 记录明确失败原因；
4. 同一SKU存在多条机台指令时，每次处理 MUST 动态读取实际剩余账本；
5. 前一条已消费全部待排量时，后续指令 MUST 失败且不得重复消费。

#### Scenario: 重复历史数据只处理一次

- **GIVEN** 历史班次4存在两条相同机台和后物料记录
- **WHEN** 系统构建反选指令
- **THEN** 系统 MUST 只生成一条有效反选指令
- **AND** 重复记录 MUST 写入跳过原因日志

### Requirement: 多产品状态必须按当前优先级选择实际消费状态

历史模具交替计划保持物料级语义，不包含产品状态。系统 MUST 从同物料当前待排SKU中选择优先级最高且实际账本仍有正余量的产品状态：

1. 每条指令处理时动态读取当前“物料+产品状态”复合账本；
2. 空产品状态 MUST 按正规状态 `S` 归一化；
3. 成功后只消费最终选择状态的账本；
4. 其他产品状态不得被重复扣减。

#### Scenario: 同物料多状态选择有余量的优先状态

- **GIVEN** 后物料同时存在状态 `S` 和 `T`
- **AND** `S` 的当前排序更高且实际余量大于0
- **WHEN** 系统解析反选目标SKU
- **THEN** 系统 MUST 选择状态 `S`
- **AND** 排产成功后 MUST 只扣减状态 `S` 的运行态账本

### Requirement: 前序已满足时不得重复排产

系统 MUST 按“机台+后物料+当前映射班次”判断续作或换活字块前序结果是否已经满足历史关系：

1. 已满足时 MUST 标记反选成功；
2. MUST NOT 再次生成排程结果或消费SKU余量；
3. MUST 补充结果来源、机台登记和保护上下文；
4. 没有真实交替时 MUST NOT 伪造模具交替计划。

#### Scenario: 续作已经满足历史关系

- **GIVEN** 历史班次4指定机台 `K1001` 和后物料 `A`
- **AND** 续作结果已在当前班次1把物料 `A` 安排到 `K1001`
- **WHEN** 系统执行反选
- **THEN** 系统 MUST 标记已满足
- **AND** 系统 MUST NOT 重复生成结果或扣减待排量

### Requirement: 反选失败不得阻断普通新增排产

系统 MUST 在指定机台无产能、被占用、SKU无余量、硬约束不满足、资源不足或排程窗口内最终无法找到合法换模首检班次时执行以下处理：

1. MUST 保留续作和换活字块前序结果；
2. MUST 回滚本次切换、首检和模具预占；
3. MUST NOT 在反选阶段提前新增未排结果；
4. MUST NOT 把目标SKU移出普通新增待排队列；
5. 普通新增阶段 MUST 继续使用当前真实状态选择其他候选机台；
6. MUST 记录历史班次、映射班次、机台、后物料和失败原因。

同批次存在多条历史反选指令时，系统 MUST 采用“历史指定机台首次尝试、普通新增回落”两阶段顺序：任一有效历史指令尚未获得首次尝试机会时，当前SKU MUST 保留在待排队列并延后普通回落；全部历史指令完成首次尝试后，系统 MUST 从待排队列队首恢复普通新增。该顺序控制 MUST NOT 预占模具、换模、首检或机台资源，且 MUST NOT 提前写入未排结果。

历史优先轮次 MUST 保证确定性收敛：

1. 指定机台通过基础硬过滤、但被正式窗口产能计算判定为整个窗口无剩余产能时，系统 MUST 在当前轮把该指令标记为失败；
2. 系统完成一整轮待排队列遍历后，若当前队列关联的未尝试历史指令数量没有下降，系统 MUST 将这些不可执行指令结算失败；
3. 残留指令结算后，系统 MUST 在下一轮恢复普通新增候选竞争；
4. 系统 MUST NOT 通过无上限轮询或固定大轮次阈值等待指令状态自行变化；
5. 无进展结算 MUST NOT 新增未排结果，也不得修改机台、模具、换模、首检或计划量账本。

#### Scenario: 指定机台失败后普通新增选择其他机台

- **GIVEN** 历史指定机台在本批完整排程窗口内最终无合法换模首检产能而反选失败
- **AND** 目标SKU仍有待排量
- **AND** 另一台普通候选满足全部约束
- **WHEN** 系统继续执行普通新增排产
- **THEN** 目标SKU MUST 允许选择另一台机台
- **AND** 反选阶段 MUST NOT 提前写入未排结果

#### Scenario: 前一条反选失败必须延后到后一条指定机台尝试之后回落

- **GIVEN** 第一条历史指令指定SKU `A` 使用机台 `K1001`
- **AND** 第二条尚未执行的历史指令指定SKU `B` 使用机台 `K1002`
- **AND** SKU `A` 在 `K1001` 反选失败
- **WHEN** SKU `A` 准备进入普通新增回落
- **THEN** SKU `A` MUST 保留在待排队列且本轮不执行普通选机
- **AND** SKU `B` MUST 先获得一次尝试 `K1002` 的机会
- **AND** 第二条指令完成尝试后，SKU `A` MUST 在下一轮恢复普通候选竞争

#### Scenario: 指定机台无窗口产能时历史优先轮次必须收敛

- **GIVEN** 历史指令指定SKU `A` 使用机台 `K1001`
- **AND** `K1001` 通过指定机台基础硬过滤
- **AND** 正式窗口产能计算确认 `K1001` 在整个排程窗口的剩余产能为0
- **WHEN** 系统执行历史指定机台首次尝试
- **THEN** 系统 MUST 在当前轮将该历史指令标记为失败
- **AND** 系统 MUST NOT 在后续轮次重复计算相同的 `K1001 + A` 指令
- **AND** 全部历史指令结算后，SKU `A` MUST 恢复普通新增候选竞争

### Requirement: 反选成功必须同步完整状态并保护固定关系

反选成功后，现有排产主链 MUST 完整同步排程结果、机台状态、机台占用、SKU账本、模具及附属资源、单控配对、机台数统计、来源SKU和模具交替计划生成所需上下文。

后续普通新增 MUST NOT 让同一状态SKU重复选择已反选成功的同一机台。同SKU多机台收口 MUST NOT 清空、搬走或删除反选保护结果。其他SKU在机台真实释放后 MAY 按现有规则继续承接。

#### Scenario: 后置多机台收口保护反选结果

- **GIVEN** 同一SKU在反选机台和普通新增机台均生成结果
- **WHEN** 系统执行同SKU多机台尾量收口
- **THEN** 系统 MUST 保留反选机台与该SKU的排产关系
- **AND** 系统 MUST NOT 把反选结果的全部班次量搬到其他机台

### Requirement: 当前模具交替计划必须由本批次真实结果生成

系统 MUST 继续由结果校验阶段根据当前结果的真实 `mouldChangeStartTime`、前后物料和 `isTypeBlock` 生成模具交替计划：

1. MUST NOT 手工复制历史交替计划；
2. MUST NOT 复制历史交替时间；
3. 相同当前结果 MUST NOT 生成重复交替计划；
4. 当前交替类型 MUST 与实际换模或换活字块结果一致。

#### Scenario: 当前交替时间来自本批结果

- **GIVEN** 历史计划存在指定机台和后物料关系
- **WHEN** 本批反选成功并生成当前结果
- **THEN** 当前模具交替计划时间 MUST 来自本批结果的真实换模开始时间
- **AND** 当前时间 MAY 与历史计划时间不同
