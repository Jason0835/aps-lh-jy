# 计划性维修（05）设备停机计划规则

**状态：已实现** | **实现日期：2026-07-08** | **规则修订日期：2026-07-10** | **适用链路：硫化排程初始化、设备停机计划加载、班次产能扣减、续作/新增SKU区分**

## Purpose

统一硫化排程中"计划性维修"停机类型（`machine_stop_type = 05`）的处理口径。计划性维修期间机台完全离线，但不改变机台在机物料身份；维修前后均允许原物料继续以续作方式生产。维修时间段继续扣减停机产能，维修计划开始时间所在班次固定只排 2 条产能（不按时间折算），该值支持硫化参数配置。

本规则在已有设备停机统一扣产能逻辑基础上，仅为 05 保留固定排产量限制；05 不再写入维修就绪状态，也不触发强制换模或换活字块，不影响其他停机类型的处理口径。

## Requirements

### Requirement: 计划性维修必须从设备停机计划统一加载

系统 SHALL 从设备停机计划表（`T_MDM_DEVICE_PLAN_SHUT`）加载停机类型为 `05`（计划性维修）的记录，与精度校验、润滑、巡检点检等其他非清洗类停机类型使用相同的加载逻辑（按排程窗口交集加载）。

设备停机类型处理方式映射如下：

| 停机类型编码 | 业务含义 | 排程处理方式 |
| --- | --- | --- |
| `00` | 精度校验 | 统一扣时间产能 |
| `01` | 润滑 | 统一扣时间产能 |
| `02` | 巡检点检 | 统一扣时间产能 |
| `03` | 预见性维护 | 统一扣时间产能 |
| `04` | 预防性维护 | 统一扣时间产能 |
| `05` | 计划性维修 | 统一扣时间产能 + 保留原物料续作 + 维修开始班次固定排产量 |
| `06` | 临时性故障 | 统一扣时间产能 + 标记维修计划（维修后可续产） |
| `07` | 干冰清洗 | 转换为清洗窗口，独立扣减 |
| `08` | 喷砂清洗 | 转换为清洗窗口，独立扣减 |
| `09` | 盘点 | 统一扣时间产能，不触发换模/预热 |

加载口径：

```text
普通设备停机：按排程窗口交集加载（beginDate <= 窗口结束 AND endDate >= 窗口开始）
```

计划性维修属于普通设备停机，MUST 使用普通加载口径。

#### Scenario: 计划性维修停机计划被正确加载

- **Given** 设备停机计划表中存在机台 K1105 的计划性维修记录，停机类型为 `05`
- **And** 维修时间段 `2026-07-08 08:00:00 ~ 2026-07-08 16:00:00` 与排程窗口相交
- **When** 系统加载设备停机计划
- **Then** 该维修记录 MUST 被加载到 `context.getDevicePlanShutList()`

### Requirement: 计划性维修时间段必须统一扣减班次可生产计划量

系统 SHALL 将计划性维修时间段作为不可生产时间段，复用项目已有的停机扣产能逻辑（`ShiftCapacityResolverUtil.resolvePlannedStopOverlapSeconds` / `collectMergedPlannedStopIntervals`）统一折算扣减。

扣减规则：

```text
维修重叠秒数 = 维修时间段与班次时间窗的交集总秒数（多条记录先合并区间再计算）
班次可生产秒数 = 班次总秒数 - 所有非清洗类停机重叠秒数（含计划性维修）
班次最大计划量 = (可生产秒数 / 硫化时间秒数) 向下取整 * 模台数
```

计划性维修与其他非清洗类停机类型在时间产能扣减上 MUST 保持完全一致的处理口径。

#### Scenario: 计划性维修时间段扣除对应计划量

- **Given** 机台 K1105 在 `2026-07-08 08:00:00 ~ 16:00:00` 有计划性维修停机
- **And** 该时间段落在某班次窗口内
- **When** 系统计算该班次最大计划量
- **Then** 维修时间段（8 小时）MUST 从班次可生产时间中扣除
- **And** 班次最大计划量 MUST 按扣除后的可生产时间折算

#### Scenario: 跨班次维修时按实际重叠时间分别扣除

- **Given** 机台 K1105 在 `2026-07-08 12:00:00 ~ 2026-07-08 20:00:00` 有计划性维修停机
- **And** 早班窗口为 `06:00 ~ 14:00`，中班窗口为 `14:00 ~ 22:00`
- **When** 系统计算各班次最大计划量
- **Then** 早班 MUST 扣除 `12:00 ~ 14:00` 重叠时段（2 小时）
- **And** 中班 MUST 扣除 `14:00 ~ 20:00` 重叠时段（6 小时）

### Requirement: 计划性维修前后必须保留原物料续作

系统 SHALL 确保计划性维修（`05`）只改变停机时间段内的有效产能，不改变机台当前物料及续作身份。只要 MES 在机物料或滚动继承物料满足现有续作准入，维修前后均 MUST 以续作方式继续生产原物料。

在 `DataInitHandler` 初始化机台运行态时，计划性维修停机类型 MUST 保留 `planStopStartTime`、`planStopEndTime`、`stopType` 及原始 `devicePlanShutList` 停机记录，但 MUST NOT 设置 `hasRepairPlan` 和 `repairPlanTime`，避免从排程窗口起点提前抬高机台准备就绪时间。

在 `ScheduleAdjustHandler` 区分续作/新增 SKU 时，05 停机 MUST NOT 阻断 MES 在机物料或滚动继承物料的续作分类。维修时间段仍由 `ShiftCapacityResolverUtil` 的停机区间合并和班次产能计算统一扣减。

#### Scenario: 计划性维修仅保留停机窗口

- **Given** 机台 K1105 有计划性维修停机记录（停机类型 `05`）
- **When** 系统初始化机台运行态
- **Then** `planStopStartTime`、`planStopEndTime` 和 `stopType` MUST 正确保留
- **And** `hasRepairPlan` MUST 为 false
- **And** `repairPlanTime` MUST 为空
- **And** 机台准备就绪时间 MUST NOT 因 05 维修从窗口起点提前抬高

#### Scenario: 计划性维修前后原物料继续续产

- **Given** 机台 K1105 计划性维修前在机 SKU 为物料 A
- **And** 物料 A 满足现有续作准入条件
- **When** 系统区分续作/新增 SKU 并安排维修前后班次
- **Then** 物料 A MUST 保持续作类型
- **And** 维修时间段内产能 MUST 正确扣减
- **And** 维修结束后 MUST 允许继续排产物料 A
- **And** MUST NOT 因 05 维修生成换模、换活字块或首检

#### Scenario: 临时性故障维修后可继续续产

- **Given** 机台 K1105 有临时性故障停机记录（停机类型 `06`）
- **When** 系统初始化机台运行态
- **Then** `hasRepairPlan` MUST 为 true
- **And** 维修结束后机台 CAN 以续作方式继续排产

### Requirement: 计划性维修开始班次必须固定排产 2 条

系统 SHALL 为计划性维修配置固定排产量参数（`PLANNED_REPAIR_FIXED_QTY`，参数编码 `SYS0308010`，默认值 2 条）。当设备停机计划中存在 `05`-计划性维修且其计划开始时间落在某班次窗口内时，该班次 MUST 固定只排 `plannedRepairFixedQty` 条，不按时间折算。

固定排产量由 `ShiftCapacityResolverUtil.isPlannedRepairStartShift` 判断当前班次是否为维修开始班次，在 `resolveShiftCapacityWithDowntime` 核心方法中命中时直接返回固定排产量，跳过正常时间折算逻辑。

参数配置生效后，修改 `T_LH_PARAMS` 中 `SYS0308010` 的值即可调整固定损失量，无需改代码。

#### Scenario: 维修开始班次固定排产 2 条

- **Given** 机台 K1105 在 `2026-07-08 10:00:00` 开始计划性维修
- **And** 早班窗口为 `06:00 ~ 14:00`
- **And** 固定排产量参数配置为 2
- **When** 系统计算早班最大计划量
- **Then** 早班最大计划量 MUST 为 2 条（不按时间折算）
- **And** 中班和夜班 MUST 按正常时间折算计算产能

#### Scenario: 参数化配置生效

- **Given** 硫化参数 `SYS0308010` 配置为 3
- **When** 系统计算计划性维修开始班次最大计划量
- **Then** 该班次最大计划量 MUST 为 3 条（不按时间折算）

#### Scenario: 参数未配置时使用默认值

- **Given** 硫化参数 `SYS0308010` 未配置
- **When** 系统计算计划性维修开始班次最大计划量
- **Then** 该班次最大计划量 MUST 为 2 条（默认值，不按时间折算）

### Requirement: 计划性维修不得影响其他停机类型处理逻辑

系统 SHALL 确保计划性维修规则的引入不影响以下已有停机/占用时间逻辑：

1. 干冰清洗（`07`）的候选加载、班次归属、每日上限和重叠处理
2. 喷砂清洗（`08`）的候选加载、禁排日期和重叠处理
3. 精度校验（`00`）的停机扣减
4. 临时性故障（`06`）的维修计划标记和准备就绪时间抬高
5. 盘点（`09`）的停机扣减（不触发换模/预热/维修标记）
6. 保养窗口的触发、安排和产能扣减
7. 换模、换活字块的重叠处理和产能扣减

#### Scenario: 计划性维修与临时性故障并存时各自独立处理

- **Given** 机台 K1105 同时有计划性维修（`05`）和临时性故障（`06`）记录
- **When** 系统初始化机台运行态
- **Then** `hasRepairPlan` MUST 为 true
- **And** `repairPlanTime` MUST 只取临时性故障的开始时间
- **And** 计划性维修时间段仍 MUST 参与停机产能扣减

### Requirement: 计划性维修停机计划必须打印关键日志

系统 SHALL 在初始化机台运行态时，对每台机台的计机性维修停机计划打印日志，包含以下关键字段：

1. 机台编号
2. 维修开始时间
3. 维修结束时间

日志 MUST 简洁，不得打印敏感信息，不得用日志掩盖异常。

#### Scenario: 计划性维修停机计划日志输出

- **Given** 机台 K1105 有计划性维修停机记录
- **When** 系统初始化机台运行态
- **Then** 日志 MUST 输出机台编号、维修开始时间和维修结束时间
- **And** 日志 MUST 明确说明按停机窗口扣减产能并保留原物料续作
- **And** 日志级别为 INFO
