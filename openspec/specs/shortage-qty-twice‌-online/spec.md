新增排产补充规则：仅历史欠产、后续无计划、前日已生产过或前日排程结果已排过的 SKU，本次跳过排产。

请在新增排产逻辑中增加判断：

适用场景：
- SKU 仅因“本月前日累计欠产 > 0”进入新增排产；
- 从当前排程日 T 到月底，月计划 dayN 均为 0，即后续无计划量；
- 判断该 SKU 是否最近一次已排过时，优先根据当前排程日 T 前一日的日完成量判断：
  - 若前日日完成量 > 0，表示该 SKU 前日已生产过；
  - 若前日日完成量 = 0 或为空，则兜底判断接口排程日期前一日的硫化排程结果是否有有效排产量；
  - 若接口排程日期前一日硫化排程结果有有效排产量，也视为该 SKU 前日已生产过；
  - 若前日日完成量 = 0 或为空，且前日硫化排程结果也未排过，表示前日未生产过。

处理规则：
- 若满足“仅历史欠产 + T～月底无计划 + 前日日完成量 > 0 或前日硫化排程结果已排过”，则本次不再新增排产；
- 该 SKU 需写入未排结果；
- 未排原因建议为：仅历史欠产、后续无月计划，且前日已有完成量或前日排程结果已排过，本次跳过补排；
- 输出日志，包含 SKU、欠产量、排程日期、前日日完成量、前日排程结果是否已排过、跳过原因。

注意：
- 该规则覆盖仅历史欠产形成的新增收尾 SKU，但不影响续作补偿 SKU；
- 不能优先用“前日硫化排程结果是否排过”替代前日日完成量；前日排程结果只作为前日日完成量为 0 或为空时的兜底依据；
- 前日完成量按当前排程窗口 T 前一日判断；兜底用的前日硫化排程结果按接口传入排程日期前一日判断，例如接口传入 2026-06-14 时查询 schedule_date = 2026-06-13；
- 如果 T～月底仍有任意月计划量，则不适用该规则，继续走原新增排产逻辑；
- 如果前日日完成量为 0 或为空，且前日硫化排程结果也未排过，则允许继续按现有历史欠产补排逻辑判断；
- 不影响正常有计划 SKU、续作排产、换活字块排产。
