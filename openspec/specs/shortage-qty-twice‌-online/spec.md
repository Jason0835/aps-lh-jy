新增排产补充规则：仅历史欠产、后续无计划、前日已生产过的 SKU，本次跳过排产。

请在新增排产逻辑中增加判断：

适用场景：
- SKU 仅因“本月前日累计欠产 > 0”进入新增排产；
- 从当前排程日 T 到月底，月计划 dayN 均为 0，即后续无计划量；
- 根据当前排程日 T 前一日的日完成量判断是否最近一次已排过：
  - 若前日日完成量 > 0，表示该 SKU 前日已生产过；
  - 若前日日完成量 = 0 或为空，表示前日未生产过。

处理规则：
- 若满足“仅历史欠产 + T～月底无计划 + 前日日完成量 > 0”，则本次不再新增排产；
- 该 SKU 需写入未排结果；
- 未排原因建议为：仅历史欠产、后续无月计划，且前日已有完成量，本次跳过补排；
- 输出日志，包含 SKU、欠产量、排程日期、前日日完成量、跳过原因。

注意：
- 该规则覆盖仅历史欠产形成的新增收尾 SKU，但不影响续作补偿 SKU；
- 前日完成量按当前排程窗口 T 前一日判断；
- 如果 T～月底仍有任意月计划量，则不适用该规则，继续走原新增排产逻辑；
- 如果前日日完成量为 0 或为空，则允许继续按现有历史欠产补排逻辑判断；
- 不影响正常有计划 SKU、续作排产、换活字块排产。
