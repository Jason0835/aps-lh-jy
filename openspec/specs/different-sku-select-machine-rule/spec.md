## 不同类型 SKU 选机规则

### 一、前提：SKU 排序保持项目现有逻辑不变

不同类型 SKU 的选机规则，只影响“轮到某个 SKU 后如何选机台”，不改变 SKU 本身的排序顺序。

项目现有 SKU 排序优先级保持为：

1. 试制组
2. 量试组
3. 正规组

其中：

- 小批量 SKU 归入正规组；
- 小批量 SKU 与正规 SKU 按正规组统一排序规则排序；
- 不再额外增加“小批量优先于正规使用单控机台”的特殊约束。

也就是说，**谁先选机台，由 SKU 排序结果决定；轮到某个 SKU 后，再按该 SKU 类型执行对应选机规则。**

---

## 二、不同类型 SKU 选机规则

### 1. 试制 SKU

硬约束：

- 只能选择单控机台；
- 只能安排在中班开始生产；
- 首检计划量也归入中班；
- 不允许选择普通机台。

### 2. 量试 SKU

选机优先级：

1. 优先选择单控机台；
2. 单控机台不可用、产能不足或不满足约束时，可以选择普通机台。

### 3. 小批量 SKU

选机优先级：

1. 优先选择单控机台；
2. 单控机台不可用、产能不足或不满足约束时，可以选择普通机台。

注意：

- 小批量仍归入正规组排序；
- 小批量和正规 SKU 竞争单控机台时，不做“小批量额外优先”；
- 按正规组内统一 SKU 排序结果决定谁先选机台。

### 4. 正规 SKU

选机优先级：

1. 优先选择普通机台；
2. 普通机台中，条件少、约束少的机台优先；
3. 普通机台不可用、产能不足或不满足约束时，可以选择单控机台。

---

## 三、实现口径

### 1. 先排序 SKU

```text
SKU 排序：
试制组 > 量试组 > 正规组

正规组内：
小批量 SKU 与正规 SKU 统一排序
````

### 2. 再按 SKU 类型选择候选机台

```text
试制：
  候选机台 = 单控机台
  生产班次 = 中班
  首检班次 = 中班

量试：
  候选机台优先级 = 单控机台 > 普通机台

小批量：
  候选机台优先级 = 单控机台 > 普通机台

正规：
  候选机台优先级 = 普通机台 > 单控机台
  普通机台内部排序 = 条件少/约束少优先
```

### 4. 关键修正点

```text
去除旧约束：
当单控机台资源紧张时，小批量优先级高于正规 SKU 使用单控机台。

修正为：
小批量和正规 SKU 是否优先使用单控机台，不单独按“小批量优先”判断；
而是严格按照 SKU 排序结果决定。
```

核心一句话：

```md
不同类型 SKU 的选机规则不改变 SKU 排序优先级。排产时先按现有 SKU 排序规则确定 SKU 顺序，再在轮到具体 SKU 时，根据其类型执行对应的机台类型硬约束或优先级。小批量 SKU 归入正规组并与正规 SKU 统一排序，不再额外享有单控机台抢占优先级。
```
