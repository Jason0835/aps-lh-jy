# 硫化排程 SKU 加机台规则优化需求文档

## Purpose

统一新增排产和续作排产的欠产增机台、逐日后看及严格收尾目标保护规则，保证收尾排产不被历史欠产账本重复放大，避免无效扩机台和收尾重复排产。

## Requirements

### Requirement: 严格收尾目标优先于欠产账本

系统必须（MUST）在续作 SKU 已建立严格收尾目标后保持该目标，不得再用历史欠产或日计划账本剩余量抬高排产上限。

#### Scenario: 续作完成严格收尾目标

- **WHEN** 一台续作机台已经完整覆盖 SKU 的严格收尾目标
- **THEN** 续作补偿量必须为 0，且不得生成补偿 SKU 回流新增排产

### Requirement: 非收尾继续沿用原增机台规则

系统必须（MUST）保持非收尾 SKU 的欠产阈值、窗口有效产能和逐日后看规则不变。

#### Scenario: 非收尾 SKU 仍有产能缺口

- **WHEN** 非收尾 SKU 按现有欠产阈值和逐日后看规则判定当前机台不足
- **THEN** 系统必须继续复用现有候选筛选和产能模拟增加机台

### Requirement: dayN 只作为节奏与资源判断依据

系统必须（MUST）将月计划日计划量 dayN 作为排产节奏和资源判断依据，不得作为正规/量试非收尾 SKU 的实际排产硬上限。

dayN 只能用于：

* 判断 SKU 哪一天可以开始排产；
* 判断是否允许 SKU 提前生产；
* 判断当前机台数是否满足当前日/后看日计划节奏，从而决定是否需要新增机台；
* 判断提前生产发生后，被前移的计划节奏是否需要参与加机台判断。

dayN 不得用于：

* 直接限制当天实际排产量；
* 作为非收尾 SKU 的硬排产目标；
* 因 dayN 较小强制非收尾机台只排少量，导致班产浪费；
* 用 day1 + day2 + day3 简单作为所有非收尾 SKU 的硬目标量；
* 覆盖收尾目标量、硫化余量、胎胚库存、欠产追补、晚班不可换模补满等规则。

#### Scenario: 非收尾新增排产不得被 dayN 小量截断

- **WHEN** 正规/量试非收尾 SKU 的实际业务目标量大于 T～T+2 日计划剩余额度
- **AND** dayN 节奏判断仍认为需要新增机台或保留机台产能
- **THEN** 新增排产本轮剩余量必须按业务目标量继续排产
- **AND** dayN 只能继续参与加机台节奏模拟
- **AND** 系统不得把该 SKU 目标量裁成 day1 + day2 + day3 或剩余日计划小量

#### Scenario: 正式非收尾目标量不得被窗口剩余日计划覆盖

- **WHEN** 正式非收尾新增 SKU 的业务目标量大于 `windowRemainingPlanQty`
- **AND** `windowRemainingPlanQty` 或 day1 + day2 + day3 小于业务目标量
- **THEN** 多机台拆量的基准目标必须保留 SKU 业务目标量
- **AND** `windowRemainingPlanQty` 只能作为 dayN 节奏模拟和资源判断输入
- **AND** 系统不得因为窗口剩余日计划较小提前移出新增待排队列

#### Scenario: 新增排产 dayN 扩机台模拟不得被本轮目标量截断

- **WHEN** 正式非收尾新增 SKU 的月计划日计划为 162、162、162
- **AND** 当前本轮剩余目标量为 144
- **AND** 单台机台日理论产能为 54
- **THEN** dayN 扩机台模拟必须继续使用完整的 162、162、162 日计划节奏
- **AND** 系统必须推导该 SKU 当前窗口需要 3 台总机台，即 T 日需要新增 2 台机台
- **AND** 系统不得因本轮剩余目标量 144 将 dayN 模拟账本截断成首日小账本
- **AND** 公共 dayN 模拟账本构造方法也必须保留完整日计划节奏，不得再隐式按本轮剩余目标量截断
- **AND** 已有同物料机台只有在达到 dayN 标准机台数或模拟后无未满足量时，才能判定为已满足新增机台规则
- **AND** 当前候选机台换模损失只能影响实际排产量，不得把 dayN 标准机台数从 3 台继续放大
- **AND** 实际排产落表量仍必须由业务目标量、日计划扣账、胎胚库存、硫化余量、模具资源和机台可用窗口共同控制

### Requirement: 欠产未超过阈值时必须逐日判断是否加机台

系统必须（MUST）在本月前日累计欠产未超过阈值时，按业务日逐日判断是否需要增加机台：

1. 先判断当前日月计划是否已被当前机台数满足；
2. 当前日已满足时，当前日不加机台，且不再后看下一日；
3. 当前日未满足时，再判断下一日计划是否已被当前机台数满足；
4. 下一日也未满足时，当前日必须按候选机台排序逐台增加机台。

判断当前日和下一日是否满足时，必须使用当前机台数对应的 SKU 单日理论产能，不得使用已被提前排产、挤量或候选实际排满后的实际排产量代替单日标准产能。

#### Scenario: 首日小计划满足时不后看下一日

- **WHEN** 新增排产 SKU 首日月计划量为 8
- **AND** 当前一台机台的单日理论产能已满足首日计划
- **THEN** 首日不得因为后续日计划较高而提前新增机台
- **AND** 系统必须滚动到后续业务日后再重新判断是否需要新增机台

#### Scenario: 次日和后一天均超过单机日标准时当前日新增机台

- **WHEN** 新增排产 SKU 月计划日计划为 8、60、60
- **AND** SKU 单机日标准产能为 48
- **AND** 当前业务日滚动到第二日
- **THEN** 系统必须先判断第二日 60 未被当前一台机台满足
- **AND** 再判断第三日 60 也未被当前一台机台满足
- **AND** 第二日必须按候选机台排序增加一台机台

#### Scenario: 日计划 50 连续两日超过单机日标准时当前日新增机台

- **WHEN** 新增排产 SKU 月计划日计划为 8、50、50
- **AND** SKU 单机日标准产能为 48
- **AND** 当前业务日滚动到第二日
- **THEN** 第二日必须按候选机台排序增加一台机台

#### Scenario: 已有续作机台满足逐日节奏时不新增候选机台

- **WHEN** 新增排产 SKU 月计划日计划为 50、50、50
- **AND** SKU 单机日标准产能为 50
- **AND** 已有一台同物料续作机台能够满足当前日和后看日的 dayN 节奏
- **THEN** 系统必须判定当前新增候选机台排产量为 0
- **AND** 不得因业务目标量或月计划余量仍有剩余而新增第二台机台

## 1. 文档目的

本文档用于统一硫化排程中 SKU 加机台的最新规则，重点解决以下问题：

1. SKU 本月前日累计欠产超过阈值时，如何动态判断是否需要加机台；
2. 加机台数量不能固定写死为 1 台，必须根据窗口后剩余欠产动态计算；
3. 新增排产和续作排产需要同时适配欠产超过阈值的强制增机台判断；
4. 欠产未超过阈值时，不能为了小额欠产盲目加机台；
5. T～T+2 窗口无日计划时，需要区分后续是否还有日计划；
6. 收尾、非收尾、仅补历史欠产等不同场景需要明确排量边界；
7. 实现过程中必须优先复用项目已有逻辑，避免重新另起一套完整排程流程。

## 4. 核心业务目标

### 4.1 总目标

硫化排程不能简单按照“有欠产就加机台”，也不能固定只加 1 台机台。

系统需要综合判断：

1. 本月前日累计欠产是否超过阈值；
2. 当前已选机台在 T～T+2 窗口内的有效产能；
3. T～T+2 排程窗口内是否存在日计划；
4. T+3 到月底是否还有后续日计划；
5. SKU 是否收尾；
6. 加机台后是否造成低效短排；
7. 候选机台是否满足项目已有筛选和排序规则；
8. 模具、定点、特殊机台、单控机台、换模限制等约束是否满足。

### 4.2 欠产超过阈值时的目标

当：

```text
本月前日累计欠产 > 欠产阈值
```

说明该 SKU 进入强制增机台判断。

但注意：

```text
欠产超过阈值 ≠ 必须清空全部欠产
欠产超过阈值 ≠ 固定只加 1 台
欠产超过阈值 ≠ 无限加机台
```

正确目标是：

```text
加机台后，排程窗口结束时的剩余欠产能够回到欠产阈值以内。
```

只要：

```text
窗口后剩余欠产 <= 欠产阈值
```

就必须立即停止继续加机台。

### 4.3 欠产未超过阈值时的目标

当：

```text
本月前日累计欠产 <= 欠产阈值
```

不进入强制增机台判断。

此时不允许为了小额欠产盲目加机台，加机台目标是：

```text
先判断当前机台数是否能够满足当前日月计划量；
当前日已满足时，不再继续后看下一日计划；
当前日未满足时，再判断下一日计划是否被当前机台数满足；
下一日也未满足时，当前日按候选机台排序逐台加机台；
判断当前日和下一日是否满足时，按当前机台数 × SKU单日理论产能计算，不用实际挤量后的排产量代替。
```

## 6. 核心计算口径

### 6.1 本月前日累计欠产

本次规则只处理本月欠产。

要求：

- 不处理上个月欠产；
- 不使用上个月欠产参与本月加机台判断；
- 不使用本月前日累计超产抵扣本日计划；
- 本月前日累计欠产最小按 0 处理；
- 如果某日出现超产，只是不继续形成欠产，不作为负数继续向后滚动。

### 6.2 T 日需排产量

```text
T 日需排产量 = T 日月计划量 - T 日晚班已完成量 + 本月前日累计欠产
```

说明：

- T 日晚班已完成量需要扣减；
- 不叠加前日累计超产；
- 不允许用超产抵扣当前需排产量。

### 6.3 窗口计划量汇总

```text
T～T+2 日月计划量汇总 = T 日月计划量 + T+1 日月计划量 + T+2 日月计划量
```

### 6.4 窗口需消化量

```text
窗口需消化量 = 本月前日累计欠产 + T～T+2 日月计划量汇总 - T 日晚班已完成量
```

窗口需消化量用于计算窗口后剩余欠产，但不能理解为必须全部清空。

### 6.5 当前已选机台窗口有效产能总和

```text
当前已选机台窗口有效产能总和 = 当前已选机台在 T～T+2 排程窗口内逐班有效产能累加
```

注意：

- 不能简单使用 `班产 × 班次数`；
- 必须按 8 个班次逐班计算；
- 必须扣除换模、换活字块、清洗、停机、精度保养、已占用班次、晚班不可换模等影响。

### 6.6 窗口后剩余欠产

本需求统一采用“窗口总量统一计算法”。

```text
窗口后剩余欠产 =
本月前日累计欠产
+ T～T+2 日月计划量汇总
- T 日晚班已完成量
- 当前已选机台窗口有效产能总和
```

也可写为：

```text
窗口后剩余欠产 = 窗口需消化量 - 当前已选机台窗口有效产能总和
```

### 6.7 推荐采用方法 1

本次统一选择方法 1：

```text
窗口后剩余欠产 =
本月前日累计欠产
+ T～T+2 日月计划量汇总
- T 日晚班已完成量
- 当前已选机台窗口有效产能总和
```

选择原因：

1. 公式直观，便于新增排产和续作排产复用；
2. 每增加一台机台后，只需重新累计“当前已选机台窗口有效产能总和”；
3. 不需要区分“当前机台产能”和“新增机台产能”两套口径；
4. 降低重复扣减、重复计算、分支混乱的风险；
5. 与“每增加一台机台就重新判断是否回到阈值以内”的目标一致。

### 6.8 与“窗口需消化量覆盖”的冲突处理

如果旧逻辑中存在以下判断：

```text
当前已选机台窗口有效产能 >= 窗口需消化量
```

该判断会变成“必须清空全部欠产”，与最新规则冲突。

最新规则应改为：

```text
窗口后剩余欠产 <= 欠产阈值
```

等价转换为：

```text
当前已选机台窗口有效产能 >= 窗口需消化量 - 欠产阈值
```

最终代码实现建议直接使用：

```text
窗口后剩余欠产 <= 欠产阈值
```

避免理解偏差。

## 7. 欠产超过阈值时的加机台规则

### 7.1 触发条件

```text
本月前日累计欠产 > 欠产阈值
```

满足该条件时，进入强制增机台判断。

### 7.2 重要说明

“是否进入强制增机台判断”和“最多加到多少台”必须拆开实现。

- 欠产超过阈值，只是进入强制增机台判断的前置条件；
- 不代表一定要持续加机台；
- 不代表必须清空全部欠产；
- 不代表固定只加 1 台。

### 7.3 判断流程

```text
开始
│
├─ 1. 判断本月前日累计欠产是否 > 欠产阈值
│  ├─ 否：不进入强制增机台判断
│  └─ 是：继续
│
├─ 2. 计算当前已选机台窗口有效产能总和
│
├─ 3. 计算窗口后剩余欠产
│
├─ 4. 判断窗口后剩余欠产是否 <= 欠产阈值
│  ├─ 是：当前机台数已足够，不再加机台
│  └─ 否：继续按候选机台排序逐台尝试加机台
│
├─ 5. 每增加一台候选机台
│  ├─ 校验候选机台是否满足已有筛选和排序规则
│  ├─ 校验模具、定点、特殊机台、单控机台、换模限制等规则
│  ├─ 校验是否造成低效短排
│  ├─ 累计该机台窗口有效产能
│  ├─ 重新计算窗口后剩余欠产
│  ├─ 若窗口后剩余欠产 <= 欠产阈值，立即停止加机台
│  └─ 若仍 > 欠产阈值，继续尝试下一台候选机台
│
└─ 6. 若无更多可用候选机台
   └─ 停止加机台，并输出产能仍不足日志
```

### 7.4 加机台边界

新增机台必须同时满足：

1. 通过项目已有候选机台筛选；
2. 符合项目已有候选机台排序结果；
3. 满足机台匹配规则；
4. 满足模具校验规则；
5. 满足定点机台规则；
6. 满足特殊机台/特殊材料规则；
7. 满足单控机台规则；
8. 满足换模限制、晚班不可换模等规则；
9. 不造成低效短排；
10. SKU 已收尾时，不能导致超排；
11. 不能绕过已有业务规则直接加机台；
12. 不能因为欠产超过阈值就无限加机台；
13. 不能固定只加 1 台，必须按窗口后剩余欠产动态判断。

## 8. 欠产未超过阈值时的新增排产加机台规则

### 8.1 触发条件

```text
本月前日累计欠产 <= 欠产阈值
```

此时不进入强制增机台判断。

### 8.2 规则目标

欠产未超过阈值时，加机台目标不是清空全部小额欠产，而是保障后续日计划不被当前机台产能拖垮。

### 8.3 当前日前置判断

欠产未超过阈值时，每一天进入“后看下一日计划”前，必须先判断当前机台数是否已经满足当前日月计划量。

判断规则：

- 若当前机台数可满足当前日月计划量：
  - 当前日不加机台；
  - 不再继续判断下一日计划量；
  - 直接跳过当前日加机台判断。
- 若当前机台数无法满足当前日月计划量：
  - 才进入原有逐日后看逻辑；
  - 使用当前机台数判断下一日计划量是否满足；
  - 若下一日也无法满足，则当前日需要加机台。

当前日计划判断量口径：

- 当前日计划判断必须沿用日计划账本初始化口径，不能只看原始日月计划量；
- T 日已经存在晚班已完成量时，T 日判断计划量按以下公式计算：

```text
T 日判断计划量 = MAX(0, T 日月计划量 - T 日晚班已完成量)
```

- T+1、T+2 以及 T+3 后看日仍使用对应日期原始日月计划量，不扣减 T 日晚班已完成量；
- 该扣减只影响“当前日是否已满足、是否继续后看下一日”的增机台判断，不改变 T～T+2 窗口扣账边界，也不能提前消耗 T+3 到月底的后续计划。

示例：

- T 日先判断是否满足 T 日月计划量：
  - 满足：T 不加机台，不再判断 T+1；
  - 不满足：继续判断是否满足 T+1，决定 T 是否加机台。
- T+1、T+2 同理递推。

日志要求：

- 需要输出 SKU、当前日期、当前机台数、当前日产能、当前日计划量；
- 需要输出是否满足当前日、是否进入后看、后看日期、后看日计划量、是否加机台；
- 日志只用于排查判断过程，不允许用日志掩盖异常或改变原有排程结果。

### 8.4 逐日后看判断

#### T 日判断 T+1 日计划

- 前提：T 日当前机台数无法满足 T 日月计划量；
- 如果当前机台数可以满足 T+1 日月计划量，则 T 日不加机台；
- 如果当前机台数无法满足 T+1 日月计划量，则 T 日需要加机台。

#### T+1 日判断 T+2 日计划

- 前提：T+1 日当前机台数无法满足 T+1 日月计划量；
- 如果当前机台数可以满足 T+2 日月计划量，则 T+1 日不再加机台；
- 如果当前机台数无法满足 T+2 日月计划量，则 T+1 日需要继续加机台。

#### T+2 日

- 前提：T+2 日当前机台数无法满足 T+2 日月计划量；
- T+2 虽然是当前排程窗口最后一天，但在当前日未满足且存在 T+3 日月计划时，欠产未超过阈值的新增排产场景仍必须继续判断 T+3 日月计划量；
- 如果当前机台数可以满足 T+3 日月计划量，则 T+2 不再加机台；
- 如果当前机台数无法满足 T+3 日月计划量，则 T+2 需要继续加机台。
- 如果不存在 T+3 日月计划，且当前机台数无法满足 T+2 日月计划量，则 T+2 直接判定需要继续加机台，不能因为没有下一生产日而把当前日不足误判为已满足。

说明：

- T+2 后看 T+3 只用于判断是否需要保留或新增机台，必须以存在下一生产日为前提；
- 实际排产扣账仍按 T～T+2 窗口控制，不能提前消耗 T+3 到月底的后续计划；
- 欠产超过阈值的强制增机台仍按 T～T+2 窗口后剩余欠产判断，不因本规则改变。

#### Scenario: 窗口末日无下一生产日但当前日计划不足

- **WHEN** 本月前日累计欠产未超过阈值，且当前机台数无法满足窗口末日的日月计划量
- **AND** 日计划账本中不存在下一生产日可继续后看
- **THEN** 系统必须将窗口末日判定为首次需要加机台的业务日期
- **AND** 不得因为下一生产日为空而返回“当前机台数已满足”
- **AND** 日志必须输出 SKU、窗口末日、当前机台数、当前日产能、当前日计划量、是否进入后看、是否加机台

### 8.5 加机台数量计算

```text
后一天缺口量 = 后一天日月计划量 - 当前机台在后一天的有效产能
```

如果后一天缺口量大于 0，则：

1. 复用项目已有候选机台筛选逻辑；
2. 复用项目已有候选机台排序逻辑；
3. 复用项目已有机台有效产能计算逻辑；
4. 按排序结果逐台增加机台；
5. 每增加一台机台，重新计算后一天缺口；
6. 直到满足后一天日计划缺口，或无可用机台为止。

## 9. 窗口无日计划特殊规则

当某 SKU 在 T～T+2 窗口内日月计划量均为 0 时，需要结合 T+3 到月底是否还有后续日计划进行区分。

> 前提：本规则主要用于本月存在历史欠产的 SKU，即 `本月前日累计欠产 > 0`。

### 9.1 T～T+2 无日计划，且 T+3 到月底也无计划

说明本月后续已经没有计划，该 SKU 可按收尾处理。

处理规则：

- 排产目标量复用项目已有收尾目标量口径；
- 严格按目标量排产；
- 不允许超排；
- 允许最后一个班次不足班产。

### 9.2 T～T+2 无日计划，但 T+3 到月底仍有计划

说明 SKU 不是整体收尾，只是当前窗口无日计划。

处理规则：

- 对未命中续作、准备进入新增排产的 SKU，若本月前日累计欠产小于等于 0，则直接列入未排，不得进入换活字块或新增选机；
- 上月超欠产只参与硫化余量计算，不能作为提前消耗本月 T+3 到月底远期计划的依据；
- 上述新增 SKU 的未排原因统一为：`当前排程窗口无日计划，后续远期有计划，禁止提前消耗未来计划`；
- 本窗口只允许补本月前日累计欠产；
- 排产目标量 = 本月前日累计欠产；
- 严格按欠产量排；
- 不允许超排；
- 不允许提前消耗 T+3 到月底的后续计划；
- 不允许因为非收尾就强行满班产；
- 当前机台可以下机，但不能把 SKU 判断为整体收尾；
- 后续新增排产仍需继续识别该 SKU。

## 10. 新增排产排量边界规则

### 10.1 普通非收尾，且窗口存在日计划

普通 SKU 非收尾，且 T～T+2 窗口内存在日月计划时，只要某个班次被安排生产，原则上按满班产排。

示例：

```text
班产 = 16
剩余需求 = 8
```

普通非收尾场景下，该班次仍排 16，不排 8。

目的：

- 满足日计划；
- 消化本月欠产；
- 提高机台利用率；
- 避免新增机台短排后立即下机。

### 10.2 SKU 收尾排产

SKU 收尾时，必须严格按照目标量排产，不允许超排。

收尾目标量沿用项目已有口径，例如：

```text
收尾目标量 = MAX(硫化余量, 胎胚库存)
```

要求：

- 复用项目已有收尾目标量计算逻辑；
- 不要另写一套收尾目标量计算；
- 收尾时允许最后一个班次不足班产；
- 收尾场景不能为了满班产而超排。
- 严格收尾目标一旦建立，历史欠产和日计划账本只保留追溯口径，不得再次抬高排产目标。
- 续作已排量达到严格收尾目标时，不得生成补偿 SKU 回流新增排产。

### 10.3 仅补本月历史欠产

当窗口无日计划但 T+3 到月底仍有计划时，本窗口只允许补本月历史欠产。

要求：

- 严格按欠产量排；
- 不允许超排；
- 不允许提前消耗 T+3 到月底的后续计划；
- 不允许为了满班产强行多排。

## 11. 机台窗口有效产能计算规则

机台窗口有效产能必须按 T～T+2 排程窗口内的有效班次逐班累加，不能简单使用：

```text
班产 × 班次数
```

必须复用项目已有有效产能计算和产能扣减逻辑，至少需要扣除：

1. 换模时间；
2. 换活字块时间；
3. 清洗时间；
4. 停机时间；
5. 精度保养时间；
6. 已被其他 SKU 占用的班次；
7. 晚班不可换模导致的不可用时间；
8. 项目已有其他产能扣减规则。

有效产能计算必须与最终落班次排产逻辑保持同一口径，不能出现：

```text
判断时认为产能足够，但实际落班次时产能不足。
```

## 12. 推荐整体判断流程

```text
开始处理 SKU
│
├─ 1. 读取基础数据
│  ├─ SKU 类型
│  ├─ 班产
│  ├─ 硫化余量
│  ├─ 胎胚库存
│  ├─ T 日月计划量
│  ├─ T 日晚班已完成量
│  ├─ T～T+2 日月计划量
│  ├─ T+3～月底后续日月计划量
│  ├─ 本月前日累计欠产
│  ├─ 欠产阈值
│  └─ 当前已选机台窗口有效产能
│
├─ 2. 计算 T 日需排产量
│  └─ T 日需排产量 = T 日月计划量 - T 日晚班已完成量 + 本月前日累计欠产
│
├─ 3. 判断 SKU 是否收尾
│  ├─ 是：复用已有收尾目标量，严格排产，不超排
│  └─ 否：继续
│
├─ 4. 判断窗口是否无日计划
│  ├─ T～T+2 无计划，且 T+3～月底无计划
│  │  └─ 按收尾处理，严格排产，不超排
│  │
│  ├─ T～T+2 无计划，但 T+3～月底有计划
│  │  └─ 只补本月前日累计欠产，严格排产，不超排
│  │
│  └─ T～T+2 存在日计划
│     └─ 继续
│
├─ 5. 判断本月前日累计欠产是否超过阈值
│  ├─ > 阈值
│  │  ├─ 进入强制增机台判断
│  │  ├─ 计算窗口后剩余欠产
│  │  ├─ 若窗口后剩余欠产 <= 阈值，不加机台
│  │  └─ 若窗口后剩余欠产 > 阈值，按候选机台排序逐台加机台并重新计算
│  │
│  └─ <= 阈值
│     └─ 进入逐日当前日前置判断
│
├─ 6. 逐日判断是否需要加机台
│  ├─ 先判断当前日月计划是否已被当前机台数满足
│  ├─ 当前日已满足：当前日不加机台，不再后看下一日
│  ├─ 当前日未满足：再判断下一日计划是否被当前机台数满足
│  └─ 下一日也未满足：当前日按候选机台排序逐台加机台
│
└─ 7. 落班次排产
   ├─ 普通非收尾，且窗口存在日计划：班次原则上按满班产排
   ├─ SKU 收尾：严格目标量，不超排
   └─ 窗口无日计划但仅补本月欠产：严格欠产量，不超排
```

## 13. Excel 场景校验

已知：

| 项目 | 数值 |
|---|---:|
| 欠产阈值 | 150 |
| 本月前日累计欠产 | 192 |
| T～T+2 日月计划量汇总 | 144 |
| T 日晚班已完成量 | 0 |
| 单台机台窗口有效产能 | 112 |

### 13.1 初始 1 台机台

```text
窗口后剩余欠产 = 192 + 144 - 0 - 112 = 224
```

判断：

```text
224 > 150
```

说明当前 1 台机台不足，需要继续尝试增加机台。

### 13.2 增加 1 台后，共 2 台机台

两台机台窗口有效产能合计：

```text
112 + 112 = 224
```

重新计算：

```text
窗口后剩余欠产 = 192 + 144 - 0 - 224 = 112
```

判断：

```text
112 <= 150
```

说明窗口后剩余欠产已回到阈值以内，必须停止继续加机台。

### 13.3 最终结果

```text
需要新增 1 台机台；
最终共排 2 台机台；
不需要继续加第 3 台。
```

### 14.1 代码复用要求

必须优先复用项目已有逻辑，包括但不限于：

1. 新增排产主流程；
2. 续作排产主流程；
3. SKU 排序逻辑；
4. 机台候选筛选逻辑；
5. 机台候选排序逻辑；
6. 班产/有效产能计算逻辑；
7. 收尾判断逻辑；
8. 换模/换活字块耗时逻辑；
9. 晚班不可换模逻辑；
10. 机台占用判断逻辑；
11. 模具可用性校验逻辑；
12. 定点机台规则；
13. 特殊机台规则；
14. 单控机台规则；
15. 原有日志、异常、参数读取方式。

如果项目中已有公共方法或策略类可以满足要求，应优先复用或扩展。

只有现有逻辑无法覆盖新规则，或强行复用会导致代码强耦合、重复计算、逻辑混乱时，才允许小范围重构。

### 14.2 方法职责建议

建议封装新增/续作共用的欠产阈值增机台判断能力，避免在主流程中堆叠大量 if/else。

建议拆分以下职责：

1. 读取欠产阈值参数；
2. 计算本月前日累计欠产；
3. 计算 T 日需排产量；
4. 判断窗口是否无日计划；
5. 判断 T+3 到月底是否还有计划；
6. 计算机台窗口有效产能；
7. 计算窗口后剩余欠产；
8. 判断是否进入强制增机台；
9. 按候选机台排序逐台补充机台；
10. 判断排量是否允许满班产或必须严格目标量；
11. 输出加机台原因和最终结果日志。

### 14.3 技术约束

- JDK 1.8 兼容，不使用高版本语法；
- 避免魔法值，阈值、窗口天数、默认值等使用常量或参数；
- 不绕过已有机台筛选排序直接加机台；
- 不重复实现已有产能计算、收尾判断、排产落班逻辑；
- 循环加机台时避免重复全量扫描大集合；
- 候选机台、班次产能、机台占用信息建议提前构建 Map 或缓存；
- 注意内存使用，避免 OOM；
- 关键逻辑补充详细中文注释。

### 14.4 注释要求

代码注释需重点说明：

1. 欠产阈值判断逻辑；

## 16. 验收场景

### 场景 1：本月前日累计欠产超过阈值，当前机台不足

前置条件：

- 欠产阈值 = 150；
- 本月前日累计欠产 = 192；
- T～T+2 日月计划量汇总 = 144；
- 单台机台窗口有效产能 = 112。

期望结果：

- 1 台机台时窗口后剩余欠产 = 224，大于 150；
- 需要继续加机台；
- 加 1 台后，共 2 台机台，窗口后剩余欠产 = 112，小于等于 150；
- 停止加机台；
- 最终共排 2 台，不继续加第 3 台。

### 场景 2：本月前日累计欠产超过阈值，但当前机台已足够

期望结果：

- 进入强制增机台判断；
- 计算窗口后剩余欠产；
- 若窗口后剩余欠产 <= 欠产阈值，则不新增机台。

### 场景 3：本月前日累计欠产未超过阈值，当前机台可满足后续日计划

期望结果：

- 不进入强制增机台判断；
- 不为了小额欠产新增机台；
- 小额欠产允许滚动。

### 场景 4：本月前日累计欠产未超过阈值，但当前机台无法满足 T+1 日计划

期望结果：

- T 日需要加机台；
- 加机台数量按 T+1 日计划缺口动态计算；
- 普通非收尾班次按满班产排。

### 场景 5：T 日加机台后，已能满足 T+2 日计划

期望结果：

- T+1 不应重复加机台；
- 避免低效短排和无效扩机台。

### 场景 5.1：本月前日累计欠产未超过阈值，T+2 需要判断 T+3 日计划

前置条件：

- 排程日期：2026-04-03；
- SKU：3302001236；
- SKU 状态：非收尾；
- 硫化余量：1432；
- 班产：16；
- T～T+2 日计划：day1=0，day2=8，day3=80；
- T+3/day4 日计划：96。

期望结果：

- 不进入欠产超过阈值的强制增机台判断；
- T+2 必须继续判断 T+3 日计划；
- day4=96 时，单台机台 3 班产能为 48，第三天需要 2 台机台才能保障后一天日计划；
- SKU 非收尾且硫化余量充足时，保留/新增的多台机台在 T+2 可用班次也应按班产排满；
- 期望 K2025 班次3～班次8 计划量均为16；
- 期望 K1002 班次3～班次8 计划量均为16；
- 不能只排到满足 T+2 窗口目标量后提前停机。

### 场景 6：T～T+2 日计划为 0，且 T+3 到月底也无计划

期望结果：

- 按 SKU 收尾处理；
- 复用已有收尾目标量；
- 严格目标量排产；
- 不超排；
- 允许尾班不足班产。

### 场景 7：T～T+2 日计划为 0，但 T+3 到月底仍有计划

期望结果：

- SKU 不是整体收尾；
- 本窗口只补本月前日累计欠产；
- 不提前消耗后续计划；
- 不因为非收尾强行满班产；
- 当前机台可以下机，但不能影响后续新增排产继续识别该 SKU。

### 场景 8：SKU 收尾场景

期望结果：

- 严格按照项目已有收尾目标量排产；
- 不允许为了满班产而超排；
- 最后一个班次允许不足班产。

### 场景 9：普通非收尾且窗口存在日计划

期望结果：

- 只要班次被安排生产，原则上按满班产排；
- 不为了普通非收尾尾量制造半班、短班；
- 但不得突破已有低效短排、机台占用、晚班不可换模等约束。

### 场景 10：当天不满足但后一天满足时不增加机台

前置条件：

- 日计划为 `T=8、T+1=92、T+2=46、T+3=46`；
- 单台机台日产能为46；
- 单台机台可以满足 T 日计划，但不能满足 T+1 日计划。

期望结果：

- T 日当前机台可满足8，T日不加机台；
- T+1当前机台不能满足92，但可以满足T+2的46，T+1不加机台；
- T+2当前机台可满足46，T+2不加机台；
- 最终只保留1台机台，不得因T+1当天计划92单独触发增机。

### 场景 11：新增机台从逐日判断命中的业务日生效

前置条件：

- 排程窗口日计划为 `T=8、T+1=92、T+2=92`；
- 单台机台可以满足 T 日计划，但不能满足 T+1 和 T+2 日计划；
- dayN 逐日判断在 T+1 将启用机台数从1台增加为2台。

期望结果：

- T日只允许首台机台进行正常生产，不得提前启用第二台机台的正常产能；
- 第二台机台不得在T日提前换模；应在T+1业务日内首个允许换模班次开始换模；
- 若 T+1 首个业务班次为晚班且不可换模，第二台机台应顺延到 T+1 早班换模；换模完成时间正好等于早班结束时间时，首检数量归入该早班；
- 第二台机台的正常生产计划量不得早于 T+1 换模和首检完成后的首个可生产班次；
- 第二台机台从 T+1 开始参与日产能和班次分配；
- 如果后续日期继续增加第 3 台及更多机台，各新增机台必须按对应逐日决策日期分别生效，不能统一提前到首次增机日期。

### 场景 12：续作严格收尾已完成但历史欠产超过阈值

前置条件：

- SKU 已判定收尾；
- 硫化余量为 32，班产为 18；
- 一台续作机台两个班次可排 18+14，完整覆盖严格收尾目标；
- 本月历史欠产超过增机台阈值，日计划账本仍有剩余额度。

期望结果：

- 严格收尾目标保持 32，不被历史欠产或账本剩余量抬高；
- 续作机台排满 32 后补偿量为 0；
- 不生成补偿 SKU，不进入 S4.5 新增排产，不新增机台。

## 17. 给 Codex 的实施提示词

请基于项目现有硫化排程代码，优化 SKU 欠产超过阈值时的加机台逻辑，适用范围包括续作排产和新增排产。

核心规则如下：

1. 欠产超过阈值只是进入强制增机台判断的前置条件，不代表要清空全部欠产，也不能固定只加 1 台。
2. 统一使用以下公式计算窗口后剩余欠产：

```text
窗口后剩余欠产 =
本月前日累计欠产
+ T～T+2 日月计划量汇总
- T 日晚班已完成量
- 当前已选机台窗口有效产能总和
```

3. 每增加一台机台后，都必须重新累计当前已选机台窗口有效产能，并重新计算窗口后剩余欠产。
4. 只要 `窗口后剩余欠产 <= 欠产阈值`，立即停止继续加机台。
5. 不要使用“当前已选机台窗口有效产能 >= 窗口需消化量”作为最终判断，因为这会变成强制清空全部欠产；正确判断是“剩余欠产回到阈值以内”。
6. 机台窗口有效产能必须按 T～T+2 的 8 个班次逐班计算，复用已有换模、换活字块、清洗、停机、精度保养、已占用班次、晚班不可换模等产能扣减逻辑，不能简单用班产乘班次数。
7. 加机台时必须复用项目已有候选机台筛选、排序、机台匹配、模具校验、定点机台、特殊机台、单控机台、低效短排、SKU 收尾与超排控制逻辑，不能绕过已有规则直接加机台。
8. 欠产未超过阈值时，不进入强制增机台判断；新增排产按逐日后看逻辑判断当前机台是否能支撑 T+1/T+2/T+3 后续日计划，其中 T+2 必须继续判断 T+3 日计划。
9. T～T+2 窗口无日计划时：
   - 若 T+3 到月底也无计划，按 SKU 收尾处理，严格目标量，不超排；
   - 若 T+3 到月底仍有计划，本窗口只补本月前日累计欠产，严格欠产量，不提前消耗后续计划，不强行满班产。
10. 普通非收尾且窗口存在日计划时，班次原则上按满班产排；SKU 收尾或仅补历史欠产时，必须严格目标量，不允许超排。
11. 优先复用已有方法、策略和缓存；只有现有逻辑无法覆盖时，才允许小范围重构。
12. 注意性能，避免循环加机台时重复全量扫描大集合；候选机台、班次产能、机台占用信息建议提前构建 Map 或缓存，避免 OOM。
13. 关键代码必须补充详细中文注释，说明欠产阈值判断、窗口后剩余欠产计算、机台窗口有效产能计算、停止加机台原因，以及复用/扩展了哪些已有逻辑。

请输出：
1. 代码修改方案；
2. 涉及类和方法；
3. 关键逻辑伪代码或代码片段；
4. 回归测试点；
5. 风险点和兼容性说明。

## 18. 最终规则摘要

新增排产和续作排产的加机台核心不是“有欠产就加机台”。

- 当本月前日累计欠产超过阈值时，进入强制增机台判断，但目标不是清空全部欠产，而是让窗口后剩余欠产回到阈值以内；每加一台机台都要重新计算，达标即停止。
- 当本月前日累计欠产未超过阈值时，不为了小额欠产盲目加机台，只判断当前机台是否能支撑后续日计划；新增排产必须按 T 判断 T+1、T+1 判断 T+2、T+2 判断 T+3 的口径逐日后看。
- 当窗口无日计划时，需要区分 T+3 到月底是否还有计划：无后续计划则按收尾清量，有后续计划则只补本月欠产。
- 普通非收尾且窗口存在日计划时，班次原则上按满班产排；SKU 收尾或仅补本月欠产时，必须严格目标量，不允许超排。
- 实现时必须优先复用项目已有 SKU 排序、机台筛选排序、有效产能计算、收尾目标量、低效短排判断和排产落班逻辑。
