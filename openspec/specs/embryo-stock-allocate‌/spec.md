# 胎胚库存账本与共用胎胚分摊主规格

## Purpose

规范硫化排程中胎胚库存参与目标量计算、共用胎胚 SKU 额度分配、T 日收尾硬控和结果库存落库口径。

核心原则：

1. 落库不分摊：排程结果 `EMBRYO_STOCK` 始终保存原始胎胚库存。
2. 内部分摊：共用胎胚只在排产内部按 SKU 额度控制消费。
3. 组级硬控：命中胎胚库存收尾硬目标时，总计划量不得超过同一个胎胚库存消费账本。
4. 先分 SKU，再分机台：同 SKU 多机台共享 SKU 额度，不再按机台分摊胎胚库存。
5. 单胎胚放宽：单胎胚（非共用胎胚）命中胎胚收尾标记且 SKU 为收尾时即可按胎胚库存排产，不再强制要求 T 日收尾。

## Requirements

### Requirement: 胎胚库存收尾准入

系统必须（MUST）根据胎胚共用关系区分准入条件：

**单胎胚（非共用胎胚）准入条件：**

- 胎胚收尾标识为是；
- SKU 已标记收尾（`skuTag == ENDING`）。

单胎胚不再强制要求 `endingDaysRemaining == 1`（T 日收尾），只要满足上述两个条件即可按胎胚库存排产。

**共用胎胚准入条件：**

- 胎胚收尾标识为是；
- SKU 已标记收尾；
- SKU 的 `endingDaysRemaining == 1`，即 T 日收尾。

共用胎胚保持原有 T 日同日收尾判断，确保分摊口径不变。

#### Scenario: 单胎胚胎胚收尾

- **WHEN** SKU 不与其他有效 SKU 共用胎胚，且胎胚收尾标识为是，且 SKU 为收尾 SKU（不要求 T 日收尾）
- **THEN** SKU 排产目标量必须等于原始胎胚库存
- **AND** 必须创建该胎胚自己的消费账本
- **AND** 不得使用 `MAX(胎胚库存, 硫化余量)`

#### Scenario: 单胎胚未命中胎胚库存硬目标

- **WHEN** SKU 胎胚收尾标识不为是，或 SKU 未标记收尾
- **THEN** SKU 收尾目标量仍为 `MAX(胎胚库存, 硫化余量)`
- **AND** SKU 非收尾目标量仍为硫化余量

#### Scenario: 共用胎胚分摊前基于实际余量刷新收尾天数

- **WHEN** 系统进入共用胎胚库存分摊逻辑
- **AND** SKU 的 `endingDaysRemaining` 由 `markEndingSkus` 阶段以窗口计划量计算
- **AND** 实际硫化余量远小于窗口计划量
- **THEN** 系统 MUST 基于实际硫化余量（`surplusQty`）和班产（`shiftCapacity`）重新计算 `endingDaysRemaining`
- **AND** 计算公式为 `ceil(ceil(surplusQty / shiftCapacity) / DEFAULT_SHIFTS_PER_DAY)`
- **AND** 刷新后的 `endingDaysRemaining` 用于共用胎胚 T 日收尾硬目标准入判断，确保余量较少的 SKU 不因窗口计划量偏差被错误排除

### Requirement: 胎胚库存消费账本

系统必须（MUST）使用运行态对象 `EmbryoStockConsumeLedger` 控制胎胚库存消费。

账本 key 必须为：

```text
embryoCode + "_" + scheduleDate
```

其中 `scheduleDate` 必须是 T 日业务日期，格式为 `yyyy-MM-dd`，不得保存具体时分秒，不得使用收尾批次作为 key。

账本字段：

- `embryoCode`：胎胚代码；
- `scheduleDate`：T 日业务日期；
- `originalStockQty`：原始胎胚库存；
- `targetQty`：本次组级目标量，等于原始胎胚库存；
- `consumedQty`：已消费计划量；
- `remainQty`：剩余可消费量。

#### Scenario: 账本扣减

- **WHEN** 排产结果确认一笔计划量
- **THEN** 必须从 SKU 实际消费账本扣减计划量
- **AND** 若该 SKU 命中胎胚库存硬目标，必须同步从胎胚库存组级账本扣减计划量

#### Scenario: 账本恢复

- **WHEN** 已生成结果被回滚或占位结果被抢占
- **THEN** 必须恢复 SKU 实际消费账本
- **AND** 若该 SKU 命中胎胚库存硬目标，必须同步恢复胎胚库存组级账本，但不得超过组级目标量

### Requirement: 共用胎胚T日同日收尾分摊

系统必须（MUST）只在不同 SKU 共用同一胎胚，且这些 SKU 均命中胎胚库存 T 日硬目标时，创建共用胎胚组级目标。

#### Scenario: 组级目标量

- **WHEN** 共用胎胚组命中 T 日同日胎胚收尾
- **THEN** 组级目标量必须等于原始胎胚库存
- **AND** 不得使用 `MAX(胎胚库存, 硫化余量)`
- **AND** 不得沿用普通共用胎胚“只按硫化余量排产”的口径

#### Scenario: SKU级额度

- **WHEN** 需要分配共用胎胚内部额度
- **THEN** 必须按 SKU 标准产能占比分配
- **AND** 标准产能缺失或小于等于 0 时，沿用项目已有的 SKU 日标准产能兼容口径
- **AND** 最后一个 SKU 承接整数尾差，确保 SKU 额度合计等于组级目标量

#### Scenario: 换模可行性修正

- **WHEN** SKU 级初始额度分配完成
- **THEN** 系统必须进行只读可行性预评估
- **AND** 评估必须优先复用现有候选机台、机台约束、T 日早中班产能、换模/换活字块限制和模具资源口径
- **AND** 无法在 T 日合理消化的额度可以转给同胎胚组内更可行的 SKU
- **AND** 不得为了预评估污染真实机台、模具、换模计数或排程结果

### Requirement: 同SKU多机台

系统必须（MUST）先确定 SKU 级额度，再复用现有同 SKU 多机台计划量分配逻辑。

#### Scenario: 多机台共享SKU额度

- **WHEN** 同一 SKU 分配到多个机台生产
- **THEN** 多台机台必须共享同一个 SKU 内部额度和同一个胎胚组级账本
- **AND** 不得因为某个 SKU 有多个机台就优先消费完整胎胚库存
- **AND** 每条机台结果中的胎胚库存字段仍保存原始胎胚库存

### Requirement: 结果落库口径

系统必须（MUST）保证胎胚库存字段落库不做分摊。

#### Scenario: 共用胎胚结果落库

- **WHEN** 胎胚库存为 100，SKU A 内部额度 60，SKU B 内部额度 40
- **THEN** SKU A 的排程结果胎胚库存为 100
- **AND** SKU B 的排程结果胎胚库存为 100

#### Scenario: 同SKU多机台结果落库

- **WHEN** SKU A 分到机台1计划 36、机台2计划 24
- **THEN** 机台1排程结果胎胚库存为原始胎胚库存
- **AND** 机台2排程结果胎胚库存为原始胎胚库存

### Requirement: 动态有效SKU集合

系统必须（MUST）以运行态有效 SKU 集合识别胎胚共用关系。

#### Scenario: 收尾完成后剔除

- **WHEN** 共用胎胚 SKU 已收尾完成、目标量为 0 或写入未排
- **THEN** 后续必须从该胎胚有效 SKU 集合中剔除
- **AND** 重新判断剩余 SKU 是否仍满足共用胎胚 T 日同日收尾
- **AND** 若该 SKU 命中胎胚库存 T 日硬目标，必须清理该 SKU 的内部库存额度和硬目标标记
- **AND** 必须立即刷新同胎胚剩余有效 SKU 的库存分摊，使未消费额度回流给仍可排产的 SKU

#### Scenario: 共用胎胚零余量预剔除保留生产者

- **WHEN** 共用胎胚组内多个 SKU 硫化余量均为 0，且该胎胚命中胎胚库存收尾标识为是、胎胚库存大于 0
- **AND** 组内存在前一业务日（T-1）仍有日计划的 SKU（昨日仍在产，是消纳胎胚库存的自然生产者）
- **THEN** 零余量预剔除必须仅剔除非生产者 SKU，保留该生产者 SKU 不写入"共用胎胚且硫化余量为0"未排
- **AND** 非生产者剔除后由动态转单胎胚归一化将生产者转为单胎胚，按胎胚库存收尾排产
- **AND** 禁止一刀切剔除组内全部零余量 SKU，避免胎胚库存无法消纳

#### Scenario: 动态转单胎胚

- **WHEN** 剔除后只剩一个 SKU 使用该胎胚
- **THEN** 后续必须按单胎胚 T 日胎胚收尾账本处理
- **AND** 该 SKU 的胎胚库存展示和结果落库仍保存原始胎胚库存

#### Scenario: 新增排产未排后的二次分摊

- **WHEN** 共用胎胚 T 日同日收尾 SKU 在新增排产中因首检失败、候选为空、换模不可行或其他业务原因写入未排
- **THEN** 系统 MUST 通过统一未排入口从运行态有效 SKU 集合剔除该 SKU
- **AND** 系统 MUST 将其未消费的 SKU 内部额度重新分摊给同胎胚剩余有效 SKU
- **AND** 若剩余有效 SKU 仍无法消化组级胎胚库存，系统 MUST 保留明确的未排或过程日志原因，不得静默丢失库存额度

## Verification

实现或调整本能力时，至少验证以下场景：

1. 单胎胚胎胚收尾（不要求 T 日）按库存建账本排产。
2. 单胎胚非胎胚收尾仍按原有 SKU 收尾/非收尾规则。
3. 共用胎胚 T 日同日收尾按标准产能分 SKU 内部额度。
4. 共用胎胚结果 `EMBRYO_STOCK` 保存原始库存，不保存内部额度。
5. 同 SKU 多机台共享 SKU 额度和组级账本。
6. 结果计划量不得突破 SKU 剩余额度和胎胚组级账本剩余量。
7. 账本扣减和恢复后 `consumedQty/remainQty` 保持一致。
8. 换模可行性修正能将不可合理消化额度转给更可行 SKU，并记录未消化原因。
