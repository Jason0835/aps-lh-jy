请调整共用胎胚分配胎胚库存规则，以本规则为准。

核心规则：
1. 同一个胎胚被不同 SKU 共用时，不是所有场景都需要分摊胎胚库存。
2. 只有同时满足以下条件时，才需要对共用胎胚库存进行分摊：
   - 不同 SKU 共用同一个胎胚；
   - 这些共用胎胚 SKU 都已标记为收尾；
   - 这些共用胎胚 SKU 在同一天收尾。
3. 同一个 SKU 在不同机台上生产时，不进入共用胎胚库存分摊：
   - 同 SKU 多机台只属于结果拆量或展示口径；
   - 只有不同 SKU 满足共用胎胚且同一天收尾时，才进入共用胎胚库存分摊逻辑。

分摊口径：
1. 目标 SKU 实际应分配的胎胚库存，按照 SKU 标准产能占比分摊。
2. 标准产能占比分摊逻辑复用项目现有产能数据：
   - 优先使用 SKU 标准产能；
   - 标准产能缺失或小于等于 0 时，按项目现有兼容口径使用 SKU 日产能；
   - 不重新引入“日硫化量 * 台数 * 模数”的新权重算法。
3. 本规则只调整分摊触发条件、动态剔除和尾差处理，不重写共用胎胚识别主链。

排程窗口内动态处理：
1. S4.3 构建 SKU 时先读取原始胎胚库存，不在 DTO 构建阶段提前分摊。
2. S4.3 收尾标注完成后，按当前有效胎胚 SKU 集合刷新一次胎胚库存分摊。
3. 如果本次排程窗口中某一天有共用胎胚 SKU 收尾完成：
   - 从后续计算开始，必须从该胎胚有效 SKU 集合中剔除已收尾完成的 SKU；
   - 只在剩余仍未收尾、仍参与该胎胚共用的不同 SKU 之间重新判断是否分摊；
   - 如果剩余 SKU 仍为不同 SKU 且同一天收尾，重新按标准产能占比分摊；
   - 如果剩余 SKU 不满足同一天收尾，后续不分摊。
4. 如果剔除后只剩一个 SKU 使用该胎胚：
   - 后续该 SKU 识别为单胎胚/非共用胎胚场景；
   - 不再进行共用胎胚库存分摊；
   - 胎胚库存恢复为该胎胚原始库存口径。

尾差处理：
1. 胎胚库存分摊结果需要处理整数尾差。
2. 前 N-1 个 SKU 可按项目现有取整口径处理。
3. 最后一个 SKU 必须分配剩余全部胎胚库存：

   lastAllocatedStock = sharedGreenTireStock - sum(前面 SKU 已分配胎胚库存)

4. 必须确保所有 SKU 分配结果汇总严格等于共用胎胚总库存。

落地入口：
1. `ScheduleAdjustHandler`：
   - `gatherSkuByStructure` 只构建 SKU 和读取原始胎胚库存；
   - `markEndingSkus` 完成后调用统一刷新逻辑；
   - 不在月计划 DTO 构建阶段提前分摊共用胎胚库存。
2. `TargetScheduleQtyResolver`：
   - 负责维护 `activeEmbryoSkuMap`；
   - 负责根据“不同 SKU + 同一天收尾”判断是否分摊；
   - 负责运行态 SKU 收尾剔除后的重新分摊或动态转单胎胚。

日志要求：
1. 分摊或不分摊时需要输出：
   - 胎胚编码/胎胚描述；
   - 当前参与判断的 SKU 列表；
   - 是否满足“不同 SKU + 同一天收尾”；
   - 每个 SKU 的标准产能权重、分配胎胚库存；
   - 哪些 SKU 因已收尾在后续被剔除；
   - 尾差处理后的分摊结果和汇总值。
2. 分摊权重异常、胎胚库存为 0、动态转单胎胚时必须输出明确日志。

测试覆盖：
1. 同 SKU 多机台，不分摊。
2. 不同 SKU 共用胎胚，但不是同一天收尾，不分摊。
3. 不同 SKU 共用胎胚，且同一天收尾，需要分摊。
4. 排程窗口内某天一个 SKU 收尾后，后续剔除该 SKU 再计算。
5. 剔除后只剩一个 SKU，后续识别为单胎胚。
6. 多 SKU 分摊存在整数尾差，最后一个 SKU 分配剩余库存。
7. 共用胎胚库存为 0。
8. 分摊 SKU 标准产能为 0 或异常时，按项目现有兼容口径处理并输出日志。
