# 盘点纳入设备停机计划规则

**状态：已实现** | **实现日期：2026-07-08** | **更新日期：2026-07-08** | **适用链路：硫化排程初始化、设备停机计划加载、班次产能扣减、排程结果停机摘要**

## Purpose

统一硫化排程中"盘点"停机类型的处理口径。盘点作为设备停机计划的一种停机类型（`machine_stop_type = 09`），在排程时只扣除对应时间段折算的可生产计划量，不触发换模、换活字块、首检、预热等逻辑。盘点期间机台视为已完成预热，盘点结束后可直接进入生产排产。

本规则复用项目已有的设备停机计划统一扣产能逻辑，不新增独立的扣减分支，确保盘点与精度校验、计划性维修、临时性故障等其他非清洗类停机类型保持一致的处理口径。

## Requirements

### Requirement: 盘点停机计划必须从设备停机计划统一加载

系统 SHALL 从设备停机计划表（`T_MDM_DEVICE_PLAN_SHUT`）加载停机类型为 `09`（盘点）的记录，与精度校验、润滑、巡检点检、预见性维护、预防性维护、计划性维修、临时性故障等非清洗类停机类型使用相同的加载逻辑。

设备停机类型映射如下：

| 停机类型编码 | 业务含义 | 排程处理方式 |
| --- | --- | --- |
| `00` | 精度校验 | 统一扣时间产能 |
| `01` | 润滑 | 统一扣时间产能 |
| `02` | 巡检点检 | 统一扣时间产能 |
| `03` | 预见性维护 | 统一扣时间产能 |
| `04` | 预防性维护 | 统一扣时间产能 |
| `05` | 计划性维修 | 统一扣时间产能 |
| `06` | 临时性故障 | 统一扣时间产能 + 标记维修计划 |
| `07` | 干冰清洗 | 转换为清洗窗口，独立扣减 |
| `08` | 喷砂清洗 | 转换为清洗窗口，独立扣减 |
| `09` | 盘点 | 统一扣时间产能，不触发换模/预热 |

加载口径：

```text
普通设备停机：按排程窗口交集加载（beginDate <= 窗口结束 AND endDate >= 窗口开始）
清洗类停机：按 T 日及之后的计划开始时间单独加载候选
```

盘点属于普通设备停机，MUST 使用普通加载口径，不得走清洗候选独立加载路径。

#### Scenario: 盘点停机计划被正确加载

- **Given** 设备停机计划表中存在机台 K1105 的盘点记录，停机类型为 `09`
- **And** 盘点时间段 `2026-07-07 05:30:00 ~ 2026-07-07 10:00:00` 与排程窗口相交
- **When** 系统加载设备停机计划
- **Then** 该盘点记录 MUST 被加载到 `context.getDevicePlanShutList()`
- **And** 不得被清洗候选加载路径重复加载

#### Scenario: 盘点不被识别为清洗类停机

- **Given** 设备停机计划中存在停机类型为 `09`（盘点）的记录
- **When** 系统执行 `filterNonCleaningStopPlans` 剥离清洗类停机
- **Then** 盘点记录 MUST 保留在普通停机列表中
- **And` 不得被 `isCleaningStopType` 识别为清洗

### Requirement: 盘点时间段必须统一扣减班次可生产计划量

系统 SHALL 将盘点时间段作为不可生产时间段，复用项目已有的停机扣产能逻辑（`ShiftCapacityResolverUtil.resolvePlannedStopOverlapSeconds` / `collectMergedPlannedStopIntervals`）统一折算扣减。

扣减规则：

```text
盘点重叠秒数 = 盘点时间段与班次时间窗的交集总秒数（多条记录先合并区间再计算）
班次可生产秒数 = 班次总秒数 - 所有非清洗类停机重叠秒数（含盘点）
班次最大计划量 = (可生产秒数 / 硫化时间秒数) 向下取整 * 模台数
```

盘点与其他非清洗类停机类型（精度校验、计划性维修等）在产能扣减上 MUST 保持完全一致的处理口径，不得为盘点新增独立的扣减分支。

#### Scenario: 盘点时间段扣除对应计划量

- **Given** 机台 K1105 在 `2026-07-07 05:30:00 ~ 10:00:00` 有盘点停机
- **And** 该时间段落在某班次窗口内
- **When** 系统计算该班次最大计划量
- **Then** 盘点时间段（4.5 小时）MUST 从班次可生产时间中扣除
- **And** 班次最大计划量 MUST 按扣除后的可生产时间折算

#### Scenario: 盘点与其他停机重叠时合并扣减

- **Given** 机台 K1105 在同一天有盘点停机 `05:30 ~ 10:00` 和精度校验 `09:00 ~ 11:00`
- **When** 系统计算班次停机重叠秒数
- **Then** 两个停机区间 MUST 先合并为 `05:30 ~ 11:00`
- **And` 按合并后区间统一扣减，不得重复扣减重叠部分

### Requirement: 盘点不得触发换模、换活字块、首检、预热

系统 SHALL 确保盘点停机只影响可生产时间和计划量，不改变机台当前模具状态，也不触发以下逻辑：

1. 换模（mould change）
2. 换活字块（type block change）
3. 首检（first inspection）
4. 预热（preheat / warm up）

在 `DataInitHandler` 初始化机台运行态时，盘点停机类型 MUST NOT 设置 `hasRepairPlan`（仅 `TEMPORARY_FAULT` 即临时性故障设置），也 MUST NOT 设置 `hasMaintenancePlan`（保养来自精度保养计划 `LhPrecisionPlan`，与设备停机计划无关）。

盘点结束后，机台的首个可生产时间点 MUST NOT 额外扣除换模、换活字块、预热等产能。机台在盘点前的在机 SKU 状态保持不变，盘点后可直接以续作方式继续排产。

#### Scenario: 盘点不触发维修计划

- **Given** 机台 K1105 有盘点停机记录（停机类型 `09`）
- **When** 系统初始化机台运行态
- **Then` `hasRepairPlan` MUST 为 false
- **And** `repairPlanTime` MUST 为 null
- **And** 机台准备就绪时间 MUST NOT 因盘点而抬高

#### Scenario: 盘点不触发保养计划

- **Given** 机台 K1105 有盘点停机记录（停机类型 `09`）
- **When** 系统初始化机台运行态
- **Then** `hasMaintenancePlan` MUST 不受盘点影响
- **And** 保养窗口 MUST 仍由精度保养计划（`LhPrecisionPlan`）独立触发

#### Scenario: 盘点后可直接续作排产

- **Given** 机台 K1105 盘点前在机 SKU 为物料 A
- **And** 盘点时间段为 `05:30 ~ 10:00`
- **When** 盘点结束后系统安排该机台排产
- **Then** 机台 MUST 以续作方式继续排产物料 A
- **And` MUST NOT 触发换模或换活字块
- **And** MUST NOT 额外扣除预热产能

### Requirement: 盘点不得影响其他停机/占用时间逻辑

系统 SHALL 确保盘点规则的引入不影响以下已有停机/占用时间逻辑：

1. 干冰清洗（`07`）的候选加载、班次归属、每日上限和重叠处理
2. 喷砂清洗（`08`）的候选加载、禁排日期和重叠处理
3. 精度校验（`00`）的停机扣减
4. 计划性维修（`05`）的停机扣减
5. 临时性故障（`06`）的维修计划标记和准备就绪时间抬高
6. 保养窗口的触发、安排和产能扣减
7. 换模、换活字块的重叠处理和产能扣减

盘点与其他停机类型在产能扣减上通过统一的区间合并逻辑处理，不得因盘点新增特殊分支或兜底逻辑。

#### Scenario: 盘点与清洗并存时各自独立处理

- **Given** 机台 K1105 同一天有盘点停机（`09`）和干冰清洗（`07`）
- **When** 系统处理设备停机计划
- **Then** 干冰清洗 MUST 被转换为运行态清洗窗口并从普通停机列表剥离
- **And** 盘点 MUST 保留在普通停机列表中统一扣产能
- **And** 两者的产能扣减 MUST 通过区间合并避免重复扣减

#### Scenario: 盘点不影响临时性故障的维修标记

- **Given** 机台 K1105 同时有盘点停机（`09`）和临时性故障（`06`）记录
- **When** 系统初始化机台运行态
- **Then** `hasRepairPlan` MUST 为 true（因临时性故障）
- **And** `repairPlanTime` MUST 取临时性故障的开始时间
- **And` 盘点记录 MUST NOT 干扰维修计划的标记逻辑

### Requirement: 盘点停机计划必须打印关键日志

系统 SHALL 在初始化机台运行态时，对每台机台的盘点停机计划打印日志，包含以下关键字段：

1. 机台编号
2. 盘点开始时间
3. 盘点结束时间

日志 MUST 简洁，不得打印敏感信息，不得用日志掩盖异常。

#### Scenario: 盘点停机计划日志输出

- **Given** 机台 K1105 有盘点停机记录
- **When** 系统初始化机台运行态
- **Then** 日志 MUST 输出机台编号、盘点开始时间和盘点结束时间
- **And` 日志级别为 INFO
