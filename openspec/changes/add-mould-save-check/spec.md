请基于现有项目代码，调整硫化排程中的模具分配、模具占用、SKU 增机台模具校验，以及排程结果模具字段赋值逻辑。

## 一、变更目标

当前硫化排程结果中的模具相关字段存的是“SKU 对应的全部模具号”，需要改为存“本次排程实际分配使用的具体模具号”。

要求：

* 双模机台：结果中只存本次实际分配的 2 个模具号；
* 单模机台：结果中只存本次实际分配的 1 个模具号；
* 不允许继续把 SKU 关联的全部模具号直接写入排程结果。

## 二、适用范围

优先调整：

* 新增排产；
* 新增排产换模上机；
* SKU 增机台；
* 换活字块排产；
* 排程结果落库时的模具字段赋值。

如果续作排产、换活字块排产复用了同一套模具校验或结果赋值逻辑，需要同步评估影响，优先复用公共方法，避免重复实现。

## 三、换模时的模具占用规则

新增排产中，如果机台发生真正换模，需要维护本次排程过程中的“已使用模具列表”。

### 1. 释放前物料模具

机台换模前已有前物料时：

* 只释放前物料在当前机台上实际占用的模具号；
* 从本次排程的已使用模具列表中移除；
* 不能释放该 SKU 关联的全部模具号。

### 2. 绑定新物料模具

新 SKU 上机时：

* 为新 SKU 分配本次实际使用的模具号；
* 分配成功后绑定到当前机台/SKU；
* 加入本次排程的已使用模具列表；
* 后续其他 SKU 不允许再选择已被占用的模具。

## 四、换活字块时的模具占用规则

必须区分“换模”和“换活字块”。

如果是换活字块：

* 不释放当前机台在机模具号；
* 不重新分配新的模具号；
* 继续沿用当前机台在机实际占用的模具号；
* 已使用模具列表中，该机台在机模具号仍保持占用状态；
* 排程结果中的模具字段，仍写入当前机台在机实际模具号。

原因：

* 换活字块不是更换整副模具；
* 换活字块使用的是同一副模具；
* 只是更换花纹块/活字块；
* 因此不能按换模逻辑释放前物料模具，也不能重新选择新模具。

只有发生真正换模时，才释放前物料实际占用模具，并为新 SKU 重新分配模具。

## 五、SKU 模具选择规则

SKU 选择模具时，按以下顺序处理：

1. 获取当前 SKU 可用模具列表；

    * 来源：SKU 与模具关系；
    * 模具状态来源：模具台账；
    * 只允许选择状态可用的模具。

2. 排除已被其他 SKU 占用的模具；

    * 已占用模具来自本次排程的已使用模具列表；
    * 已在真正换模时释放的模具，可以重新参与选择；
    * 换活字块场景不释放模具，原在机模具仍视为占用。

3. 按“模具共用 SKU 数量”升序排序；

    * 统计口径：以模具号为维度，根据 SKU 与模具关系统计该模具关联的不同 SKU 数量；
    * 如果某模具只关联当前 SKU，数量 = 1；
    * 如果某模具被当前 SKU 和另 1 个 SKU 共用，数量 = 2；
    * 如果某模具被当前 SKU 和另 2 个 SKU 共用，数量 = 3；
    * 依次类推；
    * 数量越小优先级越高，优先选择更专属、共用更少的模具；
    * 当共用 SKU 数量相同时，按模具号升序排序，模具号小的优先选择。

4. 按机台模数选择模具：

    * 双模机台选择排序后的前 2 个可用模具号；
    * 单模机台选择排序后的前 1 个可用模具号。

5. 如果可用模具数量不足：

    * 当前机台不能被选中；
    * 继续尝试其他候选机台；
    * 不允许生成缺少模具号的排程结果；
    * 日志需输出 SKU、机台、所需模具数量、可用模具数量、不可用/已占用模具号。

## 六、SKU 增机台时的模具校验规则

SKU 增机台时，不能只判断候选机台是否可用，还必须同时判断 SKU 对应模具是否可用、模具数量是否足够。

增机台前置校验：

1. 先判断候选机台是否可用；

    * 复用项目已有机台筛选逻辑；
    * 包括机台状态、禁排、规格匹配、单控/双模、换模条件等已有约束。

2. 再判断 SKU 模具是否可用；

    * 模具来源：SKU 与模具关系；
    * 模具状态来源：模具台账；
    * 排除本次排程中已被其他 SKU 占用的模具。

3. 判断模具数量是否满足机台模数；

    * 双模机台：至少需要 2 个可用模具号；
    * 单模机台：至少需要 1 个可用模具号。

4. 模具不足或不可用时：

    * 当前候选机台不能作为新增机台；
    * 继续尝试其他候选机台；
    * 不允许生成错误排程结果；
    * 需要输出明确日志。

5. 增机台成功后：

    * 将实际分配的模具号绑定到新增机台/SKU；
    * 加入本次排程已使用模具列表；
    * 排程结果中的模具字段写入本次实际分配的模具号。

## 七、排程结果赋值要求

排程结果中的模具字段必须来自“本次实际分配的模具号”。

要求：

* 真正换模：写入新 SKU 本次实际分配的模具号；
* 换活字块：写入当前机台在机实际模具号；
* 双模：写入 2 个实际模具号；
* 单模：写入 1 个实际模具号；
* 不允许写入 SKU 的全部模具号；
* 若模具号为空或数量不足，需要记录明确日志。

## 八、性能要求

实现时注意性能和内存占用：

1. SKU 与模具关系不要在循环中频繁查库；
2. 建议提前加载 SKU-模具关系，并按 SKU 分组缓存；
3. 建议提前统计“模具号 -> 关联 SKU 数量”的 Map；
4. 已使用模具列表建议使用 Set 维护；
5. 避免在 SKU × 机台循环中重复全量扫描关系数据；
6. 不允许引入明显的 OOM 风险。

## 九、实现要求

1. 优先复用项目已有机台筛选、模具校验、结果赋值逻辑；
2. 如果已有“模具可用性校验”逻辑，应在此基础上扩展为“具体模具号分配”；
3. 换模与换活字块必须分支处理，不能共用释放模具逻辑；
4. 新增或调整的方法需要添加详细注释；
5. 关键分支需要输出日志，至少包括：

    * SKU 无可用模具；
    * 可用模具被其他 SKU 占用；
    * 单模/双模模具数量不足；
    * 真正换模释放前物料模具；
    * 换活字块沿用在机模具；
    * 新 SKU 绑定模具成功；
    * SKU 增机台因模具不足被跳过；
    * 排程结果写入实际模具号。

## 十、验收标准

1. 排程结果中模具字段只包含本次实际使用的模具号。
2. 双模机台结果中有且仅有 2 个实际模具号。
3. 单模机台结果中有且仅有 1 个实际模具号。
4. 不同 SKU 在同一时间段内不能占用同一个模具号。
5. 真正换模后，前物料实际占用的模具号会被释放。
6. 新上机 SKU 实际分配的模具号会加入已使用列表。
7. 换活字块场景不释放在机模具号，不重新分配模具号，继续使用当前机台在机实际模具号。
8. 只有真正换模时，才允许释放前物料实际占用模具，并为新 SKU 分配新的实际模具号。
9. SKU 选择模具时，会先排除已占用模具，再按共用 SKU 数量升序选择。
10. 模具共用 SKU 数量必须按 SKU 与模具关系中“该模具关联的 SKU 总数”统计：

    * 只关联当前 SKU 时数量为 1；
    * 当前 SKU 与另 1 个 SKU 共用时数量为 2；
    * 当前 SKU 与 N 个其他 SKU 共用时数量为 N + 1。
11. 共用 SKU 数量相同时，按模具号升序选择。
12. SKU 增机台时，必须同时满足“机台可用 + SKU 模具可用 + 模具数量足够”。
13. 双模增机台必须有 2 个可用模具号，单模增机台必须有 1 个可用模具号。
14. 模具不足时，不允许新增该机台，不生成错误排程结果，并输出明确日志。
15. 不影响现有新增排产的机台筛选、排序、换模时间扣减、换活字块开产、加机台等核心逻辑。
