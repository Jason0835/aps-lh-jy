# 设计说明

## 根因判断

本次问题不是候选机台匹配失败，而是新增排产在 dayN 产能模拟后，把“结构收尾/日计划窗口模拟需要关注”误用成“必须把目标量按多台均分”。当一台机台的理论窗口产能已经覆盖日标准产量或窗口计划时，仍进入多机台均衡拆量，导致首台只排一半，后续可用班次被空置。

换活字块场景下，前序换活字块结果已经贡献同物料窗口产能，但新增排产继续按新机台缺口计算，也会造成重复加机台。

## 推荐方案

沿用现有 `NewSpecProductionStrategy` 主流程，在动态机台数判断结果进入计划量分配前增加规则收口：

1. 如果当前机台理论窗口产能已经覆盖本轮窗口计划或日标准产能要求，则当前机台继续按真实可排窗口排满，不再按多机台均分拆量。
2. 如果同物料已有换活字块/续作/新增结果已满足 dayN 模拟的有效机台需求，则不再继续新增同物料机台。
3. 欠产超过阈值且窗口后剩余欠产仍超过阈值的场景，继续复用原动态加机台规则。

## 影响范围

- 入口方法：`NewSpecProductionStrategy#scheduleNewSpecs`
- 核心方法：`schedulePendingNewSpecsRound`、`resolveDynamicMachinePlanQtyByDailyCapacity`、`resolveRequiredMachineCountByDailyCapacity`
- 数据影响：仅影响运行态排程结果、日计划滚动扣减和换模结果，不涉及 SQL/XML。

## 验证策略

- 增加策略级回归测试覆盖 2026-06-02 单机可满足但被拆量的场景。
- 保留并运行既有 dayN 加机台、欠产阈值、日标准产量与换活字块相关测试。
- 必要时按日期 2026-06-02 发起真实硫化排程验证落库结果。
