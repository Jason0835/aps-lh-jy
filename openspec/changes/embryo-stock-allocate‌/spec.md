请调整共用胎胚分配胎胚库存规则，以本规则为准。

核心规则：
1. 同一个胎胚被不同 SKU 共用时，不是所有场景都需要分摊胎胚库存。
2. 只有满足以下条件时，才需要对共用胎胚库存进行分摊：
    - 不同 SKU 共用同一个胎胚；
    - 且这些共用胎胚的 SKU 在同一天收尾。
3. 同一个 SKU 在不同机台上生产时，不需要分摊胎胚库存。
    - 即：同 SKU 多机台不算“共用胎胚分摊”场景；
    - 只有不同 SKU 满足共用胎胚且同一天收尾时，才进入分摊逻辑。

分摊口径：
1. 目标 SKU 实际应分配的胎胚库存，按照 SKU 标准产能占比分摊。
2. 标准产能占比分摊逻辑复用项目现有实现，不要重新引入“日硫化量 * 台数 * 模数”的新权重算法。
3. 本次调整重点是分摊触发条件和收尾后动态剔除逻辑，不是重写分摊公式。

排程窗口内动态处理：
1. 如果本次排程窗口中，某一天有共用胎胚 SKU 收尾完成：
    - 从后续天开始，再计算该共用胎胚库存分摊时，必须剔除已经收尾完成的 SKU；
    - 只在剩余仍未收尾、仍参与该胎胚共用的不同 SKU 之间重新计算分摊。
2. 如果剔除后只剩一个 SKU 使用该胎胚：
    - 后续该 SKU 应识别为单胎胚/非共用胎胚场景；
    - 不再进行共用胎胚库存分摊。

尾差处理：
1. 胎胚库存分摊结果需要处理整数尾差。
2. 前 N-1 个 SKU 可按项目现有取整口径处理。
3. 最后一个 SKU 必须分配剩余全部胎胚库存：

   lastAllocatedStock = sharedGreenTireStock - sum(前面 SKU 已分配胎胚库存)

4. 必须确保所有 SKU 分配结果汇总严格等于共用胎胚总库存。

实现要求：
1. 优先复用项目已有的共用胎胚识别、SKU 标准产能占比分摊、收尾判断逻辑。
2. 不要把同 SKU 多机台误判为不同 SKU 共用胎胚分摊。
3. 增加必要日志，至少输出：
    - 胎胚编码/胎胚描述；
    - 当前参与分摊的 SKU 列表；
    - 是否满足“不同 SKU + 同一天收尾”；
    - 每个 SKU 的标准产能、占比、分配胎胚库存；
    - 哪些 SKU 因已收尾在后续天被剔除；
    - 尾差处理前后结果。
4. 增加单元测试覆盖：
    - 同 SKU 多机台，不分摊；
    - 不同 SKU 共用胎胚，但不是同一天收尾，不分摊；
    - 不同 SKU 共用胎胚，且同一天收尾，需要分摊；
    - 排程窗口内某天一个 SKU 收尾后，后续天剔除该 SKU 再计算；
    - 剔除后只剩一个 SKU，后续识别为单胎胚；
    - 多 SKU 分摊存在整数尾差，最后一个 SKU 分配剩余库存；
    - 共用胎胚库存为 0；
    - 分摊 SKU 标准产能为 0 或异常时，按项目现有兜底逻辑处理并输出日志。
