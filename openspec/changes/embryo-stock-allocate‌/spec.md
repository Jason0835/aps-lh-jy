# 胎胚库存账本与共用胎胚内部分摊规则

## Purpose

统一胎胚收尾场景的排产目标量、共用胎胚 SKU 额度分配、组级库存消费账本和结果库存落库口径，避免不同 SKU 或同 SKU 多机台重复消费同一胎胚库存。

## Requirements

### Requirement: T日胎胚收尾目标量

系统必须（MUST）只在“胎胚收尾=是、SKU 已标记收尾、`endingDaysRemaining == 1`”时进入胎胚库存硬目标逻辑。

#### Scenario: 单胎胚T日胎胚收尾

- **WHEN** 非共用胎胚 SKU 命中胎胚收尾且在 T 日收尾
- **THEN** 排产目标量必须等于原始胎胚库存
- **AND** 必须按 `胎胚代码 + "_" + T日业务日期` 创建独立胎胚库存消费账本

#### Scenario: 单胎胚非胎胚收尾

- **WHEN** 非共用胎胚 SKU 未命中胎胚库存 T 日硬目标
- **THEN** SKU 收尾仍按 `MAX(胎胚库存, 硫化余量)` 计算目标量
- **AND** SKU 非收尾仍按硫化余量计算目标量

### Requirement: 胎胚库存消费账本

系统必须（MUST）为胎胚库存硬目标场景维护运行态消费账本，账本 key 固定为 `embryoCode + "_" + scheduleDate`，其中 `scheduleDate` 是 T 日业务日期，只保存 `yyyy-MM-dd` 日期。

账本字段必须包含：

- `embryoCode`：胎胚代码
- `scheduleDate`：T 日业务日期
- `originalStockQty`：原始胎胚库存
- `targetQty`：本次组级目标量，固定等于原始胎胚库存
- `consumedQty`：已消费计划量
- `remainQty`：剩余可消费量

#### Scenario: 结果落地前硬裁剪

- **WHEN** 新增、续作或换活字块生成排程结果
- **THEN** 结果计划量必须同时受 SKU 剩余额度和胎胚库存账本 `remainQty` 控制
- **AND** 每确认一笔排产量后必须同步扣减 SKU 账本和胎胚组级账本
- **AND** 当 `remainQty <= 0` 时，同胎胚组不得继续排产

### Requirement: 共用胎胚T日同日收尾内部分摊

系统必须（MUST）只在不同 SKU 共用同一胎胚、胎胚收尾=是、这些 SKU 都在 T 日同日收尾时，对同胎胚组创建组级硬目标。

#### Scenario: 共用胎胚组级目标

- **WHEN** 共用胎胚 SKU 命中 T 日同日胎胚收尾
- **THEN** 组级目标量必须等于原始胎胚库存
- **AND** 不得使用 `MAX(胎胚库存, 硫化余量)`
- **AND** 不得让不同 SKU 各自按完整胎胚库存排产

#### Scenario: SKU级初始分摊

- **WHEN** 共用胎胚组创建组级账本
- **THEN** 必须按 SKU 标准产能占比分配 SKU 内部额度
- **AND** 标准产能缺失时沿用项目已有日标准产能兼容口径
- **AND** 整数尾差由最后一个 SKU 承接，确保 SKU 内部额度合计不超过组级目标量

#### Scenario: 换模可行性修正

- **WHEN** 共用胎胚 SKU 完成初始分摊
- **THEN** 必须基于现有候选机台、T 日早中班可排产能、换模/换活字块可行性、模具和机台约束进行只读预评估
- **AND** 无候选机台、T 日无法合理消化或会卡晚班不可换模的 SKU，其不可消化额度允许转给同胎胚组内更可行的 SKU
- **AND** 可行性修正不得直接生成排程结果，不得污染真实机台、模具或换模上下文

### Requirement: 胎胚库存落库不分摊

系统必须（MUST）保证排程结果中的胎胚库存字段始终保存原始胎胚库存。

#### Scenario: 不同SKU共用同一胎胚

- **WHEN** 胎胚库存为 100，SKU A 内部额度为 60，SKU B 内部额度为 40
- **THEN** SKU A 的排程结果胎胚库存必须保存 100
- **AND** SKU B 的排程结果胎胚库存必须保存 100

#### Scenario: 同SKU多机台

- **WHEN** 同一 SKU 拆分到多台机台生产
- **THEN** 每台机台结果的胎胚库存必须保存该胎胚原始库存
- **AND** 不得按机台数或机台计划量二次分摊胎胚库存

### Requirement: 动态有效SKU集合

系统必须（MUST）按运行态有效 SKU 集合判断共用胎胚关系。

#### Scenario: 共用胎胚SKU完成或未排后剔除

- **WHEN** 共用胎胚组内部分 SKU 已收尾完成、目标量为 0 或写入未排
- **THEN** 后续必须从该胎胚有效 SKU 集合中剔除这些 SKU
- **AND** 剩余 SKU 重新判断是否仍满足 T 日同日胎胚收尾
- **AND** 剩余只剩一个 SKU 时，应按单胎胚 T 日胎胚收尾独立账本处理

## Notes

- 本规则只影响胎胚库存 T 日硬目标场景。
- 非 T 日胎胚收尾、普通 SKU 收尾、非共用胎胚未收尾、非收尾满班产逻辑保持原有规则。
- 分摊和修正只用于内部排产额度控制，不得写入 `EMBRYO_STOCK` 结果字段。
- 晚班补满、主销/常规产品收尾补满、普通收尾补满、同 SKU 多机台排满、非收尾满班产逻辑均不得突破胎胚库存账本组级目标。
