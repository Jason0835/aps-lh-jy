# 多机台动态分配计划量需求描述及案例推演（最终最新版）

> 适用范围：本文档仅针对**新增排产 SKU** 的多机台动态分配计划量逻辑。  
> 不适用于**续作排产**。续作排产包含 MES 在机续作、换模续作、换活字块续作等场景，应走续作专用逻辑，不在本文档范围内。

---

## 1. 需求背景

硫化排程中，一个 SKU 在一台机台上可能无法在排程窗口内完成目标排产量，因此需要支持同一个 SKU 在多台硫化机台上动态分配计划量。

该逻辑需要同时满足：

1. 硫化排程主流程仍保持 **SKU 连续排 8 个班次**；
2. 虽然 SKU 连续排 8 班，但计划量需要按 day1/day2/day3 做日计划节奏控制；
3. 日计划未完成部分允许在窗口内滚动追补；
4. 新增排产 SKU 在单机台产能不足时，需要动态增加机台；
5. 正规/量试非收尾场景，已开机台当天剩余班次需要补满班产；
6. 试制/收尾场景严格按照目标量排产，不允许超排；
7. 同 SKU 多机台收尾时，如果属于同胎胚/共用胎胚，需要结合收尾班次、晚班不能换模、换模均衡等规则判断是否需要错开换模。

---

## 2. 适用范围说明

### 2.1 本文档适用范围

本文档只适用于：

```text
新增排产 SKU
```

即：

```text
当前 SKU 不是续作继承而来，
而是在新增排产过程中，被算法选择后安排到一台或多台机台上生产。
```

适用场景包括：

1. 新增换模排产；
2. 新增机台承接 SKU；
3. 一个 SKU 因单台机台产能不足，需要拆分到多台机台；
4. 新增排产 SKU 的目标量分配、日计划滚动追补、多机台收尾处理。

### 2.2 本文档不适用范围

本文档不适用于：

```text
续作排产
```

续作排产包括但不限于：

1. MES 已在机 SKU 续作；
2. 上一轮排程结果滚动衔接；
3. 换活字块续作；
4. 续作收尾后的反选 SKU；
5. 已经在某台机台生产中的 SKU 继续占用后续班次。

续作排产应走单独逻辑，不使用本文档中的新增排产动态加机台逻辑。

## 3. 核心排程模式

硫化排程主流程仍保持：

```text
SKU 连续排 8 个班次
```

即：

```text
一个 SKU 一旦开始排产，仍按排程窗口内连续 8 班进行排程判断。
```

但这里的“连续 8 班”不是指必须把 8 个班次全部排满，而是指：

1. SKU 在机台上的排程区间需要保持连续；
2. 不能中间跳班；
3. 不能断开后又继续排同一个 SKU；
4. 达到严格目标量后可以停止；
5. 正规/量试非收尾场景，已开机台当天剩余班次如有产能，需要补满班产。

因此，整体控制逻辑是：

```text
连续 8 班主流程不变；
按天控制排产节奏；
欠产可以向后滚动；
最终窗口累计量必须满足目标排产量。
```

## 4. 目标排产量规则

### 4.1 非收尾场景

如果 SKU 是非收尾，则目标排产量等于排程窗口内月计划日计划量累计之和。

```text
非收尾目标排产量 = day1 + day2 + day3 + ...
```

本案例窗口为 3 天：

```text
目标排产量 = day1 + day2 + day3
```

非收尾场景下需要继续区分 SKU 类型：

| SKU 类型 | 目标量性质 | 是否允许超排 | 说明 |
|---|---:|---:|---|
| 正规非收尾 | 最低目标量 | 允许适度超排 | 满足当日月度计划余量后，已开机台当天剩余班次如有剩余产能，需要全部补满班产 |
| 量试非收尾 | 最低目标量 | 允许适度超排 | 同正规非收尾 |
| 试制非收尾 | 严格目标量 | 不允许超排 | 严格按照目标量排产，达标后停止 |

### 4.2 收尾场景

如果 SKU 是收尾，则目标排产量取：

```text
收尾目标排产量 = MAX(余量, 胎胚库存)
```

收尾场景必须严格按照目标量排产：

1. 不允许超排；
2. 不允许为了补满班产而超过目标量；
3. 最后一个班次可以小于班产；
4. 达到目标量后，后续班次不再继续排该 SKU。

例如：

```text
班产 = 16
剩余目标量 = 14
```

收尾场景只能排：

```text
14
```

不能补满到 16。

## 5. 日计划滚动追补规则

day1/day2/day3 是日计划节奏控制，不是绝对日上限。

例如：

```text
day1 = 96
day2 = 48
day3 = 14
```

不能简单理解为：

```text
05-01 必须刚好完成 96；
05-02 必须刚好完成 48；
05-03 必须刚好完成 14。
```

更准确的理解是：

1. day1 是第一天目标节奏；
2. day1 未完成部分形成欠产；
3. day1 欠产可以滚动到 day2 追补；
4. day2 如果仍未完全追补，可以继续滚动到 day3；
5. 最终窗口累计量必须达到目标排产量。

计算公式：

```text
当日欠产量 = MAX(0, 当日目标量 - 当日实际排产量)
累计欠产量 = MAX(0, 累计目标量 - 累计实际排产量)
窗口剩余目标量 = 目标排产量 - 窗口累计实际排产量
```

## 6. 是否增加机台规则

### 6.1 判断口径

是否增加机台需要按天进行判断，而不是只按照整个排程窗口一次性判断。

基本思想：

```text
当天可排产量不足时，需要判断后续可用班次是否能够消耗当天欠产，并同时满足后续自身目标量。
如果后续无法消耗，则需要在当前天增加机台。
```

### 6.2 当前日欠产判断

```text
当前日欠产量 = MAX(0, 当前日目标量 - 当前已选机台当前日可排产量)
```

如果：

```text
当前日可排产量 >= 当前日目标量
```

则当前日不因日计划不足触发新增机台。

如果：

```text
当前日可排产量 < 当前日目标量
```

则当前日会产生欠产，需要继续判断后续是否可以追补。

### 6.3 后续追补能力判断

后续一天需要承接：

```text
后续一天需要承接量 = 当前日欠产量 + 后续一天自身目标量
```

如果：

```text
后续一天可排产量 >= 后续一天需要承接量
```

说明当前日欠产可以在后续一天消化，当前日可以不增加机台。

如果：

```text
后续一天可排产量 < 后续一天需要承接量
```

说明后续一天无法同时完成：

1. 前一天欠产；
2. 后续一天自身目标量。

此时需要结合配置的追补判断天数继续评估。

建议参数：

```text
lookAheadDays：欠产追补判断天数，默认 1，可配置。
```

如果在配置的追补判断范围内仍无法消耗欠产，则需要在当前日增加机台。

### 6.4 新增机台数量判断

新增机台时，不是固定增加 1 台，而是按缺口动态增加。

```text
缺口量 = 当前日目标量 + 后续需保障目标量 - 已选机台可排产量
```

按候选机台排序逐台加入，直到满足：

```text
已选机台在判断范围内的可排产量 >= 判断范围内的目标量 + 前序欠产
```

同时必须满足：

1. 机台可生产该 SKU；
2. 机台状态可用；
3. 换模时间可满足；
4. 不违反晚班不能换模；
5. 不违反换模均衡；
6. 不违反收尾不超排；
7. 不违反同 SKU 多机台收尾换模错开规则。

## 7. 计划量分配规则

### 7.1 正规/量试非收尾

正规/量试非收尾场景：

```text
目标排产量 = 月计划日计划量累计之和
```

该目标量是最低目标量。

当达到最低目标量后，如果满足当日月度计划余量，且已开机台当天对应的剩余班次仍有剩余产能，则需要全部补满班产。

即：

```text
正规/量试非收尾：
达到最低目标量后，
已开机台当天剩余班次如有产能，
需要补满班产，
允许适度超排。
```

示例：

```text
班产 = 16
当前剩余最低目标量 = 14
```

正规/量试非收尾可以排：

```text
16
```

因为已开机台当天剩余班次需要补满。

### 7.2 试制非收尾

试制非收尾场景：

```text
严格按照目标量排产；
达到目标后停止；
不补满班产；
不允许超排。
```

示例：

```text
班产 = 16
剩余目标量 = 14
```

试制非收尾只能排：

```text
14
```

### 7.3 收尾

收尾场景：

```text
目标排产量 = MAX(余量, 胎胚库存)
```

规则：

1. 严格按照目标量排产；
2. 不允许超排；
3. 不补满班产；
4. 达到目标后停止；
5. 后续班次不再继续排该 SKU。

示例：

```text
班产 = 16
剩余目标量 = 14
```

收尾只能排：

```text
14
```

不能补到 16。

## 8. 机台连续排产规则

一个 SKU 一旦在某台机台开产，该机台上的排产需要保持连续。

即：

```text
不能中间跳班；
不能断开后再继续排同一个 SKU；
从开产班次开始，按照可用班次连续排产。
```

但连续排产不代表必须排满全部 8 班。

对于收尾、试制等严格目标场景：

```text
达到目标量后可以停止；
后续班次不再继续排该 SKU。
```

对于正规/量试非收尾：

```text
达到最低目标量后，
已开机台当天对应的剩余班次如有剩余产能，
仍需要补满班产。
```

## 9. 同 SKU 多机台收尾换模错开规则

### 9.1 适用前提

该规则适用于：

```text
收尾场景 + 同 SKU 多机台生产
```

由于是同一个 SKU 多机台生产，默认认为这些机台属于：

```text
同胎胚 / 共用胎胚
```

因此，在 SKU 收尾后，后续新 SKU 上机可能会触发换模冲突，需要判断是否错开。

### 9.2 基础约束

同 SKU 多机台收尾换模处理时，需要同时满足：

1. 晚班不能换模；
2. 换模次数需要均衡；
3. 收尾不允许超排；
4. 不能为了错开换模而损失更多有效产能；
5. 不能破坏机台连续排产规则。

换模均衡限制：

| 班次 | 最大换模次数 |
|---|---:|
| 早班 | 8 次 |
| 中班 | 7 次 |
| 一天合计 | 15 次 |

即：

```text
早班换模次数 <= 8
中班换模次数 <= 7
当天换模总次数 <= 15
```

### 9.3 收尾班次是晚班时

如果同 SKU 多机台收尾发生在晚班，则：

```text
不需要为了错开换模而强行调整尾量分配；
只需要判断后续换模是否满足换模均衡。
```

原因：

```text
晚班不能换模。
```

即使多个机台在晚班同时收尾，新 SKU 也不能在晚班立即换模，只能等到后续早班换模、中班开产。

因此，如果为了错开收尾而把部分尾量从晚班挪到早班或中班，反而可能浪费产能。

### 9.4 晚班收尾示例说明

以本文案例为例，C6 是 2026-05-03 晚班。

#### 情况 1：不做收尾错开

```text
K1105：C6 = 16
K1110：C6 = 14
```

此时两个机台都在 C6 晚班收尾。

由于晚班不能换模，后续新 SKU 上机只能：

```text
C7 早班换模
C8 中班开产
```

两个机台各剩下 C7、C8 两个班次：

```text
C7：换模
C8：开产
```

因此两个机台都还可以在 C8 产生一个班次的产能。

#### 情况 2：为了错开换模，强行把 K1110 的 14 挪到 K1105 C7

```text
K1105：C6 = 16，C7 = 14
K1110：C6 = 0，C7 = 0
```

此时：

1. K1110 在 C5 后释放；
2. 但 C6 是晚班，不能换模；
3. K1110 只能 C7 早班换模，C8 中班开产；
4. K1105 在 C7 收尾，剩余 C8 只能用于后续换模，无法再开产。

这种情况下，虽然实现了收尾错开，但比情况 1 少了一个班次的有效产能。

因此：

```text
当收尾班次是晚班时，不建议为了错开换模而调整尾量。
```

本案例属于该情况，所以推荐保留：

```text
K1105 C6 = 16
K1110 C6 = 14
```

只需要继续校验：

```text
C7 早班换模次数 <= 8
C8 中班换模次数 <= 7
当天换模总次数 <= 15
```

### 9.5 收尾班次是早班或中班时

如果同 SKU 多机台收尾发生在早班或中班，则需要考虑收尾换模错开。

原因：

1. 早班/中班收尾后，后续仍可能存在可换模、可开产的班次；
2. 如果多个机台同班次收尾，后续可能集中换模；
3. 集中换模会占用后续有效产能；
4. 适当错开收尾，可以让部分机台提前释放，后续新 SKU 更早开产。

此时需要同时满足：

```text
需要换模错开；
也需要保证换模均衡。
```

### 9.6 早班/中班收尾示例说明

假设剩余目标量较多，出现以下两种分配方式。

#### 情况 1：不做收尾错开

```text
K1105：C6 = 16，C7 = 16
K1110：C6 = 16，C7 = 14
```

此时两个机台都在 C7 早班收尾。

如果不做错开，后续 C8 中班可能只能集中换模，无法再安排新 SKU 开产，造成产能浪费。

#### 情况 2：做收尾错开

```text
K1105：C6 = 16，C7 = 16，C8 = 14
K1110：C6 = 16，C7 = 0
```

此时：

1. K1110 在 C6 晚班后释放；
2. K1110 可在 C7 早班换模；
3. K1110 可在 C8 中班开产；
4. K1105 继续承接尾量，到 C8 中班收尾。

相比情况 1，情况 2 多释放出一个可开产班次，产能利用更好。

因此：

```text
当收尾班次是早班或中班时，应优先考虑收尾错开。
```

### 9.7 收尾换模错开的最终目标

同 SKU 多机台收尾换模错开的最终目的不是单纯追求“收尾班次不同”，而是：

```text
尽量不浪费产能；
同时保证换模均衡。
```

因此，不能机械地认为：

```text
多机台收尾一定要错开。
```

而应根据收尾班次判断：

| 收尾班次 | 是否需要强制错开 | 原因 |
|---|---|---|
| 晚班 | 不需要强制错开 | 晚班不能换模，强行错开可能反而浪费一个班次产能 |
| 早班 | 需要考虑错开 | 后续中班可能可以换模或开产，错开可减少产能浪费 |
| 中班 | 需要考虑错开 | 需要结合后续窗口和换模均衡判断 |

### 9.8 推荐判断流程

```text
1. 判断是否收尾。
   如果不是收尾，不进入同 SKU 多机台收尾换模错开逻辑。

2. 判断是否同 SKU 多机台。
   如果不是多机台，不进入错开逻辑。

3. 默认同 SKU 多机台视为同胎胚/共用胎胚。

4. 判断多个机台的预计收尾班次。
   如果多个机台不是同班次收尾，则不需要额外调整，只校验换模均衡。

5. 如果多个机台同班次收尾：
   5.1 如果收尾班次是晚班：
       不强制错开，只校验早班/中班换模均衡。
   5.2 如果收尾班次是早班或中班：
       尝试调整尾量分配，使部分机台提前释放，部分机台承接尾量。
       同时校验：
       - 不超排；
       - 不破坏连续排产；
       - 不损失更多有效产能；
       - 满足早班/中班/当天换模次数限制。
```

## 11. 班次结构

| 班次 | 日期 | 班次说明 |
|---|---|---|

## 14. 同 SKU 多机台收尾换模判断

第三天剩余目标量为：

```text
30
```

当前有两种可选分配方式。

### 14.1 方案 A：晚班同时收尾，不强制错开

| 机台 | C6 晚班 | C7 早班 | C8 中班 | 第三天合计 |
|---|---:|---:|---:|---:|
| K1105 | 16 | 空 | 空 | 16 |
| K1110 | 14 | 空 | 空 | 14 |
| 合计 | 30 | 空 | 空 | 30 |

该方案中：

```text
K1105 和 K1110 都在 C6 晚班收尾。
```

由于：

```text
C6 是晚班，晚班不能换模。
```

所以不需要为了错开换模而把尾量挪到 C7。

后续新 SKU 上机可以：

```text
C7 早班换模
C8 中班开产
```

两个机台都还保留了：

```text
C7 + C8 两个后续班次空间
```

即：

```text
C7 用于换模；
C8 用于新 SKU 开产。
```

因此，方案 A 不浪费额外产能。

只需要校验：

```text
C7 早班换模次数 <= 8
C8 中班换模次数 <= 7
当天换模总次数 <= 15
```

### 14.2 方案 B：强行错开收尾

| 机台 | C6 晚班 | C7 早班 | C8 中班 | 第三天合计 |
|---|---:|---:|---:|---:|
| K1105 | 16 | 14 | 空 | 30 |
| K1110 | 空 | 空 | 空 | 0 |
| 合计 | 16 | 14 | 空 | 30 |

该方案中：

```text
K1110 提前释放；
K1105 延迟到 C7 收尾。
```

但由于：

```text
C6 是晚班，K1110 即使提前释放，也不能在 C6 换模；
K1110 仍然只能 C7 早班换模，C8 中班开产。
```

同时：

```text
K1105 在 C7 才收尾，剩余 C8 只能用于后续换模，无法再开产。
```

因此，方案 B 虽然实现了收尾错开，但比方案 A 少一个班次的有效产能。

### 14.3 本案例最终选择

因为本案例的多机台收尾班次是：

```text
C6 晚班
```

且晚班不能换模，所以：

```text
不需要强制进行收尾错开。
```

最终选择方案 A：

```text
K1105 C6 = 16
K1110 C6 = 14
```

并进行换模均衡校验。
