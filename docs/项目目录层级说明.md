# 项目目录层级说明

## 项目根目录

```
aps-lh-parent/
├── pom.xml                    # 父 POM
├── aps-lh/                    # 核心排程模块（Spring Boot 启动模块）
├── aps-lh-api/                # API 定义模块（实体/DTO/枚举/常量，供 aps-lh 依赖）
├── docs/                      # 项目文档
├── openspec/                  # OpenSpec 规范
└── scripts/                   # 脚本
```

---

## aps-lh 模块分层

**基础包路径**：`com.zlt.aps.lh`

```
aps-lh/src/main/java/com/zlt/aps/
├── ApsLhApplication.java              # Spring Boot 启动类
└── lh/
    ├── config/                         # 配置层（线程池配置、属性配置）
    ├── controller/                     # 控制器层（排程入口触发接口）
    ├── service/                        # 服务接口
    │   └── impl/                       # 服务实现（数据初始化 / 核心排程 / 清洗保养 / 滚动衔接 / 持久化）
    ├── mapper/                         # MyBatis-Plus Mapper 层（约 30 个 Mapper）
    ├── handler/                        # 排程步骤处理器（模板方法模式：前置校验→数据初始化→续作排产→新品排产→结果校验→排程调整）
    ├── engine/                         # 排程引擎（核心编排层）
    │   ├── template/                   # 模板方法（定义排程执行顺序）
    │   ├── decorator/                  # 装饰器模式（日志/性能监控增强）
    │   ├── chain/                      # 责任链模式
    │   │   └── validators/             # 数据校验器（8个：机台/月计划/模具关系/SKU产能/施工/特殊材料BOM/试方/工作日历）
    │   ├── strategy/                   # 策略模式（排程算法核心）
    │   │   ├── impl/                   # 策略实现（12个：续作/新品/活字块/产能计算/收尾判定/首检均衡/插入订单/机台匹配/换模均衡/停产/SKU优先级/试产）
    │   │   └── support/                # 策略支撑类（日产能模拟/机台生产段/排产量策略等）
    │   ├── factory/                    # 工厂模式（策略工厂）
    │   └── observer/                   # 观察者模式
    │       └── listeners/              # 事件监听器（日志/MES通知/精准计划日期填充）
    ├── component/                      # 独立组件（序列号/批次号/配制解析/机台索引/订单号/执行守护）
    ├── context/                        # 排程上下文（配置上下文 + 运行时上下文）
    ├── util/                           # 工具类（约 17 个：机台匹配/模具/特殊材料/班次/产能/停产等）
    └── exception/                      # 异常定义（错误码/异常类/异常工具）
```

**资源文件**：
```
aps-lh/src/main/resources/
├── application.yml                     # 应用配置
├── mapper/lh/                          # MyBatis XML（4个复杂查询）
└── sql/                                # 数据库迁移 SQL（约 18 个）
```

## aps-lh-api 模块分层

多包结构：`com.zlt.aps.lh` / `com.zlt.aps.mdm` / `com.zlt.aps.mp` / `com.zlt.aps.cx` / `com.ruoyi.common`

```
aps-lh-api/src/main/java/

├── com/zlt/aps/lh/api/
│   ├── constant/                       # 常量定义（排程常量/参数常量/校验分组）
│   ├── enums/                          # 枚举（约 20 个：机台状态/班次/排程类型/SKU标签/试产状态/换模类型等）
│   ├── domain/
│   │   ├── entity/                     # 数据实体（约 18 个表实体）
│   │   ├── dto/                        # 数据传输对象（请求/响应/清洗窗口/保养窗口/排产控制/校验结果等）
│   │   └── vo/                         # 视图对象
│   └── util/                           # API 层工具类（班次时间/边界/类型解析）
│
├── com/zlt/aps/mdm/api/domain/entity/  # 主数据实体（工作日历/模具/物料/SKU产能/月计划余量/停机计划等）
├── com/zlt/aps/mp/api/domain/entity/   # 月计划实体（月计划结果/调整结果/生产版本/特殊材料记录等）
├── com/zlt/aps/cx/api/domain/entity/   # 成品库存实体（CX 系统）
├── com/zlt/common/annotation/          # 公共注解
└── com/ruoyi/common/core/              # 若依框架基类（BaseEntity/Excel 注解）
```

## 分层职责总结

| 层次 | 所属模块 | 职责 |
|------|---------|------|
| **controller** | aps-lh | 排程入口，接收请求触发排程 |
| **handler** | aps-lh | 排程步骤编排（模板方法模式），按序执行：校验→初始化→续作→新品→结果校验→调整 |
| **engine/template** | aps-lh | 定义排程执行骨架流程 |
| **engine/strategy** | aps-lh | 排程算法核心，每种算法策略独立实现，可插拔 |
| **engine/decorator** | aps-lh | 对排程执行器做日志/性能监控增强 |
| **engine/chain** | aps-lh | 前置数据校验链，可灵活组合校验规则 |
| **engine/observer** | aps-lh | 排程事件发布/监听（日志/MES通知/精准计划） |
| **service** | aps-lh | 业务服务编排（数据初始化、排程调度、结果持久化、滚动衔接） |
| **mapper** | aps-lh | 数据库访问映射 |
| **component** | aps-lh | 独立功能组件 |
| **context** | aps-lh | 排程全生命周期运行时数据 |
| **util** | aps-lh | 业务工具类（机台匹配/模具/班次/产能/停产等） |
| **api/entity** | aps-lh-api | 数据表结构定义，跨模块共享 |
| **api/dto** | aps-lh-api | 接口入参/出参/内部传递对象 |
| **api/enums** | aps-lh-api | 业务枚举定义，统一语义 |
| **api/constant** | aps-lh-api | 排程相关常量集中管理 |
