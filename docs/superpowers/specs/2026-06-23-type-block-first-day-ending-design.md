# 换活字块仅处理 T 日收尾机台设计

## 1. 明确结论

在 `TypeBlockProductionStrategy.scheduleTypeBlockChange(...)` 的换活字块候选机台准备阶段增加统一前置判断：仅保留预计收尾时间自然日期等于排程窗口起点 `context.scheduleDate` 的机台。

T+1、T+2 或其他非 T 日收尾机台不参与 S4.4 换活字块判断，不生成换活字块结果，不消费模具、胎胚、日计划和切换配额，保留原机台运行态和待排 SKU，由 S4.5 新增排产流程统一处理。

## 2. 改造目标

- 防止非 T 日收尾机台提前参与换活字块并占用后续新增排产资源。
- 保持 T 日收尾机台现有换活字块匹配、结果追加及失败释放行为不变。
- 复用既有候选机台汇总、机台释放和 S4.5 新增排产衔接机制。

## 3. 现有逻辑影响点

### 3.1 排程入口

- `ContinuousProductionHandler.doHandle(...)`
  - S4.4.1 完成续作及收尾排产。
  - S4.4.2 调用 `TypeBlockProductionStrategy.scheduleTypeBlockChange(...)`。
  - S4.5 后续由现有新增排产处理器消费未被换活字块阶段处理的 SKU 和机台。

该入口顺序不调整。

### 3.2 换活字块候选机台来源

`TypeBlockProductionStrategy.scheduleTypeBlockChange(...)` 当前汇总三类候选机台：

1. 已标记收尾且存在预计完工时间的机台；
2. 续作首日无计划或收尾小余量后释放的机台；
3. 根据在机前规格进行兜底反查的机台。

三类来源均需要执行相同的 T 日判断，避免任一来源绕过规则。

### 3.3 不修改范围

- 换活字块候选 SKU 的同胎胚、同模具、机台硬性准入规则；
- 定点机台、新增换模主链预留规则；
- 模具资源占用、换模次数均衡和回滚逻辑；
- 胎胚库存、日计划额度、目标量和班次分配逻辑；
- 续作收尾、续作释放和 S4.5 新增选机逻辑；
- SQL、Mapper、XML、数据库表及配置项。

## 4. 设计思路

### 4.1 日期口径

第一天 T 日使用 `context.scheduleDate`，不使用仅作为业务保存日期的 `context.scheduleTargetDate`。

判断时将机台 `estimatedEndTime` 和 `context.scheduleDate` 按系统时区转换为 `LocalDate` 后比较，只比较自然日期，不比较具体时分秒：

- T 日 00:00:00 至 23:59:59：允许进入换活字块候选；
- T+1 日 00:00:00 起：不允许进入换活字块候选；
- T-1、T+2 及其他日期：同样不允许进入。

### 4.2 统一候选过滤

在三类机台来源完成汇总后、候选排序和 SKU 匹配前执行统一过滤。过滤方法只判断日期，不修改机台状态和上下文集合。

这样可以保证：

- 所有换活字块候选来源口径一致；
- 非 T 日机台不会进入定点 SKU、普通候选、模具切换和目标量计算；
- 被过滤机台仍保留在 `machineScheduleMap`，自然进入后续 S4.5 新增排产选择范围；
- 对应 SKU 仍保留在 `newSpecSkuList`，不会产生未排结果或被提前消费。

### 4.3 日志

候选准备阶段增加汇总日志，至少包含：

- T 日；
- 过滤前候选数量；
- 非 T 日收尾跳过数量；
- 被跳过机台编码及预计收尾时间摘要。

日志只用于排查，不吞异常，不改变业务结果。

## 5. 详细实施步骤

1. 在现有换活字块回归测试中先增加 T+1、T+2 收尾机台不参与换活字块的失败用例。
2. 补充 T 日边界用例，确认 T 日 23:59 仍可进入、T+1 00:00 已被排除。
3. 在 `TypeBlockProductionStrategy` 中增加小型私有日期判断方法。
4. 在候选机台汇总后统一过滤非 T 日收尾机台，并补充中文注释和汇总日志。
5. 运行换活字块定向回归测试，确认现有同胎胚、同模具、定点和释放机台场景不受影响。
6. 执行差异检查，确保没有无关格式化、临时文件或死代码。

## 6. 数据处理说明

- 数据初始化：继续使用 `MachineScheduleDTO.estimatedEndTime` 和 `LhScheduleContext.scheduleDate`，不新增上下文字段。
- 月计划和日计划：非 T 日机台在 S4.4 不消费对应额度，待 S4.5 按原规则计算和扣减。
- 胎胚库存：非 T 日机台不进入换活字块结果追加，因此不提前扣减库存。
- 模具状态：非 T 日机台不申请换活字块切换窗口，不占用模具和换模均衡配额。
- 机台状态：过滤过程不改写机台收尾标识、预计完工时间或分配状态。
- 结果回写：非 T 日机台不生成 `SCHEDULE_TYPE=03` 结果，也不新增未排记录。

## 7. 边界场景

- T 日零点和 T 日最后一秒收尾；
- T+1 零点收尾；
- T+1、T+2 收尾；
- 预计完工时间为空；
- 收尾、续作释放、兜底反查三类来源出现相同机台；
- T 日机台不满足同胎胚或同模具条件时，仍按现有逻辑释放给 S4.5；
- 非 T 日机台存在定点物料时，不在 S4.4 提前预留或消费，交由 S4.5 处理；
- 所有候选均为非 T 日时，换活字块阶段不生成结果，新增待排列表保持不变。

## 8. 风险点

- 若误用 `scheduleTargetDate`，会把业务保存日期 T+1 当作第一天，导致规则整体偏移一天。
- 若只过滤收尾机台主来源，续作释放或兜底机台仍可能提前进入换活字块。
- 若过滤时修改机台状态或待排 SKU，会破坏 S4.5 新增排产衔接。
- 日期转换必须沿用系统时区，避免跨日边界判断漂移。

## 9. 验证建议

- 定向运行 `ContinuousProductionTypeBlockRegressionTest`。
- 重点断言非 T 日机台没有生成换活字块结果，SKU 仍保留在 `newSpecSkuList`。
- 回归已有 T 日换活字块成功、匹配失败释放、定点预留、模具配额回滚、胎胚库存扣减场景。
- 执行 `git diff --check`，确认差异格式正确。
- 本次规则本身不要求数据库或接口真实复跑；若后续指定业务日期，可再按指定日期启动应用并调用 `/lhScheduleResult/execute` 完成结果表、未排表、换模表和过程日志闭环验证。
