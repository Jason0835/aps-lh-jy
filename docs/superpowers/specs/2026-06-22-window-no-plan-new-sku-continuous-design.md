# 窗口无计划新增拦截与续作余量排完设计

## 1. 明确结论

本次采用“续作与新增分类后分别处理”的方案：

- 新增 SKU 在 T～T+2 无日计划、窗口后仍有月计划、本月历史欠产为 0、有效上月欠产不大于 0 时，不进入新增排产，直接写入未排结果。
- 续作 SKU 不受上述新增准入限制；只要仍有硫化余量，就继续复用项目现有续作排产、收尾、降模和增机台逻辑。
- 续作余量排完后，复用现有释放机台机制，将续作机台交给换活字块或其他新增 SKU。
- 本规则不属于 SKU 提前生产，不修改 `EarlyProductionChecker`，不改变 T～T+2 窗口定义。

## 2. 改造目标

解决按产能满排模式把窗口外未来计划提前消耗的问题，同时保留续作生产连续性：

1. 防止无窗口需求、无欠产来源的新增 SKU 因月计划余量被提前上机。
2. 已在机续作 SKU 有硫化余量时继续排完，不能因窗口 dayN 为 0 直接释放。
3. 续作排完后及时释放机台，不阻塞其他 SKU 的换活字块或新增排产。
4. 不新增特殊增机规则，续作是否增机及增机数量完全以项目现有逻辑为准。

## 3. 业务判定口径

### 3.1 新增 SKU 不排产条件

未命中 MES 在机续作的 SKU 同时满足以下条件时，本窗口不排产：

- `windowPlanQty <= 0`：T～T+2 原始日计划合计为 0；
- `futureMonthPlanQtyAfterWindow > 0`：T+3 到月底仍有计划，说明不是本月整体收尾；
- `monthlyHistoryShortageQty <= 0`：本月 T 日前无历史欠产；
- `effectiveLastMonthOverdueQty <= 0`：无有效上月欠产；上月超产为负数时同样不得成为提前排产依据。

命中后写入未排结果，未排数量沿用项目“窗口无计划”口径记 0，原因明确记录为：

`T～T+2窗口无日计划且无欠产，窗口后仍有月计划，本次不排产`

### 3.2 不应被拦截的新增场景

以下任一场景继续复用现有排程逻辑：

- T～T+2 存在日计划；
- 本月存在历史欠产；
- 存在有效上月欠产；
- T+3 到月底也没有计划，属于现有整体收尾判断范围；
- SKU 已匹配为续作。

### 3.3 续作 SKU 处理

续作 SKU 在窗口无日计划时按硫化余量处理：

- `surplusQty > 0`：不得命中“窗口无计划直接释放”，继续进入现有续作收尾、目标量、日计划账本同步、降模和增机逻辑，直到本轮可排余量完成或窗口产能耗尽。
- `surplusQty <= 0` 且无本月历史欠产：沿用现有窗口无计划释放逻辑，写入未排并释放机台。
- 续作余量在窗口内排完：沿用现有续作完成后的机台释放、换活字块承接及 S4.5 新增选机逻辑。
- 续作余量超过窗口或现有机台能力：不新增本规则专属处理，完全交由项目已有续作增机和未排逻辑。

## 4. 现有逻辑影响点

### 4.1 `ScheduleAdjustHandler`

- 继续在 S4.3 构建完整 `SkuScheduleDTO`，以便 MES 在机匹配能够识别真实续作 SKU。
- 在 `classifyContinuousAndNewSkus` 完成续作匹配后，只对未匹配续作、准备进入 `newSpecSkuList` 的 SKU 应用新增准入判断。
- 命中不排条件时写入未排，不加入 `newSpecSkuList`，并从结构待排视图及运行态有效胎胚集合中移除，避免影响结构收尾、共用胎胚和候选排序。
- 将有效上月超欠产数量保存在 `SkuScheduleDTO`，供分类后的新增准入判断使用。

### 4.2 `SkuScheduleDTO`

- 增加有效上月超欠产运行态字段。
- 多机台续作复制 DTO 时必须同步复制该字段，避免同一物料不同续作机台判断口径不一致。

### 4.3 `ContinuousProductionStrategy`

- 调整 `shouldReleaseWindowNoPlanContinuousSku`：除“窗口无计划、无历史欠产”外，还必须确认硫化余量不大于 0 才直接释放。
- 硫化余量大于 0 时继续走现有收尾目标量上调和日计划账本同步，避免结果被 0 日计划回裁。
- 不修改现有续作增机、补偿、降模和机台释放实现。

### 4.4 不修改范围

- `EarlyProductionChecker` 与 SKU 提前生产备注；
- 新增排产候选机台、模具、胎胚、首检和换模限制；
- 续作增机台算法；
- SQL、Mapper XML、数据库表结构和参数配置。

## 5. 数据流

1. S4.3 加载月计划、完成量、上月超欠产和 MES 在机数据。
2. 构建 SKU 时保留窗口计划、窗口后计划、本月历史欠产、有效上月超欠产和硫化余量。
3. 先按 MES 在机信息匹配续作 SKU。
4. 已匹配续作 SKU 进入 S4.4：有余量继续排，排完后释放机台。
5. 未匹配续作的 SKU 准备进入新增列表前执行窗口无计划准入：命中则写未排并移出待排运行态。
6. 其余新增 SKU 继续进入 S4.4 换活字块和 S4.5 新增排产主链。

## 6. 日志要求

新增 SKU 被拦截时输出一条 INFO 日志，至少包含：

- 工厂；
- 排程窗口；
- 物料编码；
- T～T+2 计划量；
- T+3 到月底计划量；
- 本月历史欠产；
- 有效上月超欠产；
- 硫化余量；
- 判断结果和未排原因。

续作窗口无计划但因有余量继续排产时输出一条 INFO 日志，至少包含物料、机台、余量、窗口计划量和处理结果。

## 7. 测试设计

### 7.1 新增 SKU

- 窗口计划 0、窗口后有计划、两类欠产均为 0：不进入新增列表，写入未排。
- 本月历史欠产大于 0：继续进入新增排产。
- 有效上月欠产大于 0：继续进入新增排产。
- 窗口内有计划：继续进入新增排产。
- 窗口及月底均无计划：保留既有整体收尾逻辑。

### 7.2 续作 SKU

- 窗口计划 0、窗口后有计划、硫化余量大于 0：继续生成续作结果，不写“窗口无计划”未排。
- 续作余量排完后，原机台进入释放集合，可被其他 SKU 承接。
- 窗口计划 0、硫化余量为 0、无历史欠产：不生成续作结果，写入未排并释放机台。
- 续作增机行为保持现有测试结果，不增加本规则专属机台。

### 7.3 真实复跑

以 `factoryCode=116`、`scheduleDate=2026-06-14` 复跑并核对：

- 新增场景下 `3302002637` 在结果表为 0 条；
- 未排表存在 1 条明确原因记录；
- 不生成以 `3302002637` 为目标物料的换模计划；
- 日计划滚动台账不再把月余量 276 注入 2026-06-13；
- 若构造或存在真实续作场景，则有余量时继续续作，排完后的机台可被其他 SKU 使用。

## 8. 风险与控制

- 风险：过早剔除 SKU 会导致 MES 在机续作无法登记释放。控制：必须先完成续作匹配，再只拦截未匹配续作的新增 SKU。
- 风险：拦截条件扩大到月底无计划场景会破坏整体收尾。控制：要求 `futureMonthPlanQtyAfterWindow > 0`。
- 风险：只改续作释放条件但未同步收尾账本会被 0 日计划回裁。控制：继续复用现有 `upsizeEndingTargetQty` 及账本同步逻辑。
- 风险：被拦截新增 SKU 残留在共用胎胚有效集合中。控制：同步移除结构待排和活动胎胚运行态。

## 9. 验收标准

- 新增 SKU 不再提前消耗 T+3 到月底计划。
- 续作 SKU 有硫化余量时继续排产，余量完成后释放机台。
- 续作增机逻辑与现有项目保持一致。
- 未排结果、过程日志和最终四表数据可对账。
- focused tests、模块编译、`git diff --check` 和 2026-06-14 真实复跑均通过。
