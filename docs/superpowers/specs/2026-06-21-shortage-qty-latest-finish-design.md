# 仅历史欠产 SKU 最近一次完成量跳过规则设计

## 1. 明确结论

复用现有本月日完成量数据 `materialMonthDailyFinishedQtyMap`，在 S4.5 新增排产前置判断中，从排程日 T-1 向当前月月初倒序查找 SKU 最近一次非空日完成量。若完成量大于 0，且 SKU 同时满足“仅历史欠产、T～月底无计划”，则本次不再新增排产，并写入未排结果。

## 2. 改造目标

将原来固定读取 T-1 日完成量的判断，调整为读取当前月月初至 T-1 范围内最近一次非空日完成量，解决 T-1 无完成数据但更早一次已有完成量时 SKU 被再次排产的问题。

目标场景为排程日期 2026-06-14、物料 3302001139：月计划 T～T+2 日计划量均为 0，且后续至月底无计划量；若当前月最近一次非空日完成量大于 0，则不排产并进入未排结果。

## 3. 现有逻辑影响点

- 新增排产入口：`NewSpecProductionStrategy#schedulePendingNewSpecsRound`。
- 跳过规则：`shouldSkipHistoryShortageOnlyPreviousProducedSku` 及其完成量解析方法。
- 数据初始化：`LhBaseDataServiceImpl#loadMaterialMonthFinishedQty` 已加载月初至 T-1 的逐日完成量。
- 上下文数据：`LhScheduleContext#materialMonthDailyFinishedQtyMap`。
- 未排结果：复用 `addUnscheduledResult`，不新增落库链路。
- 日完成量表：`T_LH_DAY_FINISH_QTY`，不新增 Mapper 或 XML。
- 不修改续作、换模、换活字块、胎胚库存、日计划扣减和机台匹配逻辑。

## 4. 设计思路

### 4.1 数据范围

仅查询当前排程月份月初至 T-1，不跨月读取历史完成量。该范围与“本月历史欠产”业务口径一致。

### 4.2 最近一次非空完成量

- 从 T-1 向月初逐日倒序读取 `materialMonthDailyFinishedQtyMap`。
- `DAY_FINISH_QTY` 有值即视为有效数据；数值为 0 也是有效的最近一次数据。
- 若最近记录完成量为 0，则不继续向更早日期寻找大于 0 的记录，本次不因该规则跳过。
- `DAY_FINISH_QTY` 为 `NULL` 时不纳入逐日完成量 Map，继续向前寻找最近一次非空数据。
- 当前月不存在任何非空完成量时，保持原新增排产判断。

### 4.3 跳过条件

同时满足以下条件时跳过：

1. SKU 不是续作补偿 SKU；
2. 本月历史欠产量大于 0；
3. 当前窗口 dayN 计划量均为 0；
4. T+3 至月底无后续计划量；
5. 当前月月初至 T-1 最近一次非空日完成量大于 0。

### 4.4 未排与日志

- 未排数量复用本月历史欠产量。
- 未排原因固定为：`仅历史欠产、后续无月计划，且最近一次（前一次）已有完成量，本次跳过不排`。
- 日志包含 SKU、欠产量、排程日期、最近一次日完成量和跳过原因。

## 5. 数据处理说明

`loadMaterialMonthFinishedQty` 继续一次查询月初至 T-1 的完成数据，同时维护月累计完成量和逐日完成量。月累计完成量原有计算语义不变；逐日 Map 只聚合完成量字段非空的记录，用于区分“完成量为 0”和“没有有效完成量数据”。排产阶段只读上下文 Map，不新增数据库访问。

命中规则后，SKU 从新增待排队列移除并写入未排结果，不进入后续收尾目标量、胎胚库存、选机、换模和日计划账本消费流程。

## 6. 边界场景

- T-1 无记录、T-2 完成量大于 0：跳过排产。
- 最近一次非空完成量为 0、更早日期完成量大于 0：以最近一次 0 为准，继续原排产逻辑。
- T-1 完成量为空、T-2 完成量大于 0：忽略空值并命中 T-2，跳过排产。
- 当前月没有非空完成量：继续原排产逻辑。
- T～月底存在任意计划量：不适用该规则。
- 续作补偿 SKU：不适用该规则。
- 月初 T 日：查询范围为空，不适用该规则。

## 7. 风险点

- 若把 0 误判为无数据，会错误回溯到更早的大于 0 记录并拦截排产，因此必须保留 0 的日期键。
- 若跨月查找，会用上月完成量拦截本月欠产补排，因此必须限制在当前月。
- 逐日 Map 还供历史欠产统计读取；调整空值处理时必须保证月累计完成量和已有历史欠产计算结果不变。

## 8. 验证建议

先以失败测试覆盖最近非空完成量的倒序查找、0 值停止和空值跳过，再做最小实现。完成后执行相关定向测试、编译、OpenSpec 严格校验和 `git diff --check`。如本地服务和数据库可用，再对 2026-06-14 发起真实排程，并核对物料 3302001139 在结果表中不存在、在未排表中存在且原因正确。
