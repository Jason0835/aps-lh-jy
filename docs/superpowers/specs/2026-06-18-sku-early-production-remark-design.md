# SKU 提前生产结果备注设计

## 1. 目标

当新增 SKU 命中提前生产相关规则并实际生成硫化排程结果时，将该 SKU 所属结构在排程窗口 T～T+2 的计划硫化机台数写入 `T_LH_SCHEDULE_RESULT.REMARK`。

本次只扩展判定结果和结果备注，不改变提前生产准入、候选机台、模具、胎胚、换模、换活字块、班次产能和日计划扣减规则。

## 2. 备注格式

- 普通提前生产：`结构计划硫化机台数：2,3,4`
- 结构切换：`[结构切换] 结构计划硫化机台数：2,3,4`
- 结构收尾：`[结构收尾] 结构计划硫化机台数：2,3,4`
- 已有备注时使用中文分号追加：`原备注；[结构切换] 结构计划硫化机台数：2,3,4`

三个机台数严格按排程目标日 T、T+1、T+2 顺序读取。某日没有结构计划时保留 `0`，不省略日期位置。

## 3. 判定结果设计

将 `EarlyProductionChecker` 当前仅返回布尔值的准入判断扩展为结构化判定结果，至少包含：

- 是否属于提前生产场景；
- 是否允许进入当前日新增机台判断；
- 场景类型：普通提前生产、结构切换、结构收尾；
- T～T+2 结构计划硫化机台数；
- 后续首个日计划日期和判定原因，用于保留现有对账日志。

保留现有布尔准入方法作为兼容入口，内部委托结构化判定，避免扩大已有调用点的改动范围。

## 4. 数据流

1. 候选机台确定首个可排业务日后，执行一次提前生产结构化判定。
2. 同一个判定结果同时用于首个可排时间对齐和当前候选机台的结果备注，避免二次判断产生口径差异。
3. 新增机台通过现有资源校验、结果回裁并确认计划量大于 0 后，才向该条 `LhScheduleResult.remark` 追加备注。
4. 同一 SKU 实际新增多台机时，每条成功结果均追加相同场景备注，保证每条落库结果可独立审计。
5. 准入失败、后续未成功排产、当前日本身有日计划以及非新增排产结果均不写该备注。

## 5. 场景口径

- 当前日结构计划机台数大于 0，且结构已排机台数未达计划：普通提前生产。
- 当前日结构计划机台数为 0，改用后续首个 SKU 计划日的结构机台数并准入：结构切换。
- 当前日和后续首个计划日结构机台数均为 0，且命中结构收尾大余量条件：结构收尾。
- 历史欠产超过阈值时沿用原强制加机台逻辑；若该 SKU 属于后续日 SKU，则记录普通提前生产备注。该分支未执行结构判断，不标记为结构切换或结构收尾。

## 6. 追加与防重复

备注追加复用统一小方法：

- 原备注为空时直接写入新片段；
- 原备注非空时追加中文分号和新片段；
- 原备注已包含完全相同片段时不重复追加；
- 不截断、不覆盖已有备注，不增加异常吞并或默认兜底。

## 7. 测试与验证

自动化测试覆盖：

- 普通提前生产生成三个结构机台数；
- 结构切换生成指定格式；
- 结构收尾生成指定格式并保留 `0`；
- 已有备注正确追加；
- 相同片段不重复追加；
- 未命中提前生产或新增排产失败时不写备注；
- 兼容原布尔准入结果。
