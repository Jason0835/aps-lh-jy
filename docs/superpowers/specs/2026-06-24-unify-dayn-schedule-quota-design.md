# 月计划日计划量 dayN 口径统一技术设计

## 1. 明确结论

本次采用“中心语义收口”方案：保留日计划账本和 `DailyMachineExpansionPlanner` 中已有的节奏判断能力，继续让 `dayN` 参与可开始排、提前生产准入、加机台、降模减机台和结果对账；同时清理新增、续作、换活字块等实际排产量落点中把 `dayN`、窗口日计划量或账本剩余额度当作非收尾硬上限的旧逻辑。

核心边界是：

- `dayN` 可以决定是否进入排产节奏判断；
- `dayN` 不得直接决定当天实际排产量；
- `dayN` 不得覆盖收尾目标量、硫化余量、胎胚库存、欠产追补和晚班不可换模补满规则。

## 2. 改造目标

本次要解决历史版本中的旧口径残留：

1. 非收尾 SKU 因 `dayN` 较小被浅排，导致可用班次产能浪费；
2. 续作、新增、换活字块链路中存在将窗口日计划量或账本剩余额度误当实际排产目标的风险；
3. 加机台和降模减机台需要继续使用 `dayN` 判断节奏，但不能反向限制最终结果；
4. 提前生产只能前移一天的计划节奏，不能把 T+2 或更远计划提前到 T 日；
5. 日计划账本和结果校验日志要保留对账能力，但不能成为后置硬控量依据。

## 3. 现有逻辑影响点

需要重点检查和修改的现有逻辑：

- `ScheduleAdjustHandler`
  - 初始化 `dailyPlanQuotaMap`、`windowPlanQty`、历史欠产、T 日晚班完成量。
  - 本次只确认入口口径，不重写账本结构。

- `DailyMachineExpansionPlanner`
  - 保留当前日/后看日计划节奏判断。
  - 保留窗口无计划、仅补历史欠产、月底无计划收尾清量等已有语义。
  - 增强日志，明确其输出是“是否需要加机台”，不是“实际排多少”。

- `TargetScheduleQtyResolver`
  - 重点排查 `windowPlanQty`、`windowRemainingPlanQty`、日计划账本剩余额度是否被用于非收尾实际排产硬上限。
  - 收尾和仅补历史欠产的严格目标仍可同步到账本，因为它们是业务目标量，不是普通 `dayN` 控量。

- `ContinuousProductionStrategy`
  - 续作排产、降模、日标准产量修正、续作账本消费、补偿 SKU 回流、共用胎胚未排。
  - 需要区分目标量缺口和 `dayN` 节奏缺口，防止补偿 SKU 只因账本剩余额度产生，也防止非收尾续作被 `dayN` 回裁为浅排。

- `NewSpecProductionStrategy`
  - 新增排产目标量、候选机台、换模首检、提前生产结果标识、账本消费。
  - 需要清理新增结果生成和回裁中把 `dayN` 当作非收尾实际上限的逻辑。

- `TypeBlockProductionStrategy`
  - 换活字块实际结果和换活字块回流新增链路。
  - 需要确认不按 `dayN` 硬截断可用班次产量，同时继续遵守活字块、模具、机台和换模约束。

- `EarlyProductionChecker` / `SkuDailyPlanQuotaUtil`
  - 保持提前生产只提前一天。
  - 临时前移视图只用于准入、节奏判断和加机台模拟，不写回原始账本，不截断实际排产量。

- `ResultValidationHandler`
  - 日计划完成校验和滚动台账继续保留。
  - 日志仅用于对账，不作为后置截断非收尾实际排产量的依据。

## 4. 设计思路

### 4.1 语义分层

将 `dayN` 的运行期作用分成三层：

1. 原始计划层：`ScheduleAdjustHandler` 从月计划 day1~day31 初始化日计划账本。
2. 节奏判断层：`DailyMachineExpansionPlanner`、`EarlyProductionChecker`、降模判断使用 `dayN` 判断是否可排、是否加机台、是否降模。
3. 实际排产层：新增、续作、换活字块生成结果时，按场景目标量和资源约束决定排产量。

实现时只调整第三层对 `dayN` 的误用，不破坏第一层和第二层。

### 4.2 收尾与非收尾分开

收尾 SKU：

- 单胎胚/非共用胎胚目标量 = `MAX(硫化余量, 胎胚库存)`；
- 共用胎胚目标量 = `硫化余量`；
- 共用胎胚余量为 0 时不排，并进入未排；
- `dayN` 不得调低或抬高收尾目标量。

非收尾 SKU：

- 不按 `dayN` 硬控量；
- 继续结合硫化余量、可用产能、班产、欠产追补、晚班不可换模、机台/模具/单控约束计算；
- 规则允许且有可用班次产能时，应尽量排满可用班次。

### 4.3 加机台和降模保持节奏判断

加机台：

- 当前机台数满足当前日 `dayN` 节奏时，不继续加机台；
- 当前机台数不满足当前日节奏时，再进入现有新增机台规则；
- `dayN` 判断结果不直接控制本次实际排产量。

降模减机台：

- 多机台续作按当前日和后看日计划节奏判断是否冗余；
- 保留机台数刚好满足 T 日当前日计划节奏时停止继续减少；
- 减机台后的重分配仍按续作目标量和资源约束执行。

### 4.4 提前生产只前移节奏

提前生产通过后，只构造当前轮次使用的临时前移计划视图：

```text
shifted[T] = original[T+1]
shifted[T+1] = original[T+2]
shifted[T+2] = original[T+3]
```

该视图仅用于准入、节奏和加机台判断。T+2 或更远日期不得直接提前到 T 日。实际排产量仍按收尾/非收尾规则和资源约束计算。

## 5. 详细实施步骤

1. 使用 CodeGraph 和定向搜索完成只读定位，列出所有 `dayN`、`windowPlanQty`、`dailyPlanQuotaMap`、账本剩余额度参与目标量或结果回裁的点。
2. 先补 focused tests：
   - 非收尾 `dayN` 小于可用产能仍按可用产能排；
   - 收尾目标量不被 `dayN` 调低或抬高；
   - 当前机台满足当前日节奏时不加机台；
   - 续作降模保留机台满足 T 日节奏后停止；
   - 提前生产只提前一天；
   - 换活字块链路不按 `dayN` 硬截断。
3. 调整新增排产实际目标量和结果回裁点，清理非收尾 `dayN` 硬上限。
4. 调整续作排产、降模、补偿回流逻辑，区分目标量缺口和节奏缺口。
5. 调整换活字块排产及回流新增链路，保持活字块/模具/机台约束不变。
6. 校正提前生产链路，确保前移视图只用于节奏判断。
7. 补充中文日志和必要注释。
8. 同步 `openspec/specs` 下主规格。
9. 运行定向测试、OpenSpec 校验、`git diff --check`。
10. 按 `2026-06-14` 真实复跑并对账结果表、未排表、换模表和过程日志。

## 6. 数据处理说明

本次不新增数据库表和字段，不改变月计划原始 day1~day31 数据。

运行期数据处理口径：

- `dailyPlanQuotaMap` 继续由月计划 `dayN` 初始化；
- 历史欠产、继承量、T 日晚班完成量继续按现有账本逻辑扣减或入账；
- 提前生产临时前移视图只存在于当前判断过程，不回写原始账本和月计划表；
- 实际结果生成后仍可消费账本用于滚动对账；
- 对收尾和仅补历史欠产的严格目标，允许同步到账本以避免后续链路回裁为旧窗口量；
- 对普通非收尾 SKU，账本剩余额度不得反向成为实际排产硬上限。

## 7. 边界场景

- `dayN` 很小但非收尾 SKU 有足够硫化余量、胎胚库存和可用班次产能；
- 收尾目标量大于当前日 `dayN`；
- 共用胎胚 SKU 硫化余量为 0 但后续仍有 `dayN`；
- 当前日无计划、T+1 有计划且满足提前生产；
- 当前日无计划、T+2 才有计划；
- 多机台续作降模后保留机台刚好满足 T 日节奏；
- 晚班不可换模导致非收尾续作需要补满晚班；
- 换活字块释放或承接后进入新增链路；
- 日标准产量修正后结果不被 `dayN` 二次回裁；
- 2026-06-14 真实复跑中的续作、新增、窗口无计划和提前生产样例。

## 8. 风险点

- 非收尾放开 `dayN` 硬控后，实际排产量可能高于历史结果；这是口径预期变化，需要通过 2026-06-14 复跑验证业务正确性。
- 如果误删日计划账本消费，会破坏提前生产、滚动补欠产、日计划完成校验和过程日志。
- 续作补偿 SKU 与来源续作 SKU 共享账本，若只看账本剩余额度可能误生成补偿，需要同时看目标量缺口和节奏缺口。
- 换活字块可能复用新增 helper，局部修复新增入口仍可能遗漏换活字块链路。
- 主规格分散在多个 capability，归档时必须同步主规格，否则规则会再次漂移。

## 9. 验证建议

定向验证：

- focused tests 覆盖新增、续作、换活字块、提前生产、日计划账本消费。
- `openspec validate unify-dayn-schedule-quota --strict`
- `git diff --check`

真实复跑：

```bash
curl -X POST 'http://localhost:9669/lhScheduleResult/execute' \
  -H 'Content-Type: application/json' \
  -d '{"factoryCode":"116","scheduleDate":"2026-06-14"}'
```

复跑后至少核对：

- `T_LH_SCHEDULE_RESULT`：非收尾是否仍被 `dayN` 浅排，收尾是否严格不超排；
- `T_LH_UNSCHEDULED_RESULT`：共用胎胚零余量和窗口无计划是否进入未排；
- `T_LH_MOULD_CHANGE_PLAN`：换模/换活字块时间和动作类型是否符合约束；
- `T_LH_SCHEDULE_PROCESS_LOG`：`dayN` 节奏判断、加机台、降模、提前生产、目标量和实际排产量是否可追溯。
