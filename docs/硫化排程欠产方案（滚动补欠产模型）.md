# 业务背景

当前项目：

- 一次排程窗口为 8 个班次
- 按 dayN 月计划控制每日计划量
- SKU 支持多机台生产
- 同一个 SKU 可能同时分配到多个机台
- 允许未来计划预占（提前生产未来 dayN 的量）
- 但绝对不允许超出窗口总计划量

例如：

排程日期：2026-05-03

月计划窗口：

```text
day1 = 96
day2 = 48
day3 = 14
````

窗口总需求：

```text
158
```

SKU：

```text
3302001724
```

班产：

```text
16
```

机台分配：

| 机台    | 第一天 | 第二天 | 第三天 |
| ----- | --- | --- | --- |
| K1105 | 16  | 48  | 48  |
| K1110 | 16  | 30  | 0   |
| 合计    | 32  | 78  | 48  |

实际目标：

* 第一天产能不足，只生产 32
* 欠产 64 滚动到第二天
* 第二天允许提前吃未来计划，进行满产
* 第三天仅生产剩余尾量
* 最终总量不能超过窗口总计划量 158

最终结果：

| 日期  | 实际生产 |
| --- | ---- |
| 第一天 | 32   |
| 第二天 | 96   |
| 第三天 | 30   |
| 合计  | 158  |

窗口总量封顶

# 核心规则

## 1、窗口总量封顶（重要）

任意时刻：

```text
累计排产量 <= 窗口总需求量
```

即：

```text
cumulativeProduced <= windowTotalPlan
```

禁止超排。

---

# 2、欠产滚动机制

每日目标量：

```text
dailyNeed =
当日月计划量
+
历史累计欠产量
```

若当天无法完成：

```text
carryLoss =
dailyNeed - actualProduction
```

欠产滚动到下一天继续生产。

---

# 3、允许未来计划预占（重要）

允许提前生产未来 dayN 的计划量。

例如：

第二天除了：

```text
day2 + carryLoss
```

之外，

还允许提前消耗：

```text
day3
```

但：

```text
累计生产总量不能超过窗口总计划量
```

---

# 4、每日最终允许生产量

公式：

```text
remainingDemand =
windowTotalPlan - cumulativeProduced
```

```text
futureAvailable =
未来剩余 dayN 计划量
```

```text
maxAllowed =
dailyNeed + futureAvailable
```

```text
actualProduction =
min(
    当日产能,
    maxAllowed,
    remainingDemand
)
```

---

# 5、最后一天规则

最后一天：

* 若仍存在剩余未完成量
* 则形成最终欠产

字段：

```text
finalLossQty
```

---

# 6、多机台分配规则

同一个 SKU 多机台生产时：

## 优先级：

### 第一优先级

机台顺序：

```text
machineSort
```

---

### 第二优先级

连续生产优先：

* 减少换模
* 减少切换
* 尽量连续排产

---

### 第三优先级

尾量集中：

最后尾量优先集中到单机台。

例如：

推荐：

```text
K1105 = 30
K1110 = 0
```

不推荐：

```text
K1105 = 18
K1110 = 12
```

避免尾量碎片化。

---

# 7、重要实现原则（必须）

必须：

```text
先按“天”计算
再按“班次”拆分
最后按“机台”分摊
```

禁止：

```text
直接按班次处理欠产
```

否则：

* 多机台逻辑会失控
* 容易超排
* 欠产无法滚动
* 日计划无法控制
