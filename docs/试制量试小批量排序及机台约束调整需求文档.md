# 试制 / 量试 / 小批量排序及机台约束调整需求文档

## 1. 背景

当前硫化排程中，试制、量试、小批量 SKU 可能存在优先于正规 SKU 排产的倾向。为使排程逻辑更贴近实际生产，需要调整为：

- 试制、量试、小批量 SKU 不再天然优先于正规 SKU；
- 所有 SKU 统一进入现有 SKU 排序逻辑；
- SKU 类型差异不体现在 SKU 排序优先级上，而体现在机台选择约束上。

本次调整的核心原则是：

> SKU 排序统一，机台选择按类型约束；单控机台内部竞争时，按试制 > 量试 > 小批量的优先级占用。

---

## 2. 调整目标

### 2.1 排序目标

试制、量试、小批量、正规 SKU 全部按照现有 SKU 排序规则统一排序，不再单独将试制 / 量试 / 小批量提前到正规 SKU 前面。

即调整为：

```text
候选 SKU = 正规 SKU + 试制 SKU + 量试 SKU + 小批量 SKU
统一执行现有 SKU 排序规则
```

### 2.2 机台选择目标

不同 SKU 类型在选机台阶段增加约束：

- 试制 SKU：只能使用单控机台；
- 量试 / 小批量 SKU：优先使用单控机台，满足条件时可使用普通机台；
- 正规 SKU：优先使用普通机台，满足条件时可使用单控机台；
- 单控机台被试制 / 量试 / 小批量同时竞争时，内部优先级为：试制 > 量试 > 小批量。

---

## 3. 适用范围

本需求适用于硫化排程中 SKU 排序、SKU 选机台、机台候选过滤、机台优先级排序等相关逻辑。

涉及 SKU 类型包括：

| SKU 类型   | 说明                                     |
| ---------- | ---------------------------------------- |
| 正规 SKU   | 正常量产 SKU                             |
| 试制 SKU   | 只能安排到单控机台的试制类 SKU           |
| 量试 SKU   | 优先安排到单控机台，必要时可使用普通机台 |
| 小批量 SKU | 与量试 SKU 采用相同机台约束策略          |

涉及机台类型包括：

| 机台类型 | 说明                                 |
| -------- | ------------------------------------ |
| 单控机台 | 用于试制、量试、小批量优先排产的机台 |
| 普通机台 | 正规 SKU 优先使用的常规硫化机台      |

---

## 4. 核心规则

## 4.1 SKU 排序规则

### 调整前可能逻辑

```text
试制 / 量试 / 小批量 SKU 优先于正规 SKU
```

### 调整后逻辑

```text
试制 / 量试 / 小批量 SKU 与正规 SKU 一起参与现有 SKU 排序
```

也就是说，SKU 排序阶段不因为 SKU 类型是试制、量试、小批量而额外提前。

现有排序维度继续保持，例如：

- 月计划优先级；
- 欠产情况；
- 收尾 / 非收尾；
- 规格、花纹、特殊材料；
- 余量、胎胚库存；
- 其他项目已有排序规则。

本次需求不新增“试制 / 量试 / 小批量优先于正规”的排序层级。

---

## 4.2 机台约束总表

| SKU 类型   | SKU 排序方式                   | 单控机台   | 普通机台   | 规则说明                                         |
| ---------- | ------------------------------ | ---------- | ---------- | ------------------------------------------------ |
| 试制 SKU   | 与正规 SKU 统一排序            | 允许       | 不允许     | 强约束，只能排单控机台                           |
| 量试 SKU   | 与正规 SKU 统一排序            | 优先允许   | 有条件允许 | 单控机台优先，普通机台需让位正规 SKU             |
| 小批量 SKU | 与正规 SKU 统一排序            | 优先允许   | 有条件允许 | 规则同量试 SKU                                   |
| 正规 SKU   | 与试制 / 量试 / 小批量统一排序 | 有条件允许 | 优先允许   | 普通机台优先，单控机台需让位试制 / 量试 / 小批量 |

---

## 4.3 单控机台内部占用优先级

当试制、量试、小批量 SKU 同时需要竞争单控机台资源时，单控机台的占用优先级为：

```text
试制 SKU > 量试 SKU > 小批量 SKU
```

该优先级只用于“单控机台选择 / 占用 / 保留”阶段，不改变第 4.1 节的全局 SKU 排序规则。也就是说：

- SKU 列表仍然按现有统一排序逻辑排序；
- 当某个 SKU 准备占用单控机台时，需要判断是否存在更高优先级的试制 / 量试 / 小批量 SKU 仍需要单控机台；
- 如果存在更高优先级 SKU 仍待排且单控机台资源不足，则低优先级 SKU 不应抢占单控机台；
- 正规 SKU 使用单控机台的优先级低于试制 / 量试 / 小批量，只有在无待排试制 / 量试 / 小批量 SKU 时才允许占用。

单控机台内部竞争判断建议如下：

| 当前 SKU 类型 | 占用单控机台前需要确认                 | 说明                             |
| ------------- | -------------------------------------- | -------------------------------- |
| 试制 SKU      | 无更高优先级限制                       | 试制最高优先级，且只能单控       |
| 量试 SKU      | 不存在待排试制 SKU 需要单控机台        | 避免量试抢占试制单控资源         |
| 小批量 SKU    | 不存在待排试制 / 量试 SKU 需要单控机台 | 避免小批量抢占试制、量试单控资源 |
| 正规 SKU      | 不存在待排试制 / 量试 / 小批量 SKU     | 正规 SKU 最后才可使用单控        |

---

## 5. 分类型详细规则

## 5.1 试制 SKU

### 规则

试制 SKU 只能安排到单控机台，不能安排到普通机台。

```text
试制 SKU 候选机台 = 单控机台
```

### 处理逻辑

1. 获取可用单控机台；
2. 按现有机台排序规则选择机台；
3. 如果不存在可用单控机台，则试制 SKU 本轮不排产；
4. 不允许将试制 SKU 降级安排到普通机台。

### 示例

| 场景                         | 处理结果                 |
| ---------------------------- | ------------------------ |
| 有可用单控机台               | 可排产                   |
| 无可用单控机台，但有普通机台 | 不排产                   |
| 单控机台被占用               | 不排产或等待后续排程机会 |

---

## 5.2 量试 SKU

### 规则

量试 SKU 优先安排到单控机台。如果没有可用单控机台，可以考虑安排到普通机台，但普通机台必须优先保障正规 SKU。

在单控机台内部竞争时，量试 SKU 的优先级低于试制 SKU，高于小批量 SKU。

```text
量试 SKU 候选机台优先级：
1. 单控机台
2. 普通机台，需满足正规 SKU 已排完或无待排正规 SKU
```

### 处理逻辑

1. 优先查找可用单控机台；
2. 如果存在可用单控机台，先判断是否存在待排试制 SKU 需要单控机台；
3. 如果不存在待排试制 SKU，则量试 SKU 可以优先使用单控机台；
4. 如果存在待排试制 SKU 且单控机台资源不足，则量试 SKU 暂不抢占单控机台；
5. 如果无可用单控机台，或因试制 SKU 优先导致量试 SKU 暂不能占用单控机台，则判断是否允许使用普通机台；
6. 只有当没有待排正规 SKU，或正规 SKU 已排完后，量试 SKU 才允许占用普通机台；
7. 如果仍不满足条件，则量试 SKU 本轮不排产。

### 推荐落地口径

为了减少对正规量产排程的影响，推荐采用保守策略：

```text
只有当正规 SKU 已排完，量试 SKU 才允许占用普通机台。
```

---

## 5.3 小批量 SKU

### 规则

小批量 SKU 与量试 SKU 使用相同的普通机台约束策略，但在单控机台内部竞争时，小批量 SKU 优先级低于试制 SKU 和量试 SKU。

```text
小批量 SKU 候选机台优先级：
1. 单控机台
2. 普通机台，需满足正规 SKU 已排完或无待排正规 SKU
```

### 处理逻辑

1. 优先查找可用单控机台；
2. 如果存在可用单控机台，先判断是否存在待排试制 / 量试 SKU 需要单控机台；
3. 如果不存在待排试制 / 量试 SKU，则小批量 SKU 可以优先使用单控机台；
4. 如果存在待排试制 / 量试 SKU 且单控机台资源不足，则小批量 SKU 暂不抢占单控机台；
5. 如果无可用单控机台，或因试制 / 量试 SKU 优先导致小批量 SKU 暂不能占用单控机台，则判断普通机台是否可以被占用；
6. 普通机台必须优先保障正规 SKU；
7. 只有当没有待排正规 SKU，或正规 SKU 已排完后，小批量 SKU 才允许占用普通机台。

---

## 5.4 正规 SKU

### 规则

正规 SKU 优先安排到普通机台。如果普通机台不足，可以考虑安排到单控机台，但单控机台必须优先保障试制、量试、小批量 SKU。

```text
正规 SKU 候选机台优先级：
1. 普通机台
2. 单控机台，需满足无待排试制 / 量试 / 小批量 SKU
```

### 处理逻辑

1. 优先查找可用普通机台；
2. 如果存在可用普通机台，则优先使用普通机台；
3. 如果普通机台不足或无可用普通机台，则判断是否允许使用单控机台；
4. 只有当没有待排试制、量试、小批量 SKU，或这些 SKU 已无法继续占用单控机台时，正规 SKU 才允许使用单控机台；
5. 如果不满足条件，则正规 SKU 不占用单控机台。

---

## 6. 机台选择推荐流程

## 6.1 总体流程

```text
1. 构建候选 SKU 列表
2. 按现有 SKU 排序逻辑统一排序
3. 依次处理排序后的 SKU
4. 根据 SKU 类型过滤候选机台
5. 根据 SKU 类型调整机台优先级
6. 执行现有机台排序逻辑
7. 选择最终机台并生成排程结果
8. 如果无符合条件机台，则当前 SKU 本轮不排产
```

---

## 6.2 试制 SKU 选机台流程

```text
if SKU 是试制:
    候选机台 = 单控机台
    if 候选机台为空:
        本轮不排产
    else:
        按现有机台排序逻辑选择机台
```

---

## 6.3 量试 / 小批量 SKU 选机台流程

```text
if SKU 是量试:
    候选机台 = 可用单控机台

    if 候选机台不为空 且 不存在待排试制 SKU 需要单控机台:
        优先选择单控机台
    else:
        if 没有待排正规 SKU 或正规 SKU 已排完:
            候选机台 = 可用普通机台
        else:
            本轮不排产

if SKU 是小批量:
    候选机台 = 可用单控机台

    if 候选机台不为空 且 不存在待排试制 / 量试 SKU 需要单控机台:
        优先选择单控机台
    else:
        if 没有待排正规 SKU 或正规 SKU 已排完:
            候选机台 = 可用普通机台
        else:
            本轮不排产
```

---

## 6.4 正规 SKU 选机台流程

```text
if SKU 是正规:
    候选机台 = 可用普通机台

    if 候选机台不为空:
        优先选择普通机台
    else:
        if 没有待排试制 / 量试 / 小批量 SKU:
            候选机台 = 可用单控机台
        else:
            本轮不占用单控机台
```

---

## 7. 推荐确认口径

## 7.1 量试 / 小批量是否可以使用普通机台

建议确认为：

```text
可以，但必须在正规 SKU 优先排产之后。
```

具体为：

- 量试 / 小批量优先单控机台；
- 如果单控机台不可用，可以使用普通机台；
- 但普通机台必须优先保障正规 SKU；
- 只有当没有待排正规 SKU，或正规 SKU 已排完后，量试 / 小批量才允许使用普通机台。

---

## 7.2 正规 SKU 是否可以使用单控机台

建议确认为：

```text
可以，但必须在没有试制 / 量试 / 小批量 SKU 需要单控机台的情况下。
```

具体为：

- 正规 SKU 优先普通机台；
- 如果普通机台不足，可以考虑单控机台；
- 但单控机台必须优先保障试制 / 量试 / 小批量；
- 只有当没有待排试制 / 量试 / 小批量 SKU 时，正规 SKU 才允许使用单控机台。

---

## 7.3 单控机台优先级确认口径

建议确认为：

```text
单控机台优先保障顺序：试制 > 量试 > 小批量 > 正规。
```

具体为：

- 试制 SKU 只能使用单控机台，且在单控机台内部优先级最高；
- 量试 SKU 优先使用单控机台，但不能抢占试制 SKU 所需的单控资源；
- 小批量 SKU 优先使用单控机台，但不能抢占试制 / 量试 SKU 所需的单控资源；
- 正规 SKU 只有在无待排试制 / 量试 / 小批量 SKU 时，才允许使用单控机台。

## 8. 日志要求

为便于排查和验证，需要在关键节点增加日志。

### 8.1 SKU 排序日志

输出每个 SKU 的排序信息，包括：

- 物料编码；
- SKU 类型：正规 / 试制 / 量试 / 小批量；
- 是否收尾；
- 月计划量；
- 欠产量；
- 余量；
- 胎胚库存；
- 当前排序层级及排序值；
- 排序后的名次。

### 8.2 机台候选日志

输出 SKU 选机台过程，包括：

- 当前 SKU；
- SKU 类型；
- 机台类型要求；
- 初始候选机台列表；
- 过滤后的候选机台列表；
- 被过滤机台及过滤原因；
- 单控机台是否可用；
- 普通机台是否可用；
- 是否触发跨类型占用规则；
- 单控机台内部优先级判断结果，例如是否存在更高优先级 SKU 待排。

### 8.3 最终选机台日志

输出最终选择结果，包括：

- 最终选中机台；
- 选中原因；
- 是否因机台类型约束导致跳过；
- 如果未排产，需要输出未排产原因。

示例：

```text
[SKU选机台] SKU=3302000001, 类型=试制, 候选机台类型=单控机台, 可用单控机台=0, 普通机台=5, 结果=不排产, 原因=试制SKU禁止使用普通机台
```

```text
[SKU选机台] SKU=3302000002, 类型=量试, 单控机台不可用, 正规SKU待排数量=3, 结果=不排产, 原因=普通机台需优先保障正规SKU
```

```text
[SKU选机台] SKU=3302000003, 类型=正规, 普通机台不可用, 试制/量试/小批量待排数量=0, 结果=允许使用单控机台
```

```text
[SKU选机台] SKU=3302000004, 类型=小批量, 可用单控机台=1, 待排试制数量=0, 待排量试数量=2, 结果=不占用单控机台, 原因=单控机台优先保障量试SKU
```

## 9.8 单控机台优先保障试制 SKU

### 输入

- 试制 SKU：1 个待排；
- 量试 SKU：1 个待排；
- 小批量 SKU：1 个待排；
- 单控机台：1 台可用。

### 输入

- 小批量 SKU：1 个待排；
- 单控机台：1 台可用。

### 期望

单控机台优先分配给量试 SKU，小批量 SKU 不得抢占。

## 10. 实施建议

1. 尽量复用现有 SKU 排序逻辑，不新增试制 / 量试 / 小批量优先排序层级；
2. 将 SKU 类型带来的差异放在选机台阶段处理；
3. 将机台类型判断封装为独立方法，例如：
   - `isSingleControlMachine(machine)`；
   - `isNormalMachine(machine)`；
   - `isTrialSku(sku)`；
   - `isPilotSku(sku)`；
   - `isSmallBatchSku(sku)`；
   - `isRegularSku(sku)`；
   - `hasPendingTrialSkuNeedSingleControl()`；
   - `hasPendingPilotSkuNeedSingleControl()`；
   - `hasHigherPrioritySingleControlSku(currentSku)`；
4. 对跨类型占用规则进行集中判断，避免散落在多个地方；
5. 增加日志，确保可以清楚看到 SKU 排序、机台过滤、最终选机台、跳过原因；
6. 保持最小改动，避免大范围重构现有排程主流程。

---

## 11. 最终需求口径

试制、量试、小批量 SKU 不再单独提前于正规 SKU 排序，全部 SKU 按现有 SKU 排序规则统一排序。

机台选择阶段按 SKU 类型增加约束，并增加单控机台内部占用优先级：

```text
试制 > 量试 > 小批量 > 正规
```

该优先级只用于单控机台资源竞争，不改变全局 SKU 排序。


1. 试制 SKU：
   - 只能安排到单控机台；
   - 不允许安排到普通机台；
   - 如果无可用单控机台，则本轮不排产。

2. 量试 SKU / 小批量 SKU：
   - 优先安排到单控机台；
   - 如果无可用单控机台，可以考虑安排到普通机台；
   - 但普通机台必须优先保障正规 SKU；
   - 只有当没有待排正规 SKU，或正规 SKU 已排完后，量试 / 小批量 SKU 才允许占用普通机台。

3. 正规 SKU：
   - 优先安排到普通机台；
   - 如果普通机台不足，可以考虑安排到单控机台；
   - 但单控机台必须优先保障试制 / 量试 / 小批量 SKU；
   - 只有当没有待排试制 / 量试 / 小批量 SKU 时，正规 SKU 才允许占用单控机台。
